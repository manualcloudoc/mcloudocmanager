# BackupDoc - 슈퍼관리자가 백업관리자 계정 관리하는 방법

​\
슈퍼관리자는 백업 관리자 계정 목록을 조회하고 백업관리자 계정을 추가, 삭제하거나, 계정의 기본 정보, 계정에 할당된 백업 용량과 원본 서버 정보를 변경할 수 있습니다.\
​

## 백업관리자 계정 조회

1\.   BackupDoc 관리자 웹페이지에서 **계정 관리** 메뉴를 선택하면 등록되어 있는 모든 백업 관리자 계정의 목록이 표시됩니다.\
2\.   상단의 검색창에 백업 관리자의 이름, 아이디, 이메일, 또는 회사명을 입력한 후 검색을 클릭하면 검색 조건에 해당하는 계정 정보가 목록에 표시됩니다.\
​​3.   계정 목록에서 이름 또는 아이디를 클릭하면 해당 계정에 대한 상세 정보(계정 기본 정보, 용량 상세 목록, 원본 서버 목록)가 표시됩니다. 상세 정보에 대한 내용은 아래 [**계정 상세 정보 확인 및 수정**](backupdoc-2.md) 목차를 참고합니다.​\
![](<../../.gitbook/assets/img_000 (15).png>)

## 백업 관리자 계정 추가

1\.   계정 관리 화면에서 계정 목록 하단의 추가 버튼을 클릭하면 ‘계정 추가’ 창이 활성화됩니다.\
![](<../../.gitbook/assets/img_001 (5).png>)​\
2\.   이름, 아이디, 비밀번호, 이메일, 회사명 등 기본 정보와 백업에 이용될 백업 서버 주소, 저장 경로를 입력하고 하단의 **추가** 버튼을 클릭합니다. 항목 좌측에 \*표시가 있으면 필수 항목이므로 반드시 입력해야 합니다.

1. ​서버: 백업에 사용하는 서버 IP (백업된 데이터가 저장되는 서버)
2. 저장 경로: 백업에 사용하는 서버 내의 저장 경로 (서버 내의 스토리지 경로)

## 계정 정보 확인 및 수정

계정 목록에서 이름이나 아이디를 클릭하면 계정에 대한 상세 정보를 확인하고, 필요 시 수정할 수 있습니다.

![](<../../.gitbook/assets/img_002 (7).png>)

### 기본 정보

계정의 기본 정보가 표시되고 우측 상단의 수정 버튼을 클릭하면 아이디를 제외한 기본 정보를 수정할 수 있습니다. 수정 시 저장 경로를 변경하면 백업독에 이미 백업된 파일이 있을 경우, 변경된 경로로 다시 풀 백업이 진행됩니다.\
​\
![](<../../.gitbook/assets/img_003 (3).png>)​

### 용량 상세 목록​​

백업 관리자가 백업을 위해 할당 받는 용량 상세 목록입니다. 용량은 여러 개로 등록할 수 있으며, 가용 기간에 포함된 용량들의 합으로 실제 사용가능한 용량이 반영됩니다. 용량을 추가, 수정 및 삭제가 가능합니다.

![](<../../.gitbook/assets/img_004 (2).png>)

### 원본 서버 목록

백업 관리자가 사용하는 원본 서버 목록입니다. 원본 서버 추가, 수정 및 삭제가 가능합니다.\
​\
![](<../../.gitbook/assets/img_005 (2).png>)\
​

## 계정 삭제

백업 관리자 계정을 강제 탈퇴시킬 수 있습니다. 계정 상세 보기 화면 하단의 **강제탈퇴** 버튼을 클릭하여 처리하며 강제탈퇴 시 계정과 백업 데이터가 완전히 삭제됩니다.\
​\
![](<../../.gitbook/assets/img_006 (2).png>)
