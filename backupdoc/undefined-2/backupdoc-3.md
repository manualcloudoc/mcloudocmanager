# BackupDoc - 슈퍼관리자가 백업 로그 확인하는 방법

​\
슈퍼관리자는 BackupDoc 관리자 웹페이지에서 백업이 실행된 로그 확인할 수 있습니다. 백업 로그는 백업 시작 시와 종료 시에 기록되며, 백업 종료 후에는 작업의 성공/실패 여부와 전송된 파일 수 및 용량을 확인할 수 있습니다.\
​\
1\.   관리자 웹페이지에서 **백업 로그**를 선택하면 전체 기간에 대한 모든 백업 로그가 표시됩니다.\
![](<../../.gitbook/assets/img_000 (6).png>)**\*\*\*\*​**2.   검색 조건을 입력하고 **검색** 버튼을 클릭합니다.

1. **백업 날짜**:  백업이 실행된 날짜를 기간으로 입력합니다.
2. **​백업 성공 여부**:  백업 결과의 성공/실패
3. **​아이디**:  백업을 수행한 백업관리자의 아이디
4. **​서버**:  백업 서버의 주소

> <img src="../../.gitbook/assets/img_001 (9).png" alt="" data-size="original">

3\.   다음과 같은 결과를 확인할 수 있습니다. &#x20;

![](<../../.gitbook/assets/img_002 (6).png>)

1. ​**작업 시간**:  백업이 실행된 날짜와 시간
2. **작업**:  백업 시작/백업 종료의 구분
3. **작업 결과**:  백업의 성공/실패
4. **아이디**:  백업관리자의 아이디
5. **백업 이름**:  백업 스케줄 설정 시 입력한 백업 스케줄 이름
6. **전송 파일 크기**:  전송된 백업 데이터의 용량
7. **전송 파일 수**:  백업된 파일 수
8. **서버**:  백업 서버의 IP
9. **저장 경로**:  백업한 원본 폴더의 경로
