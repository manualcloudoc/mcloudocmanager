# BackupDoc - 문서중앙화 서버의 백업 스케줄 관리하기

BackupDoc을 사용하여 문서중앙화 서버를 백업할 경우, 서비스관리자 또는 정보보호관리자가 관리자 웹페이지에서 백업 스케줄을 설정하고 관리할 수 있습니다. 백업 스케줄은 백업할 원본 경로와 백업 일정으로 구성됩니다.

1.1

1.2

### 스케줄 목록보기

**백업 모듈 관리 – BackupDoc - 백업 스케줄 관리**를 선택하면 설정되어 있는 스케줄 목록이 표시됩니다.

![](<../../.gitbook/assets/img_000 (12).png>)

1. **백업 이름**: 백업 스케줄 이름
2. **백업 경로**: 백업할 원본 경로
3. **백업 유형**: 매일, 매월, 매주, 매월 등 백업 시간 유형
4. **다음 백업 시간**: 다음 백업 작업을 할 시간
5. **이전 백업 시간**: 최근 백업 작업한 시간
6. **스냅샷 개수**: 해당 백업 스케줄의 스냅샷 현재 개수와 최대 개수
7. **사용 여부**: 해당 백업 스케줄에 대한 사용 여부(사용/미사용)
8. **작업**: **즉시 백업** 버튼을 클릭하면, 지정된 백업 일정과 관계없이 해당 스케줄의 백업을 즉시 실행할 수 있습니다.

### 스케줄 추가하기

1\.   목록 상단의 **추가** 버튼을 클릭합니다.

![](<../../.gitbook/assets/img_001 (17).png>)

2\.   ‘스케줄 추가’ 창에서 아래의 정보를 입력한 후, 하단의 **계속** 버튼을 클릭합니다.\
![](<../../.gitbook/assets/img_002 (10).png>)&#x20;

1. 백업 이름: 백업 스케줄을 쉽게 구분하기 위해 지정합니다. 이름을 입력하고 **중복 확인**을 클릭합니다. 삭제된 스케줄 이름도 중복 확인 시 중복으로 처리됩니다.
2. 백업 경로: **폴더 선택** 버튼을 클릭하여 백업할 원본 폴더의 위치를 선택하고 **확인**을 클릭합니다.

> <img src="../../.gitbook/assets/img_003 (7).png" alt="" data-size="original">

1. 사용 여부: 지정할 스케줄을 사용할 것인지 여부(사용/미사용)을 설정합니다.

3\.   백업 스케줄 일정(시작일, 종료일, 백업 간격)과 최대 스냅샷 수를 지정한 후, **확인**을 클릭합니다.![](<../../.gitbook/assets/img_004 (5).png>)

1. 백업 주기별 설정

> 1) 매시간 백업(Hourly): 시작 시간과 시간 간격(1\~12시간)을 지정합니다. 백업 용량이 너무 많아질 수 있어 주의가 필요합니다.
> 2) 매일 백업(Daily): 시작일(년월일), 백업시간(시분)을 지정합니다.
> 3) 매주 백업(Weekly): 요일(월/화/수/목/금/토/일)에 따른 백업시간(시분)을 지정합니다.
> 4) 매월 백업(Monthly): 날짜(1\~마지막날), 백업시간(시분)을 지정합니다.

1. 종료일: 종료일을 지정할 경우 체크하여 사용합니다. 종료일을 지정하지 않은 경우 백업 스케줄에서 지정한 시간에 계속 백업이 실행됩니다.
2. 보관할 최대 스냅샷 수: 백업으로 저장할 최대 스냅샷 수를 정합니다. 최대 스냅샷 수를 초과 하는 경우 오래된 스냅샷부터 삭제됩니다.

①   **매시간 백업**: 위의 그림과 같이 백업 스케줄을 시작할 일자와 시간, 백업할 시간 간격을 설정합니다.&#x20;

②   **매일 백업**: 백업 스케줄을 시작할 일자와 시간을 지정하면 매일 지정한 시간에 증분 백업이 실행됩니다. \
![](<../../.gitbook/assets/img_005 (5).png>)

③   **매주 백업**: 일주일 중 백업이 진행할 요일과 선택한 요일에 백업할 시간을 지정하면 해당 요일의 해당 시간에 증분 백업이 이뤄집니다.

![](<../../.gitbook/assets/img_006 (5).png>)

④   **매월 백업**: 매월 지정된 날짜의 지정된 시간에 증분 백업이 이뤄집니다. 날짜는 복수개로 선택할 수 있고, 짝수날 또는 홀수날을 한꺼번에 선택할 수도 있습니다.

![](<../../.gitbook/assets/img_007 (4).png>)

### 스케줄 수정하기

등록된 백업 스케줄의 백업 일정 및 최대 보관 스냅샷 수를 수정할 수 있습니다. 백업 이름, 원본경로는 수정할 수 없습니다.1.   수정할 백업 스케줄 좌측의 체크박스를 클릭한 후, 우측 상단의 **수정**을 클릭합니다.![](<../../.gitbook/assets/img_008 (2).png>)  \
2\.   ‘수정’ 창에서 변경이 필요한 항목을 설정한 후, **확인**을 클릭합니다.![](<../../.gitbook/assets/img_009 (2).png>)&#x20;

![](<../../.gitbook/assets/img_010 (1).png>)

### 스케줄 삭제하기

백업스케줄 목록에서 삭제할 항목 좌측의 체크박스를 선택한 후 상단의 **삭제**를 클릭하여 삭제합니다.

![](<../../.gitbook/assets/img_011 (1).png>)

### 즉시 백업하기

지정된 백업 일정 이외에도 필요한 경우 즉시 백업을 실행할 수 있습니다. 백업 스케줄 목록에서 **작업** 항목의 **즉시 백업**을 클릭하면 백업이 바로 실행됩니다. 최초 백업인 경우에는 풀 백업, 기존 백업이 있는 경우에는 증분 백업으로 백업이 진행됩니다.![](<../../.gitbook/assets/img_012 (1).png>)
