# 공용자료실 관리하기

공용자료실을 사용하기 위해서는 관리자가 다음과 같은 설정 작업을 수행해야 합니다.

1. **공용자료실 활성화**

> 공용자료실 기능을 활성화합니다.

1. **중앙문서함의 특정 문서함이나 폴더를 공용자료실로 지정**

> 중앙문서함의 특정 문서함이나 폴더를 공용자료실로 설정합니다.

1. **부서별 공용자료실을 위한 폴더 권한 적용 옵션 설정**

> 부서별로 전용 공용자료실을 운영하려면 폴더에 설정된 권한을 공용자료실에도 적용하는 옵션을 활성화합니다.

1. **탐색기를 통한 공용자료실 접근 차단을 위한 폴더 권한 설정**

> 일반 사용자의 공용자료실에 대한 접근을 전용 메뉴로만 제한하고, 윈도우 탐색기를 통한 접근을 차단하기 위해 폴더 권한을 설정합니다.

다음은 이러한 설정 작업을 수행하는 과정입니다.

1\.   웹 화면의 좌측 메뉴에서 **시스템 설정** – **설정 홈 연결** 메뉴를 클릭합니다.  2.   설정 홈 화면으로 바뀌면 **일반 설정** – **윈도우 에이전트** 메뉴를 클릭합니다. \
3\.   윈도우 에이전트와 관련된 설정 작업을 할 수 있는 화면이 나타납니다. **공용자료실**로 이동한 후 공용자료실과 관련된 다음 항목들을 설정합니다.

![](<../.gitbook/assets/img_000 (19).png>)

1. **공용자료실 사용 여부**

> 공용자료실의 사용 여부를 설정합니다. 기본 설정은 **사용 안 함**입니다.

1. **공용자료실 경로 선택**

> 현재 공용자료실로 사용 중인 문서함과 폴더가 표시됩니다. 다른 문서함이나 폴더로 변경하려면 **변경**을 클릭합니다.\*\*‘공용자료실 경로 선택’\*\*창이 나타나면 드롭다운 박스를 클릭하여 문서함을 선택합니다. 선택한 문서함의 특정 폴더를 공용자료실로 사용하려면 아래에서 해당 폴더를 선택한 후 **변경**을 클릭합니다.
>
> <img src="../.gitbook/assets/img_001 (19).png" alt="" data-size="original">
>
> ![Warning](<../.gitbook/assets/img_002 (16).png>)공용자료실로 **문서함**을 지정하면 모든 사용자가 문서함의 하위 폴더에 저장된 파일까지 읽거나 다운로드할 수 있어 주요 자료가 의도치 않게 노출될 위험이 있습니다. 그러므로, 가능하면 문서함 대신 폴더를 공용자료실로 사용하기를 권장합니다.

1. **공용자료실 경로에 대한 폴더 권한 적용 여부**

> 공용자료실 사용 시 기존 폴더 권한을 적용하여, 사용자별로 폴더 사용을 제한하거나(사용함), 폴더 권한과 관계없이 모든 사용자의 접근을 허용하는 것(사용 안 함) 중 선택하는 옵션입니다. 기본 설정은 **사용 안 함**입니다.
>
> 1. 이 옵션을 **사용함**으로 설정하면 설정된 폴더 권한이 공용자료실에 적용되어, 권한이 있는 사용자만 해당 폴더를 공용자료실에서 볼 수 있습니다. 따라서, 폴더마다 다른 부서가 사용하도록 폴더 권한이 설정된 상황에서 이 옵션을 사용하면 각 부서의 공용자료실에 서로 다른 폴더가 표시됩니다.

> 3. 이 옵션을 **사용 안** 함으로 설정하면 사용자의 권한과 관계없이 공용자료실의 모든 폴더 목록이 표시됩니다.

> 예를 들어, **마케팅팀**은 **A 폴더**의 권한을, **영업팀**은 **B 폴더**의 권한을 보유한 상황에서 이 옵션을:
>
> 1. **사용함**으로 설정하면 마케팅팀의 공용자료실에는 A 폴더가, 영업팀의 공용자료실에는 B 폴더가 각각 표시됩니다.
> 2. **사용 안 함**으로 설정하면 마케팅팀과 영업팀의 공용자료실에 A 폴더와 B 폴더가 모두 표시됩니다.
>
> ![Notes](<../.gitbook/assets/img_003 (14).png>)공용자료실에 저장된 자료를 조회하거나 다운로드하려면 **목록보기**와 **읽기** 권한이 모두 필요합니다. 폴더 권한을 설정하는 방법은 \*\*사용자 매뉴얼 – [폴더 권한 설정하기](https://github.com/manualcloudoc/mcloudoc-user-manual/blob/main/zoho-export/markdown/%EC%82%AC%EC%9A%A9%EC%9E%90-%EB%A7%A4%EB%89%B4%EC%96%BC/basic/%ED%8F%B4%EB%8D%94-%EA%B6%8C%ED%95%9C-%EC%84%A4%EC%A0%95%ED%95%98%EA%B8%B0.md)\*\*를 참고합니다.

4\.   공용자료실로 사용할 문서함이나 폴더가 에이전트 트레이 메뉴의 **공용자료실**을 통해서만 접근할 수 있고 윈도우 탐색기에서 보이지 않도록 해당 문서함이나 폴더의 **폴더 권한**을 설정해야 합니다. 이를 위해 우측 상단에 있는 **메인 홈**을 클릭하여 메인 홈으로 이동합니다.

&#x20;![](<../.gitbook/assets/img_004 (10).png>)  \
5\.   메인 홈의 우측 상단에서 **서비스관리자**를 클릭한 후 **Agent 실행** 메뉴를 실행합니다.

![](<../.gitbook/assets/img_005 (9).png>)\
6\.   윈도우 에이전트가 실행되면 윈도우 탐색기에서 공용자료실로 지정한 문서함이나 폴더를 우클릭한 후 컨텍스트 메뉴에서 \*\*폴더 권한 - 권한 설정..\*\*을 클릭합니다.

![](<../.gitbook/assets/img_006 (9).png>)

7\.   폴더의 속성을 설정할 수 있는 창이 나타납니다. 상단에 있는 **이 폴더에 권한을 추가하여 설정** 항목을 클릭하여 활성화합니다. 아래에 있는 항목들이 설정 가능한 상태로 바뀌면 **추가**를 클릭합니다.

![](<../.gitbook/assets/img_007 (8).png>)

8\.   ‘**부서(그룹) 및 사용자 선택**’ 창이 나타납니다. 모든 직원에게 적용될 수 있도록 좌측의 목록의 최상위에 있는 부서를 선택하고 \*\*추가>>\*\*를 클릭합니다. 부서가 우측 목록에 추가되면 **확인**을 클릭합니다.

![](<../.gitbook/assets/img_008 (7).png>)

9\.   다시 폴더 속성 창으로 돌아오면 **사용 권한** 옵션에 있는 **모든 권한** 항목을 **거부**로 선택하고 **확인**을 클릭합니다. &#x20;

![](<../.gitbook/assets/img_009 (6).png>)

10\.   선택한 부서의 하위 부서에도 권한 설정을 적용할지 묻는 팝업 창이 나타나면 **예**를클릭합니다.

11\.   공용자료실 설정이 올바르게 되었는지 확인하기 위해 윈도우 에이전트의 트레이 메뉴에서 \*\*공용자료실…\*\*을 클릭합니다.

![](<../.gitbook/assets/img_010 (5).png>)

12\.   ‘**공용 자료실**’ 창이 나타납니다. 좌측에 공용자료실로 지정한 문서함이나 폴더가 최상위에 표시되고(문서함이나 폴더의 이름은 표시되지 않음) 우측에 하위 폴더와 폴더에 저장된 파일이 표시되는지 확인합니다. &#x20;

![](<../.gitbook/assets/img_011 (4).png>)

![Notes](<../.gitbook/assets/img_012 (4).png>)‘공용 자료실’ 창의 최상위 폴더는 공용자료실로 설정한 문서함이나 폴더의 실제 이름 대신 **공용 자료실**로 표시됩니다.
