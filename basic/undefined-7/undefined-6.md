# 조직도에서 부서문서함 생성 및 관리하기

{% hint style="info" %}
메뉴 경로: **일반 관리 - 인사 조직 관리 - 조직도 - 조직도 관리 - 조직도 편집기 실행** 버튼 클릭
{% endhint %}

&#x20;관리자는 조직도 편집기를 통해 부서문서함을 생성하거나 관리, 삭제할 수 있습니다.

#### <mark style="color:$primary;">부서문서함 생성하기</mark>

부서문서함은 부서원이 협업할 수 있도록 생성된 문서 저장 공간입니다.  부서문서함은 전사 또는 부서 단위로 생성 가능하며, 드라이브 형태로 구분되어 사용자에게 표시됩니다.&#x20;

1. 부서를 선택합니다. 이때 **전사 아이콘**을 선택하면 전사문서함을 생성할 수 있습니다.
2. 주 메뉴의 **도구 - 부서문서함 생성**을 선택합니다.
3. '**부서문서함 생성**’ 창의 **작업 목록**에서 부서를 선택한 후 **할당 용량, 서버위치/중요도, 부서문서함 이름, 연결할 드라이브** 등 생성할 문서함 정보를 입력합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
부서문서함의 **할당 용량** 및 **서버위치/중요도**에 대한 설명은 [**신규 도메인 등록하기**](../undefined-4/undefined.md)를 참고하세요.
{% endhint %}

4. **생성**을 클릭합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
필요에 따라 같은 부서에 여러 부서문서함을 생성할 수 있습니다.
{% endhint %}

![](<../../.gitbook/assets/img_003 (51).png>)

#### <mark style="color:$primary;">부서문서함 관리하기</mark>

생성된 부서문서함의 할당 용량, 이름, 연결할 드라이브 정보를 변경할 수 있습니다.

1. 부서문서함의 정보 변경이 필요한 부서를 선택합니다.
2. 주 메뉴의 **도구 - 부서문서함 관리**를 선택합니다.
3. ‘**부서문서함 관리**’ 창의 **작업 목록**에서 작업할 부서문서함을 선택합니다.
4. **할당 용량,  부서문서함 이름, 연결할 드라이브**를 변경 입력합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
부서문서함의 **서버 위치, 중요도** 정보는 변경할 수 없습니다.
{% endhint %}

5. **변경**을 클릭하여 변경 사항을 적용합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
여러 부서를 한꺼번에 선택하여 각 부서의 문서함을 한꺼번에 관리할 수 있습니다.
{% endhint %}

다음 동영상에서 부서문서함 용량을 변경하고, 변경된 용량을 확인하는 방법과 용량이 부족할 경우의 처리 방법을 참고합니다.

#### <mark style="color:$primary;">부서문서함 삭제하기</mark>

더 이상 사용하지 않는 부서문서함을 삭제합니다. 삭제된 부서문서함 드라이브는 더 이상 사용자에게 표시되지 않습니다.

1. 부서문서함을 삭제할 부서를 선택합니다.
2. 주 메뉴의 **도구 - 부서문서함 관리**를 선택합니다.
3. ‘**부서문서함 관리**’ 창의 **작업 목록**에서 삭제할 부서문서함을 선택합니다.
4. **삭제**를 클릭합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
삭제된 부서문서함 데이터는 서버에 그대로 존재합니다.
{% endhint %}
