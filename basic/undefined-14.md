# 파일 버전 관리하기

버전관리는 이전 버전의 파일을 조회하여 복원하는 기능입니다. 랜섬웨어 감염으로 파일이 암호화되거나 작업 중 예상치 못하게 데이터가 손실된 경우 언제든지 복구할 수 있습니다. 이전 버전의 파일은 사용자가 파일을 저장할 때마다 자동으로 생성되며 손실되지 않도록 다른 위치에 저장됩니다.

![Notes](<../.gitbook/assets/img_000 (28).png>)버전관리는 Microsoft 365 파일, CAD 파일, PDF, Photoshop 이미지 파일, 사진, 비디오 등을 포함한 모든 파일 형식을 지원합니다. 스토리지 용량을 고려하여 버전 관리가 가능한 최대 파일 크기와 버전 관리에서 제외할 프로세스 등을 관리자가 설정할 수 있습니다.&#x20;

### 이전 버전으로 파일 복원하기

이전 버전으로 복원할 수 있는 파일 목록을 조회 후 파일을 복원할 수 있습니다. 다만, 파일이 수정된 시간을 기준으로 10분 이내에 생성된 이전 버전의 파일은 조회가 안될 수도 있습니다.복원된 파일은 원본 파일과 동일한 위치에 복원됩니다.&#x20;

1\.    **일반 모듈 관리 - Basic - 버전 관리 - 조회 & 복원**을 클릭합니다.

![](<../.gitbook/assets/img_001 (27).png>)

2\.    검색 조건을 입력합니다.

1. **기간 선택(필수)**: 복원할 파일이 백업된 날짜와 시간을 선택합니다.
2. **문서함(필수)**: 복원할 파일이 저장된 문서함을 선택합니다.
3. **프로세스**: 프로세스명을 입력합니다. 특정 프로세스만 검색할 때 입력합니다.
4. **검색어:**

&#x20;         \-    **파일 경\*\*\*\*로**: 복원할 파일의 경로를 검색할 때 입력합니다.

> \-    **파일명**:  복원할 파일의 이름을 검색할 때 입력합니다.

3\.    **검색**을 클릭합니다.

4\.    복원하고 싶은 파일의 **복원** 버튼을 클릭합니다. 목록에 같은 파일명이 여러 개인 경우 **생성일**을 참고하여 복원할 파일을 선택합니다. 만약 이전 버전의 파일이 복원될 위치에 같은 이름의 파일이 이미 존재하면 복원되지 않습니다. 이 경우 해당 파일을 찾아서 파일명을 변경한 후 다시 **복원** 버튼을 클릭합니다.\
![Notes](<../.gitbook/assets/img_002 (23).png>)복원된 파일명은 ‘**원본파일명\_복원된 파일의 생성연도월일시분초.확장자**’입니다. \
&#x20;                                                                                                                                                                                                                     ![](<../.gitbook/assets/img_003 (20).png>)

### 이전 버전으로 파일 일괄 복원하기

랜섬웨어 감염으로 특정 문서함 내 파일 전체가 암호화된 경우 간편하게 모든 파일을 일괄 복원할 수 있습니다. 일괄 복원은 백그라운드로 진행되며, 복원 파일의 개수 및 크기에 따라 긴 시간이 소요될 수 있습니다. 동일한 파일명으로 여러 개의 파일들이 존재할 경우에는 가장 최근 파일이 복원됩니다.

1\.     **일반 모듈 관리 - Basic - 버전 관리 - 조회 & 복원**을 클릭합니다.

2\.     검색 조건을 입력합니다.

3\.     **검색**을 클릭합니다.

4\.     목록 우측 상단의 **일괄 복원** 버튼을 클릭합니다.

![Notes](<../.gitbook/assets/img_004 (14).png>)이미 복원된 적이 있어서 동일한 파일명이 이미 존재할 경우에는 기존 파일명에 숫자가 추가되어 복원됩니다. (예를 들어, ‘영업매출보고서\_20221015103020.xlsx’ 파일이 일괄 복원으로 다시 복원되는 경우 파일명은 ‘영업매출보고서\_20221015103020\_1.xlsx’입니다.)
