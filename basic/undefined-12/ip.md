# IP대역에 따라 사용자별로 로그인 제어하기

시스템 설정에서 **IP 인증**을 사용하도록 설정한 경우, 사용자의 IP에 따라 로그인을 제한할 수 있습니다.관리자는 로그인을 허용할 IP대역을 사전에 등록해야 하며, 등록되지 않은 IP대역에서의 로그인은 차단됩니다. IP인증의 종류는 다음과 같습니다.

* **안전인증 IP**: 전체 사용자에 대해 로그인을 허용하는 IP대역입니다.
* **기본인증 IP**: 특정 사용자에 대해 로그인을 허용하는 IP대역으로, 사용자별로 설정됩니다. 사용자가 웹페이지에 처음 로그인할 때 자동으로 등록되거나,관리자가 직접 사용자의 기본인증 IP를 등록할 수 있습니다. 기본 인증 IP로는 사용자 당 1개의 IP대역만 등록할 수 있습니다.
* **추가인증 IP**: **기본인증 IP** 외에 로그인을 추가로 허용하는 IP대역으로, 사용자별로 설정됩니다. 사용자가 웹에서 신청한 후 관리자가 승인하거나, 관리자가 직접 사용자의 추가인증 IP를등록할 수 있습니다.

### <mark style="color:$primary;">IP 인증 설정하기</mark>

**IP 인증**을 생성하여 전체 사용자에 대해 로그인을 허용할 IP대역(**안전인증 IP**)을 설정하거나, 사용자별로 로그인을 허용할 IP대역(**기본인증 IP** 또는 **추가인증 IP**)을 설정할 수 있습니다.

1. **일반 모듈 관리 - Basic - 로그인 공통 제어 - IP 인증 - 인증 관리**를 클릭합니다.

![](<../../.gitbook/assets/img_000 (18).png>)

2. IP 인증 목록 우측 상단의 **추가** 버튼을 클릭합니다.
3. **'IP 인증 생성'**&#xCC3D;에서 다음과 같은 항목을 설정합니다.

<div align="left"><img src="../../.gitbook/assets/img_001 (11).png" alt="" width="293"></div>

* **IP 인증 종류**: 생성할 IP 인증의 종류를 선택합니다.

> - **안전인증 IP** : 전체 사용자에 대해 로그인을 허용할 IP대역을 설정합니다.
> - **기본인증 IP:** 사용자별로 로그인을 허용할 기본 IP대역을 설정합니다. 사용자별로 1개의 기본인증 IP대역을 설정할 수 있습니다.
> - **추가인증 IP**: 사용자별로 로그인을 허용할 추가 IP대역을 설정합니다. 안전인증 IP와 기본인증 IP 외에 추가적으로 다른 IP대역에서도 접속이 필요한 경우에 사용하며, 필요에 따라 복수의 추가인증 IP대역을 설정할 수 있습니다.

* **사용자**(필수): **기본인증 IP** 또는 **추가인증 IP**를 생성할 사용자를 선택합니다. 검색창에서 이름 또는 ID로 검색한 후 목록에서 선택할 수 있습니다. **안전인증 IP** 설정 시에는 **사용자** 항목이 표시되지 않습니다.
* **시작 IP**, **끝 IP**(필수): 로그인을 허용할 IP대역을 입력합니다.
* **시작일**(필수): 설정한 IP인증 적용을 시작할 날짜를 달력에서 선택합니다.
* **만료일**: 설정한 IP인증의 적용 기한을 선택합니다.

4. **확인**을 클릭합니다.

### <mark style="color:$primary;">설정된 IP 인증 검색 및 변경하기</mark>

등록되어 있는 IP 인증을 검색하고 설정을 변경할 수 있습니다.

#### IP 인증 검색하기

1. **일반 모듈 관리 - Basic - 로그인 공통 제어 - IP 인증 - 인증 관리**를 클릭합니다.
2. **IP 인증 종류**를 선택합니다.
3. 선택한 IP 인증 종류에 따라 검색 조건으로 **사용자 이름** 또는 **ID**, **시작 IP**, **끝 IP**를 사용할 수 있습니다.

* **사용자 이름/ID**로는 사용자별로 설정되는 **기본인증 IP**와 **추가인증 IP**만 검색할 수 있습니다.

4. **검색**을 클릭하여 목록을 확인합니다.

#### IP 인증 변경하기

등록되어 있는 IP 인증에서 IP대역의 범위, 시작일, 만료일을 변경할 수 있습니다.

1. IP 인증 목록에서 변경할 IP 인증 우측의 **연필** 아이콘을 클릭합니다.
2. **‘IP 인증 수정’** 창에서 설정 항목을 변경합니다.

* **IP 인증 종류**와 **사용자**(안전인증IP의 경우 해당 없음)는 변경할 수 없습니다. 인증 종류나 사용자를 변경하여 설정하고자 할 경우, 해당 IP 인증을 삭제하고 새로 생성해야 합니다.

3. **확인**을 클릭합니다.

#### IP 인증 삭제하기

IP 인증 목록 우측의 **휴지통** 아이콘을 클릭하여 해당 IP 인증을 삭제할 수 있습니다.

### <mark style="color:$primary;">추가인증 IP 요청을 승인/반려하는 방법</mark>

**안전인증 IP**와 **기본인증 IP** 외의 다른 IP대역에서 로그인이 필요한 경우, 사용자는 웹페이지에서 **추가인증 IP**대역 등록을 요청할 수 있습니다. 관리자는 다음과 같은 방법으로 사용자의 IP 인증 등록 요청을 승인 또는 반려할 수 있습니다.

1. **일반 모듈 관리 – Basic – 로그인 공통 제어 – IP 인증 – 요청 처리**를 클릭합니다.

![](<../../.gitbook/assets/img_002 (15).png>)

2. 목록에서 처리할 요청을 체크하여 선택한 후 **승인** 또는 **반려** 버튼을 클릭합니다.
3. **‘승인’/**‘**반려’** 창에서 확인을 클릭합니다. 반려 시에는 사용자가 요청이 반려된 이유를 확인할 있도록 **반려 사유**를 입력합니다.

#### 추가인증 IP 요청의 처리 이력 보기

사용자가 **추가인증 IP**를 신청/취소한 이력과 관리자가 승인/반려한 이력을 조회할 수 있습니다.

1. **일반 모듈 관리 – Basic – 로그인 공통 제어 – IP 인증 – 요청 처리 이력**을 클릭합니다.

![](<../../.gitbook/assets/img_003 (12).png>)

2. 다음과 같은 검색 정보를 입력한 후 **검색**을 클릭합니다.

* **작업 기간**(필수): **작업일시**(요청/취소 건에 대해서는 사용자가 요청/취소한 일시, 승인/반려 건에 대해서는 관리자가 요청을 처리한 일시)를 기준으로 검색할 수 있습니다.
* **작업 종류**(필수): **요청, 승인, 반려, 취소**한 작업 중에서 선택할 수 있습니다.
* **검색 조건**: 필요 시 요청한 **사용자의 이름** 또는 **ID**로 검색할 수 있습니다.

3. 사용자의 **추가인증 IP**요청에 관한 이력을 조회합니다. **작업일시**, **요청일시**, 요청한 **사용자 이름/ID**, 요청한 IP대역의 **시작/끝 IP**, **시작일/종료일**과 **작업 종류**를 확인할 수 있습니다.
