# IP대역에 따라 에이전트별로 사용자 로그인 제어하기

웹, 모바일기기, Mac, 윈도우 등 에이전트 종류별로 문서중앙화 서비스에 로그인을 허용/차단할 IP대역을 설정하여 사용자의 로그인을 제어할 수 있습니다.

**IP** **인증**(안전인증 IP, 추가인증 IP, 기본인증 IP)과 **IP대역에 따른 에이전트별 인증**을 모두 사용하도록 설정한 경우, 모든 조건을 만족시켜야만 로그인이 성공합니다.

### <mark style="color:$primary;">에이전트별 허용/차단 IP대역 설정하기</mark>

에이전트의 종류별로 로그인을 허용할 IP대역과 차단할 IP대역을 등록할 수 있습니다.  등록된 IP대역에는 우선 순위가 부여되며, 목록의 상단에 위치할수록(우선 순위 번호가 작을수록) 우선적으로 적용됩니다. 등록되지 않은 IP대역/에이전트에서 접속 시에는 기본적으로 로그인을 제한하지 않습니다.

1. **일반 모듈 관리 – Basic - 로그인 공통 제어 – 에이전트별 IP대역 설정**을 클릭합니다.

![](<../../.gitbook/assets/img_000 (17).png>)

2. 페이지 우측 상단의 **추가** 버튼을 클릭합니다.
3. **‘IP 정보 추가’** 창에서 에이전트별로 로그인을 허용/차단할 IP대역 정보를 입력합니다.

* **시작 IP**, **끝 IP**(필수 입력)
* **접속 설정**: 허용, 차단 중에서 선택합니다. 입력한 IP대역에서의 로그인을 허용/차단할 수 있습니다.
* **에이전트**: 윈도우 에이전트, Mac 에이전트, 모바일 기기, 웹 중에서 적용할 에이전트를 선택합니다.

<div align="left"><figure><img src="../../.gitbook/assets/img_001 (12).png" alt="" width="392"><figcaption></figcaption></figure></div>

4. **확인**을 클릭하면, **등록된 IP 목록**에 IP 대역 정보가 추가됩니다.
5. 다음 목차의 설명을 참고하여, 우선 순위 조정이 필요한 경우 우선 순위를 변경합니다.
6. 우측 하단에 있는 **적용** 버튼을 클릭합니다.

{% hint style="danger" %}
**에이전트별 IP대역 설정** 시 추가/삭제/변경 후 **적용** 버튼을 클릭해야만 서버에 변경 사항이 적용됩니다.
{% endhint %}

### <mark style="color:$primary;">등록된 IP대역의 우선 순위 및 설정 변경하기</mark>

특정 에이전트 및 IP에서 로그인 시도 시 적용될 수 있는 허용/차단 IP대역이 두 개 이상일 경우, 우선 순위에 따라 허용/차단이 결정됩니다. 아래의 예시에서 사용자 IP가 동일한 에이전트에 대해 설정된 두 개의 IP대역에 모두 속하는 경우, 우선 순위가 높은 설정(아래 예시에서는 **허용**)이 적용되어 로그인이 허용됩니다.

![](<../../.gitbook/assets/img_003 (5).png>)

#### 등록된 IP대역의 우선 순위 변경하기

1. **일반 모듈 관리 – Basic - 로그인 공통 제어 – 에이전트별 IP대역 설정**을클릭합니다.
2. 우선 순위 변경이 필요한 IP대역의 **우선 순위** 번호를 마우스로 클릭한 후 위/아래로 드래그하여 원하는 위치로 이동합니다.  우선 순위가 높을수록 목록의 상단에 위치해야 합니다.
3. **적용** 버튼을 클릭합니다.

#### 등록된 IP대역의 설정 변경하기

1. **등록된 IP 목록**에서 설정을 변경할 IP대역 우측의 **연필** 아이콘을 클릭합니다.
2. **‘IP 정보 수정’** 창에서 변경할 정보를 입력하고 **확인**을 클릭합니다.
3. **적용** 버튼을 클릭합니다.       &#x20;

#### 등록된 IP대역 삭제하기

1. **등록된 IP 목록**에서 삭제할 IP대역 우측의 **휴지통**을 클릭합니다.
2. **적용** 버튼을 클릭합니다.
