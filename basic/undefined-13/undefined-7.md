# 사용자 개인문서함의 데이터 인계 이력 보기

문서중앙화 솔루션은 사용자의 개인 문서함에 있는 모든 문서들을 다른 사용자의 개인 문서함으로 그대로 옮겨주는 ‘데이터 인계’ 기능을 제공합니다. 데이터 인계 기능을 사용하면 승진이나 부서 이동, 퇴직 등으로 인해 후임자에게 업무를 인계해야 하는 경우에 쉽고 간편하게 데이터를 전달할 수 있습니다.&#x20;

데이터 인계 기능은 관리자와 사용자가 모두 사용할 수 있습니다. 자신의 업무를 인계하고자 하는 ‘사용자’는 인수자를 지정한 후 승인권자의 승인을 받는 과정을 거쳐서 데이터 인계 기능을 사용할 수 있고, ‘관리자’는 인계자와 인수자를 직접 지정하여 바로 데이터 인계 기능을 실행할 수 있습니다. \
![Notes](<../../.gitbook/assets/img_000 (44).png>)사용자와 관리자가 데이터 인계 기능을 사용하는 방법은 각각 **사용자 매뉴얼** \*\*- [데이터 인계 기능 소개](https://github.com/manualcloudoc/mcloudoc-user-manual/blob/main/zoho-export/markdown/%EC%82%AC%EC%9A%A9%EC%9E%90-%EB%A7%A4%EB%89%B4%EC%96%BC/basic/%EB%8D%B0%EC%9D%B4%ED%84%B0-%EC%9D%B8%EA%B3%84-%EA%B8%B0%EB%8A%A5-%EC%86%8C%EA%B0%9C.md)\*\*와 본 매뉴얼  \*\*[사용자 개인문서함의 데이터 인계하기](../undefined-5/undefined-3.md)\*\*를 참고하세요.&#x20;

관리자는 사용자와 관리자를 통해 데이터 인계 기능이 사용된 이력을 조회할 수 있습니다. 이력을 조회할 때에는 기간, 상태, 사용자나 승인권자 등과 같은 세부 검색 조건을 설정하여 원하는 정보만 선별하여 출력되도록 할 수 있습니다. 설정할 수 있는 세부 검색 조건들은 다음과 같습니다.

|           |                                       |
| --------- | ------------------------------------- |
| **조건**    | **설명**                                |
| **기간**    | 특정 기간 동안의 데이터 인계 조회                   |
| **작업 상태** | 특정 상태(신청, 반려, 취소, 인계 완료 등)의 데이터 인계 조회 |
| **사용자**   | 특정 사용자나 관리자와 관련된 데이터 인계 조회            |

다음은 이러한 검색 조건들을 사용하여 특정한 데이터 인계 이력을 출력할 수 있는 예들입니다.

•    이번 주에 신청된 데이터 인계 목록

•    지난 2년간 처리된 모든 데이터 인계 목록

•    김길동 사용자가 신청 취소한 데이터 인계 목록

•    홍길동 승인권자가 지난 한달 동안 승인하거나 반려한 데이터 인계 목록

이렇게 조회한 데이터 인계 이력은 엑셀 파일로 저장하여 다양한 용도로 활용할 수 있습니다. \
![Notes](<../../.gitbook/assets/img_001 (43).png>)데이터 인계 이력 조회는 웹 메뉴에서만 제공되고 윈도우 에이전트에서는 제공되지 않습니다.  \
관리자가 웹 메뉴를 사용하여 데이터 인계 이력을 조회하는 방법은 다음과 같습니다.

1\.     웹 페이지에 로그인한 후 화면 왼쪽 메뉴에서 **일반 모듈 관리** – **Basic** - **문서함 관리** – **개인문서함 데이터 인계** – **이력 보기** 메뉴를 클릭합니다.

![](<../../.gitbook/assets/img_002 (38).png>)\
2\.     다음과 같은 **이력 보기** 화면이 나타나면 그림 아래 설명을 참고하여 조회할 조건을 설정합니다.\
​\
![](<../../.gitbook/assets/img_003 (33).png>)

|           |                                                                                                                                                                                                                                                                                                  |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **항목**    | **설정 방법**                                                                                                                                                                                                                                                                                        |
| **신청 기간** | 조회할 기간에 해당되는 버튼(당일, 1주일, 1개월, 3개월, 6개월, 직접입력)을 클릭합니다. ‘직접입력’ 버튼을 클릭한 후에는 달력을 클릭한 후 시작 날짜와 마지막 날짜를 직접 선택합니다.                                                                                                                                                                                      |
| **작업 상태** | 조회하고자 하는 데이터 인계의 상태를 클릭하여 체크합니다. 여러 항목을 동시에 체크할 수 있습니다. •     전체: 모든 상태 •     신청: 승인권자가 승인 혹은 반려하기 이전 상태 •     인계 완료: 승인권자가 승인하여 인계 작업이 완료된 상태 •     반려: 승인권자에 의해 반려된 상태 •     취소: 사용자가 데이터 인계를 신청한 후 취소한 상태 •     인계중: 승인권자가 승인했지만 아직 인계 작업이 완료되지 않은 상태                                         |
| **검색 조건** | 목록에서 조회할 데이터 인계와 관련된 사용자의 종류를 선택한 후 입력란에 사용자나 관리자의 ID 혹은 이름을 입력합니다. •     신청자 ID /신청자 이름: 데이터 인계를 신청한 사용자나 관리자 •     소유자 ID/소유자 이름: 자신의 데이터를 인계한 사용자 •     인수자 ID/인수자 이름: 데이터를 인수한 사용자 •     승인권자 ID/승인권자 이름: 데이터 인계를 승인하거나 반려한 승인권자 ID나 이름은 일부만 입력할 수도 있습니다. 이 항목을 설정하지 않으면 모든 사용자를 대상으로 검색됩니다. |

3\.     다음과 같이 항목을 설정한 후 **검색** 버튼을 클릭합니다.

![](<../../.gitbook/assets/img_004 (26).png>)

4\.    설정된 조건(지난 1개월간 처리된 모든 데이터 인계 이력)에 해당되는 데이터 인계 이력이 검색되어 화면 아래에 출력됩니다.

![](<../../.gitbook/assets/img_005 (22).png>)

출력된 데이터 인계 이력의 각 항목들은 다음과 같은 정보를 나타냅니다.

|                    |                                                                                       |
| ------------------ | ------------------------------------------------------------------------------------- |
| **항목**             | **정보**                                                                                |
| **신청 일시**          | 사용자나 관리자가 데이터 인계를 신청한 날짜와 시각                                                          |
| **신청 사유**          | 사용자가 데이터 인계를 신청할 때 입력한 신청 사유                                                          |
| **신청자/작업자 이름(ID)** | 데이터 인계를 신청한 사용자의 이름(ID). 관리자가 데이터 인계를 실행한 경우에는 ‘정보보호관리자(ID)’ 혹은 ‘서비스관리자(ID)’ 로 표시됩니다. |
| **소유자 이름(ID)**     | 자신의 데이터를 인계한 사용자의 이름(ID)                                                              |
| **인수자 이름(ID)**     | 데이터를 인수한 사용자의 이름(ID)                                                                  |
| **승인권자 이름(ID)**    | 데이터 인계 신청을 승인하거나 반려한 승인권자의 이름(ID)                                                     |
| **작업일시**           | 데이터 인계 신청이 현재 상태로 된 날짜와 시각                                                            |
| **작업 상태**          | 데이터 인계 신청의 현재 상태                                                                      |

기본적으로 **이력 보기** 화면에는 데이터 인계 상태가 변경된 시간의 내림차순으로 배열되어 가장 최근의 데이터 인계가 가장 위에 표시됩니다. **신청일시▲** 항목을 클릭하면 이와 반대의 순서로 배열되도록 할 수 있습니다.\
목록의 오른쪽 위에 있는 **엑셀** 아이콘 ![](<../../.gitbook/assets/img_006 (19).png>)을 클릭하면 출력된 내용을 엑셀 파일로 저장할 수 있습니다.![Notes](<../../.gitbook/assets/img_007 (17).png>)화면에 출력된 내용을 엑셀 파일로 저장하는 방법은 \*\*[조회한 자료를 엑셀 파일로 저장하는 방법](../undefined-2/undefined-2.md)\*\*을 참고하세요.

5\.    신청 사유가 길어서 목록에 다 출력되지 않는 경우에는 데이터 인계의 **신청 사유**를 클릭합니다. 그러면 해당 데이터 인계에 대한 정보만 보여주는 **상세보기** 화면이 나타납니다. 오른쪽 아래에 있는 **목록** 버튼을 눌러 이전 **이력 보기** 화면으로 돌아갈 수 있습니다.

![](<../../.gitbook/assets/img_008 (15).png>)
