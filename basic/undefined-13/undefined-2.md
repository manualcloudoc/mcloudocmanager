# 폴더 권한 현황∙복사 사용하기

관리자는 중앙문서함의 모든 부서문서함과 하위 폴더에 대한 폴더 권한 현황을 확인하고, 특정 폴더에 설정된 폴더 권한을 원하는 폴더에 복사할 수 있습니다.

​

### 폴더 권한 현황 확인하기

관리자가 부서문서함과 하위 폴더에 설정된 폴더 권한 현황을 확인하는 방법은 아래와 같습니다. \
1\.    관리자 웹페이지에서 **일반 모듈 관리 – Basic – 문서함 관리 – 폴더 권한 관리 – 권한 현황∙복사**를 선택합니다.

![](<../../.gitbook/assets/img_000 (98).png>)

2\.    **권한현황∙복사** 페이지의 좌측 문서함/폴더 목록에서 부서문서함과 각 문서함별 폴더를 선택하면 페이지 우측에 선택한 폴더의 정보와 권한 현황이 표시됩니다.

문서함/폴더 목록에서  ![](<../../.gitbook/assets/img_001 (91).png>)은 권한이 설정된 폴더,  ![](<../../.gitbook/assets/img_002 (76).png>)은 권한이 설정되지 않은 폴더를 표시합니다.

1. **선택한 폴더 정보**: 선택한 폴더, 부모 폴더(상위 폴더), 부모 폴더의 권한 계승 여부를 확인할 수 있습니다.
2. **권한 현황** : 권한 구분(상속 권한/설정 권한)과 대상 종류(사용자/부서/직위)에 따라 선택하여 조회할 수 있습니다.
3. 페이지 우측 하단에 권한이 설정된 대상별로 **권한 구분**(설정 권한/상속 권한), **대상 종류**(사용자, 부서, 직위), **대상 이름**, **각각의 권한에 대한 허용 여부**(허용: o, 거부: x), **폴더관리자**(폴더관리자 여부), **권한 만료일**(폴더 권한 자동 해제시) ,**부모 폴더명**(상속 권한일 경우)이 표시됩니다.

> ![Notes](<../../.gitbook/assets/img_003 (62).png>)폴더 권한에 대한 자세한 내용은 **사용자 매뉴얼** [**폴더 권한 소개**](https://support.mcloudoc.com/portal/ko/kb/articles/%ED%8F%B4%EB%8D%94-%EA%B6%8C%ED%95%9C-%EC%86%8C%EA%B0%9C)를 참조하세요.

### 폴더 권한 현황 검색하기

각 문서함에 설정된 권한 현황을 사용자 ID별, 사용자 이름별, 부서명별로 검색할 수 있습니다.

1\.    **권한현황∙복사** 페이지 우측 상단의 **검색** 버튼을 클릭합니다.\
![](<../../.gitbook/assets/img_004 (47).png>)

2\.   ‘**검색**’ 창에서 검색하고자 하는 **문서함**을 선택하고, **검색조건**(사용자ID, 사용자 이름, 부서명), \*\*권한 구분(\*\*상속 권한/설정 권한)을 설정한 후 **검색** 버튼을 클릭합니다.&#x20;

3\.   해당 부서문서함에 설정된 검색조건에 해당하는 모든 폴더 권한 현황을 확인할 수 있습니다. 검색 창 하단에 **권한 구분**(설정 권한/상속 권한), **경로**(폴더 경로), **각각의 권한에 대한 허용 여부**(허용: o, 거부: x), **폴더관리자**(폴더관리자 여부), **권한 만료일**(폴더 권한 자동 해제시) ,**권한 대상자**(권한을 상속한 대상)가 표시됩니다.

![](<../../.gitbook/assets/img_005 (40).png>)

상속 권한인 경우 권한 대상자 항목에 권한을 상속한 대상이 표시됩니다. 설정 권한의 경우에는 권한 대상자 항목에 해당 값이 없습니다.

### 폴더 권한 복사하기

권한이 설정된 문서함, 폴더의 권한(상속 권한, 설정 권한 모두 포함)을 다른 폴더에 복사하는 방법은 다음과 같습니다.

1\.    관리자 웹페이지에서 **일반 모듈 관리 – Basic – 문서함 관리 – 폴더 권한 관리 – 권한 현황∙복사**를 선택합니다.

2\.    **권한현황∙복사** 페이지의 문서함 및 폴더 목록에서 권한을 복사할 원본 폴더를 선택합니다. 이때 권한이 설정된 폴더(![](<../../.gitbook/assets/img_006 (33).png>))를 선택해야 하며, 권한이 설정되지 않은 폴더(![](<../../.gitbook/assets/img_007 (27).png>)) 선택시에는 **권한 복사** 버튼이 비활성화됩니다.

3\.    페이지 우측 상단의 **권한 복사** 버튼을 클릭합니다.

![](<../../.gitbook/assets/img_008 (23).png>)

4\.    ‘**목적지 폴더 선택**’ 창에서 원본 폴더의 권한을 복사할 목적지 폴더를 선택 후 **확인** 버튼을 클릭합니다.

![](<../../.gitbook/assets/img_009 (19).png>)

1. **하위 폴더 포함**이 체크된 경우 원본 폴더의 권한과 하위 폴더의 권한을 함께 복사합니다. 단, 원본 폴더의 하위 폴더 권한은 목적지 폴더의 동일한 이름의 하위 폴더로만 복사됩니다. 목적지 폴더에 동일한 이름의 하위 폴더가 존재하지 않으면 하위 폴더의 권한은 복사되지 않습니다.
2. **하위 폴더 포함**이체크되지 않은 경우에는 원본 폴더의 권한(하위 폴더의 권한은 제외)이 목적지 폴더에 복사됩니다.
