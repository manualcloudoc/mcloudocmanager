# 문서이벤트 알림 관리하기

부서문서함 또는 개인문서함에 있는 파일에 대하여 읽기, 쓰기, 삭제 등의 문서이벤트가 발생한 경우 해당 내용을 사용자가 알림 메일로 받도록 설정할 수 있습니다. 관리자는 문서이벤트 알림 대상이 되는 문서함과 이벤트 종류, 알림 수신자를 설정하고, 사용자가 알림 수신을 설정한 내역을 조회하여 이를 변경할 수 있습니다.

#### <mark style="color:$primary;">문서이벤트 알림 설정하기</mark>

관리자는 다음과 같은 내용을 설정하여 문서이벤트를 관리합니다.

* **알림 대상이 되는 문서함의 종류**
* **알림 대상이 되는 문서이벤트의 종류**
* **해당 이벤트가 발생할 경우 메일을 받을 대상**

1. **Basic - 문서함관리 - 문서이벤트 알림 – 알림 설정**을 클릭합니다. 기본적으로 현재 설정되어 있는 상태가 표시됩니다.

<figure><img src="../../.gitbook/assets/img_000 (36).png" alt=""><figcaption></figcaption></figure>

2. 문서이벤트 알림을 받을 문서함과 문서함 내의 파일에 발생하는 이벤트의 종류를 선택합니다.

* 알림을 받을 문서함의 알림 수신 항목을 ON상태 ![](<../../.gitbook/assets/img_001 (35).png>) 가 되도록 합니다.
* 문서이벤트 종류를 한 개 이상 선택합니다.

<figure><img src="../../.gitbook/assets/img_002 (30).png" alt=""><figcaption></figcaption></figure>

문서이벤트의 종류는 다음과 같습니다.

> * **읽기**: 파일을 열람하거나 다운로드하는 경우
> * **쓰기**: 새로운 파일을 업로드하는 경우
> * **삭제**: 파일이 삭제되는 경우
> * **이름 변경**: 문서함에 있는 파일 이름이 변경되는 경우
> * **이동**: 파일이 문서함 내 다른 폴더로 이동, 다른 문서함으로 이동, 또는 다른 문서함에서 해당 문서함으로 이동되는 경우
> * **복사**: 파일이 문서함 내 다른 폴더로 복사, 다른 문서함으로 복사, 또는 다른 문서함에서 해당 문서함으로 복사되는 경우

{% hint style="warning" icon="square-poll-horizontal" %}
문서이벤트 종류는 시스템에서 실제로 수행된 작업으로, 사용자가 실행한 작업과 다르게 나타날 수 있습니다. 예를 들어 사용자가 문서함의 파일을 열고 내용을 수정하여 저장하는 경우, 시스템 내부적으로 읽기, 쓰기, 삭제, 이름 변경 등의 작업을 수행하게 되며 그에 따른 문서이벤트가 발생합니다.
{% endhint %}

3\.   문서이벤트 알림을 받을 사용자를 설정합니다.

* 부서문서함인 경우 알림 메일을 받을 사용자를 일반 사용자와 폴더관리자 중에서 선택합니다

<div align="left"><figure><img src="../../.gitbook/assets/img_004 (18).png" alt=""><figcaption></figcaption></figure></div>

> * 일반 사용자: 알림을 받도록 지정한 문서함에 대해 체크한 권한을 모두 가진 사용자에게 알림 메일이 발송됩니다.
> * 폴더관리자: 부서문서함의 폴더관리자에게 알림이 발송됩니다.

* 이메일을 직접 입력하면 문서함에 대한 권한에 상관없이 위에서 선택한 문서함과 문서이벤트에 대하여 발생한 내역을 메일로 발송합니다.

<div align="left"><figure><img src="../../.gitbook/assets/img_005 (16).png" alt=""><figcaption></figcaption></figure></div>

4. **적용** 버튼을 클릭합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
위의 설정에 따라 문서이벤트 알림 수신 대상이 되는 사용자는 문서함별로 문서이벤트 알림 수신 여부를 설정할 수 있습니다. 사용자는 관리자가 알림을 설정하고 사용자 본인이 알림 수신을 선택한 문서함에서 발생한 문서이벤트에 대해서만 알림 메일을 수신하게 됩니다.

사용자가 문서이벤트 알림 수신 여부를 설정하는 방법과 문서이벤트 알림 메일의 예는 [**문서이벤트 알림 수신 여부 설정하기**](https://app.gitbook.com/s/vQ0BiQsGY4PT08D7Nfay/basicmodule/undefined-2/undefined-2)를 참고합니다.
{% endhint %}

#### <mark style="color:$primary;">문서이벤트 사용자 알림 설정 내역 보기</mark>

관리자가 문서이벤트 알림을 설정하여 알림 대상이 되는 문서함, 문서이벤트의 종류와 알림 메일 수신자의 자격 권한을 지정하면, 알림 수신 자격이 있는 사용자는 알림 대상 문서함별로 수신 여부를 설정할 수 있습니다. 관리자는 사용자가 문서이벤트 알림 메일 수신을 위해 선택한 문서함들을 확인하고, 필요시 문서함 선택을 변경할 수 있습니다.  &#x20;

1. **Basic - 문서함관리 - 문서이벤트 알림 – 사용자 알림 설정 내역**을 클릭합니다. 기본적으로는 전체 문서함, 전체 사용자에 대한 내역이 검색되며 사용자별로 알림 수신 문서함이 목록으로 표시됩니다.&#x20;

<figure><img src="../../.gitbook/assets/img_007 (14).png" alt=""><figcaption></figcaption></figure>

2. 검색 조건을 입력하여 특정 문서함이나 사용자에 해당하는 문서이벤트 알림 설정 내역을 검색할 수 있습니다.

* **문서함**: 전체 문서함을 선택하거나 일부 문서함을 한 개 이상 선택할 수 있습니다. 기본적으로 선택 가능한 모든 문서함이 표시되며, 검색에서 제외할 문서함을 문서함 이름 우측의 X표시를 클릭하여 삭제합니다. **문서함 선택** 버튼을 클릭하여 아래와 같은 ‘**문서함 선택**’ 창에서 문서함을 선택할 수도 있습니다.

<div align="left"><figure><img src="../../.gitbook/assets/img_008 (12).png" alt=""><figcaption></figcaption></figure></div>

* **사용자 이름** 또는 **사용자 ID**: 사용자 이름이나 사용자 ID로 설정 내역을 검색합니다.

3. 알림 수신을 선택한 문서함을 변경하려면, 목록에서 우측의 변경 아이콘 을 선택합니다. ‘**문서함 변경**’ 창에 해당 사용자에게 알림이 설정된 모든 문서함이 표시됩니다. 필요시 알림을 받고자 하는 문서함을 변경하고 확인을 클릭합니다.

<div align="left"><img src="../../.gitbook/assets/img_009 (10).png" alt=""></div>
