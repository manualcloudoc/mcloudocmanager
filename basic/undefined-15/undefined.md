# 링크 공유 기능에 보안 절차 적용하기

링크 공유는 내부 사용자뿐만 아니라 계열사, 협력사 등 외부 수신자에게 중앙문서함 파일의 링크를 전달하여 대용량 파일을 간편하고 안전하게 공유할 수 있는 방법입니다. \

### 링크 공유 사용 권한 체크

사용자가 특정 파일의 링크를 공유하기 위해서는 해당 파일에 대한 링크 공유 사용 권한을 보유해야 합니다.

* 선택한 파일이 위치한 문서함(개인문서함 또는 부서문서함)에서의 링크 공유 기능 사용이 시스템에서 허용되어 있어야 합니다.
* 선택한 파일에 대해, **링크 공유를 위해 필요한 권한**(읽기 또는 읽기&쓰기)을 사용자가 보유하고 있어야 합니다.
* 선택한 파일이 **사용자에게 허용된 링크 공유 대상 파일 속성**(파일 개수, 용량, 확장자)을 만족해야 합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
선택한 파일이 위치한 문서함에서의 링크 공유 기능 사용이 허용되어 있고, 사용자가 해당 파일에 대해 **읽기** 권한을 보유한 경우에만, 사용자에게 링크 공유를 위한 컨텍스트 메뉴가 표시됩니다.
{% endhint %}

관리자는 이를 위해**설정홈**에서 링크 공유를 위해 필요한 권한을 지정(**읽기** 또는 **읽기&쓰기** 중 선택)할 수 있으며, 개인문서함 및 부서문서함에서의 링크공유 허용 여부를 링크 공유 기능별로 각각 설정할 수 있습니다.

또한, 링크 공유를 허용할 대상 파일의 속성(파일 개수, 용량, 확장자)은 **링크 공통 정책**에서 설정하여 부서 및 사용자별로 적용할 수 있습니다.

{% hint style="warning" icon="square-poll-horizontal" %}
링크 공통 정책을 설정하고 적용하는 방법은 [**링크 공통 정책 관리하기**](undefined-1.md)를 참고하세요.
{% endhint %}

### <mark style="color:$primary;">복사본 링크 생성</mark>

공유할 대상 파일에 대한 링크 생성 시, 대상 파일을 문서중앙화 서버의 임시 영역에 복사하고 원본 대신 복사본의 링크를 생성하여 공유할 수 있습니다. 복사본의 링크를 공유할 경우, 추후 원본 파일이 변경되더라도 수신자는 사용자가 링크를 생성할 당시의 파일을 다운로드하게 됩니다. 복사본은 지정된 기간 동안 보관 후 삭제됩니다.\
관리자는 **설정홈**에서 문서링크 복사 기능 사용 시 복사본 링크의 생성 여부를 지정할 수 있으며, 복사본 파일의 보관 기간도 설정할 수 있습니다.

**보안 웹링크 복사** 및 **보안 링크메일** 사용 시에는 기본적으로 복사본의 링크가 생성되며, 웹링크 복사, 링크메일의 경우 원본 링크가 생성됩니다.

{% hint style="success" icon="lightbulb-exclamation-on" %}
링크 생성 이후 원본 파일 수정 시, 원본 링크를 제공한 경우에는 수신자가 링크된 문서를 다운로드/열람 시 수정본을 참조하게 되는 반면, 복사본의 링크를 제공한 경우에는 수정 전에 복사된 사본을 참조하게 됩니다. 이 경우 수정 사항이 공유되지 않습니다.
{% endhint %}

### <mark style="color:$primary;">승인 절차</mark>

문서링크 복사, 보안 웹링크 복사 및 보안 링크메일 사용 시 관리자의 설정에 따라 승인권자의 승인 절차가 동반될 수 있습니다.  승인 필요 여부 및 결재 방식(선결재/후결재)은 기능에 따라 다음과 같이 설정할 수 있습니다.

* **문서링크 복사**:  문서링크 정책에서 수신자 유형(내부/계열사/협력사 직원)별로 승인 절차 진행 여부 및 결재 방식을 각각 설정하고, 부서/사용자별로 적용할 수 있습니다.
* **보안 웹링크 복사 및 보안 링크메일**: 설정홈에서 승인 절차 진행 여부와 결재 방식을 지정할 수 있습니다.

{% hint style="success" icon="lightbulb-exclamation-on" %}
보안 웹링크 복사 및 보안 링크메일의 결재 방식은 **설정홈 - 일반 모듈 설정 - Basic - 공유·협업 - 보안 웹링크 복사·보안 링크메일** 항목 중 **보안 링크메일 전송 시점** 설정값에 따라 정해집니다.



![](<../../.gitbook/assets/image (141).png>)
{% endhint %}

{% hint style="warning" icon="square-poll-horizontal" %}
문서링크 정책을 설정하고 적용하는 방법은 [**문서링크 정책 관리하기**](undefined-2.md)를 참고하세요.
{% endhint %}

관리자의 설정에 따라 승인이 필요한 경우, 링크 생성 시 승인권자에게 승인 요청 알림 및 메일이 발송되며, 승인권자는 이를 승인 또는 반려해야 합니다. 이때, 지정된 결재 방식이 **선결재**인 경우, 승인 여부에 따라 다음과 같은 제한이 발생합니다.

* 과거 버전에서 선결재 이전에도 링크를 보내는 것이 가능한 경우가 있습니다. 이 경우, 승인권자의 승인이 완료된 경우에 한해 수신자가 공유된 링크에서 파일을 다운로드(또는 열람)할 수 있습니다.
* 보안 링크메일 사용 시 설정에 따라 **링크메일 생성 시**가 아닌 **선결재 승인 완료 후** 메일이 전송될 수 있습니다. 보안 링크메일의 전송 시점은 **설정홈**에서 선택할 수 있습니다.

**승인권자**는 사용자가 링크 공유 요청 시 승인권자 목록에서 확인하거나, 경우에 따라 목록에서 선택할 수 있습니다. 목록에서 선택할 수 있습니다. 승인권자에 대한 자세한 사항은 사용자 매뉴얼 - [<mark style="color:$warning;">**승인권자 및 결재 방식 소개**</mark>](https://app.gitbook.com/s/vQ0BiQsGY4PT08D7Nfay/basicmodule/concept/approve) 를 참고하세요.&#x20;

{% hint style="warning" icon="square-poll-horizontal" %}
부서문서함 문서의 링크를 공유하는 경우에는 팀문서관리자와 폴더관리자가 모두 승인권자가 될 수 있으며, 관리자가 부서문서함/폴더별로 승인권자의 종류를 지정(팀문서관리자와 폴더관리자 중 택일)할 수 있습니다. 승인권자의 종류를 지정하는 방법은 [**승인권자 종류 지정하기**](../undefined-8/undefined-4.md)를 참고합니다.
{% endhint %}

{% hint style="warning" icon="square-poll-horizontal" %}
관리자가 자가 승인을 허용한 경우에는 승인권자가 사용자 본인으로 고정됩니다. 자가 승인 정책을 설정하고 부서/사용자별로 적용하는 방법은 [<mark style="color:$warning;">**자가승인 정책 관리 및 적용하기**</mark>](../undefined-8/undefined-1.md)를 참고합니다.
{% endhint %}

{% hint style="warning" icon="square-poll-horizontal" %}
승인 절차가 고객사의 전자결재 시스템과 연동하여 동작하도록 구성할 수 있습니다. 관련 사항은 문서중앙화 시스템 담당자와 협의하시기 바랍니다.
{% endhint %}

### <mark style="color:$primary;">수신자 인증</mark>

#### 수신자 인증 방식 소개

수신한 링크에서 파일을 다운로드/열람하기 위해 다음과 같은 수신자 인증 절차를 거칠 수 있습니다.

&#x20;•     **에이전트 로그인 인증**

> 수신자의 문서중앙화 에이전트 로그인 상태를 체크합니다.  로그인하지 않은 상태이면 로그인을 유도하여 등록된 사용자인지 확인합니다. 주로 내부 직원에게 공유 시 적용합니다.&#x20;

•      **웹 로그인 인증**

> 수신자가 인증 페이지에서 문서중앙화 계정 ID와 패스워드를 입력하여 인증하는 방법입니다.

•     **이메일** **OTP** **인증**

> 수신자가 자신의 이메일로 수신한 인증번호를 입력하여 인증하는 방법으로, 절차는 다음과 같습니다.
>
> 1. 수신한 링크를 클릭하면 인증 페이지가 표시됩니다.
> 2. 인증번호를 받을 이메일 주소를 입력한 후 **인증번호 발송** 버튼을 클릭합니다.
> 3. 입력한 이메일 주소로 수신한 인증번호를 인증 페이지의 인증번호 입력란에 입력합니다.
>
> **문서링크 복사** 요청 시 이메일OTP인증을 사용할 경우에는 사용자가 수신자의 이메일 주소를 입력하여야 하며, 이때 이메일 주소의 도메인이 관리자의 설정에 따라 제한될 수 있습니다. 또한 수신자가 이메일OTP 인증 과정에서 입력하는 이메일 주소는 사용자가 미리 지정한 수신자 이메일 주소와 일치해야 합니다.

•    **패스워드 인증**

> 사용자가 링크 공유 요청 시 패스워드(다운로드 암호)를 설정하여 패스워드 인증을 사용하도록 선택할 수 있습니다.패스워드 설정 시, 수신자는 링크에서 파일을 다운로드/열람하기 위해 설정된 패스워드를 입력해야 합니다.

•           **부서문서함 권한 인증**

> **문서링크 복사**를 이용하여 부서문서함 파일을 공유하는 경우, 수신자가 해당 부서문서함 경로에 대한 접근 권한을 보유한 경우에만 다운로드/열람할 수 있도록 설정할 수 있습니다. 이 경우, 수신자가 링크에서 파일을 다운로드/열람하기 위해서는 먼저 문서중앙화 로그인이 되어 있어야 하며, 해당 부서문서함 경로에 대한 접근 권한을 보유해야 합니다.

### <mark style="color:$primary;">다운로드/열람 제한</mark>

수신자가 공유된 링크에서 문서 다운로드/열람 시 다음과 같은 제한 사항을 적용할 수 있습니다.

#### 보안뷰어 사용

수신자가 공유된 링크에 접속 시, 문서를 다운로드 하는 대신 **보안뷰어**를 통해 문서를 열람하도록 설정할 수 있습니다.  보안뷰어를 사용할 경우, 문서를 PDF 형식으로 변환한 후에 다운로드 없이 스트리밍으로 열람하게 되므로, 문서의 유출을 방지할 수 있습니다.

**보안뷰어 사용 여부**는 각각의 링크 공유 기능별로 관리자가 **설정홈**에서 지정할 수 있습니다.

{% hint style="warning" icon="square-poll-horizontal" %}
보안뷰어 기능은 문서변환 서버를 구축한 경우에 사용 가능합니다. 자세한 사항은 [**보안뷰어 사용을 위한 문서변환 시스템 소개**](../../undefined.md)를 참고하세요.
{% endhint %}

{% hint style="success" icon="lightbulb-exclamation-on" %}
대외용 웹링크 복사 및 대외용 링크메일 사용 시의 보안뷰어 사용 여부는 **보안 웹링크 복사·보안 링크메일**의 **보안뷰어 사용 여부** 설정을 따릅니다.
{% endhint %}

#### 다운로드/열람 기간 및 횟수 제한

사용자는 링크 공유 요청 시 **다운로드 횟수** 및 **유효 기한**(또는 **사용 만료 일시**)을 설정하여, 공유된 링크에서의 다운로드(또는 열람) 허용 기간 및 횟수를 제한할 수 있습니다.  관리자는 사용자가 링크 공유 요청 시 선택 가능한 기한 및 횟수의 범위를 다음과 같이 설정할 수 있습니다.

* **문서링크 기능**:  **문서링크 정책**에서 수신자 유형(내부/계열사/협력사 직원)별로 다운로드 유효 기한 및 다운로드 최대 횟수를 설정하고, 부서/사용자별로 적용할 수 있습니다.
* **웹링크 기능**: **설정홈**에서 만료일시 최대 기한 및 다운로드 최대 횟수를 지정할 수 있습니다.

#### 압축 다운로드 용량 제한

**웹링크** 기능을 통해 수신한 링크 클릭 시, 다운로드 페이지가 표시됩니다. 이때 수신자가 다수의 파일을 한꺼번에 다운로드 하기 위해 **모두 저장** 버튼을 클릭하면 목록의 모든 파일들을 압축하여 다운로드할 수 있습니다. 관리자는 **설정홈**에서 압축 다운로드 최대 용량을 설정하여, 대용량 파일 다운로드로 인한 시스템의 과부하를 방지할 수 있습니다. 수신자가 다운로드 할 파일들의 전체 용량이 허용된 용량을 초과할 경우, 개별적으로 다운로드해야 합니다.

#### 웹 브라우저 제한

**웹링크** 기능을 통해 수신한 링크 클릭 시 표시되는 다운로드 페이지에서, 수신자가 사용하는 웹 브라우저에 따라 다운로드가 허용 또는 차단되도록 설정할 수 있습니다.  다운로드 웹 브라우저의 제한 여부 및 허용/차단할 웹 브라우저 목록은 **설정홈**에서 지정할 수 있습니다.
