# 공동 편집 문서함을 위한 DiskLock 정책 설정하기

## 공동 편집 문서함을 위한 DiskLock 정책 설정하기

공동 편집 문서함을 사용하려면 문서를 해당 문서함으로 복사하거나 이동한 후, 편집 및 저장이 가능하도록 다음 표와 같이 DiskLock 정책 설정을 변경해야 합니다.

|                                    |                                                                                                  |
| ---------------------------------- | ------------------------------------------------------------------------------------------------ |
| **DiskLock 정책**                    | **설정**                                                                                           |
| **① 탐색기 복사/이동 정책**                 | 중앙문서함  네트워크 디스크로 복사/이동 허용                                                                        |
| **② 로컬 저장금지 정책 (윈도우 애플리케이션)**      | 1. 윈도우 애플리케이션에 대해 네트워크 디스크에 쓰기를 금지하는 정책 아이템 추가 2. 윈도우 애플리케이션에 대해 공동 편집 문서함에만 쓰기를 허용하는 정책 아이템을 추가 |
| **③**로컬 저장금지 정책\*\*(엑셀 애플리케이션)\*\* | ​엑셀 애플리케이션에 대해 공동 편집 문서함에만 예외적으로 쓰기 및 삭제를 허용하는 정책 아이템 추가                                         |

&#x20;\
위와 같이 DiskLock 정책을 변경해야 하는 이유는 다음과 같습니다.   **①    탐색기 복사/이동 정책을 변경하는 이유** 공동 작업할 중앙문서함의 문서를 공동 편집 문서함으로 복사하거나 이동할 수 있어야 하기 때문입니다.이를 위해,윈도우 탐색기에서 중앙문서함에서 **네트워크 디스크**인 공동 편집 문서함으로 **파일 복사/이동이 허용**되어야 합니다. **②    윈도우 애플리케이션에 대해 로컬 저장 금지 정책 아이템을 추가하는 이유** 보안을 위해 **공동 편집 문서함을 제외**한 네트워크 디스크로 복사 및 이동은 금지해야 하기 때문입니다. 이를 위해, **첫번째 정책 아이템**은 윈도우 탐색기에서 모든 디스크에서 네트워크 디스크로의 쓰기를 금지하고, **두번째 정책 아이템**은 예외적으로 공동 편집 문서함에만 쓰기를 허용합니다.&#x20;

**③    엑셀 애플리케이션에 대해 로컬 저장 금지 정책 아이템을 추가하는 이유** 공동 편집 문서함에 저장된 엑셀 파일의 ‘통합 문서 공유’ 기능을 사용하려면, 해당 파일을 편집하고 저장할 수 있어야 하기 때문입니다. 이를 위해, 엑셀 애플리케이션에서 공동 편집 문서함에서만 쓰기를 허용하는 정책 아이템을 추가합니다.  \
![Notes](<../../.gitbook/assets/img_001 (15).png>)여기에서는 엑셀 애플리케이션의 문서를 공동 작업하는 경우를 예로 들었습니다. 다른 애플리케이션의 문서를 공동 작업할 경우, 해당 애플리케이션에 대한 정책 아이템을 추가해야 합니다.

## 탐색기 복사/이동 정책 설정하기

중앙문서함의 파일을 네트워크 디스크로 복사/이동을 허용하도록 탐색기 복사/이동 정책을 설정하는 방법은 다음과 같습니다.

1\.   웹에서 **PC 보안 모듈 관리** – **DiskLock** – **로컬저장금지** – **콘솔 실행**을 클릭합니다.&#x20;

2\.   ‘디스크락 콘솔’ 화면이 나타나면, **탐색기 복사/이동 정책**에서 **정책 관리** 메뉴를 클릭합니다.

3\.   현재 설정된 탐색기 복사/이동 정책 목록이 표시됩니다. 목록에서 공동 편집 문서함을 사용하는 사용자 PC에 적용된 탐색기 복사/이동 정책을 클릭합니다. \
4\.   다음과 같이 **중앙문서함 → 네트워크 디스크**로 복사/이동을 **허용**으로 설정한 후 **적용**을 클릭합니다. ![](<../../.gitbook/assets/img_002 (12).png>)&#x20;

### 로컬저장금지 정책 설정하기

#### 추가할 정책 아이템

공동 편집 문서함을 위해 로컬저장금지 정책에는 다음과 같은 3개의 정책 아이템을 추가해야 합니다. 정책 아이템의 이름은 다른 이름을 사용해도 됩니다.

①   **탐색기 애플리케이션에 대해 네트워크 디스크에 쓰기를 금지하는 정책 아이템**

|                 |                                                            |
| --------------- | ---------------------------------------------------------- |
| **항목**          | **설정**                                                     |
| **정책 아이템 이름**   | Reject\_탐색기에서 네트워크 디스크 쓰기                                  |
| **애플리케이션 카테고리** | Applications>>System>>Operating Systems>> Windows Explorer |
| **통제 작업**       | 쓰기 차단                                                      |
| **대상 파일**       | 네트워크 디스크                                                   |

**②   탐색기 애플리케이션이 공동 편집 문서함에만 쓰기를 허용하는 정책 아이템**

|                 |                                                            |
| --------------- | ---------------------------------------------------------- |
| **항목**          | **설정**                                                     |
| **정책 아이템 이름**   | Allow\_탐색기에서 공동 편집 문서함에 쓰기                                 |
| **애플리케이션 카테고리** | Applications>>System>>Operating Systems>> Windows Explorer |
| **통제 작업**       | 쓰기 허용                                                      |
| **대상 파일**       | 네트워크 디스크                                                   |
| **파일 경로**       | \Device\Mup<공동 편집 문서함으로 사용하는 서버의 IP 주소>                    |

**③   공동 편집 문서함에서 엑셀 애플리케이션에 대한 쓰기/삭제를 허용하는 정책 아이템**

|                 |                                                             |
| --------------- | ----------------------------------------------------------- |
| **항목**          | **설정**                                                      |
| **정책 아이템 이름**   | Allow\_공동편집함의 엑셀에서 쓰기/삭제                                    |
| **애플리케이션 카테고리** | Business Applications>>Productivity>>Spreadsheets>>MS Excel |
| **통제 작업**       | 쓰기/삭제 허용                                                    |
| **대상 파일**       | 네트워크 디스크                                                    |
| **파일 경로**       | “\Device\Mup{공동 편집 문서함으로 사용하는 서버의 IP 주소}                    |

#### 정책 아이템 추가하기

로컬저장금지 정책에 위 정책 아이템들을 추가하는 방법은 다음과 같습니다.

1\.   웹에서 **PC 보안 모듈 관리** – **DiskLock** – **로컬저장금지** – **콘솔 실행**을 클릭합니다. &#x20;

2\.   ‘디스크락 콘솔’ 화면이 나타나면, **로컬저장금지 정책**에서 **정책 관리** 메뉴를 클릭합니다.

3\.   현재 설정된 로컬저장금지 정책 목록이 표시됩니다. 목록에서 공동 편집 문서함을 사용하는 사용자 PC에 적용된 로컬저장금지 정책을 클릭한 후 도구 모음에서 **새 아이템** 아이콘을 클릭합니다.&#x20;

![](<../../.gitbook/assets/img_003 (13).png>)​\
4\.   ‘정책 아이템 추가’ 화면이 나타나면 첫번째 정책 아이템인 ‘**탐색기 애플리케이션에 대해 네트워크 디스크에 쓰기를 금지하는 정책 아이템**’을 생성하기 위해 각 항목의 값을 다음과 같이 입력한 후 **확인**을 클릭합니다.\
&#x20;![](<../../.gitbook/assets/img_004 (9).png>)

![Notes](<../../.gitbook/assets/img_005 (8).png>)‘정책 아이템 추가’ 화면의 각 항목에 대한 설명과 상세한 설정 방법은 \*\*[DiskLock 콘솔에서 로컬저장금지 정책 아이템 설정하기](../../disklock/disklock/disklock/disklock-1.md)\*\*를 참고합니다.

5\.   설정한 첫번째 정책 아이템이 다음과 같이 상단에 추가된 것을 확인할 수 있습니다.

![](<../../.gitbook/assets/img_006 (8).png>)\
6\.   두번째 정책 아이템인 ‘**탐색기 애플리케이션이 공동 편집 문서함에만 쓰기를 허용하는 정책 아이템**’을 추가하기 위해 도구 모음에서 **새 아이템** 아이콘을 클릭합니다. 그리고, ‘정책 아이템 추가’ 화면이 나타나면 화면의 각 항목의 값을 다음과 같이 입력한 후 **확인**을 클릭합니다.  \
![](<../../.gitbook/assets/img_007 (7).png>)\
7\.   설정한 두번째 정책 아이템이 다음과 같이 상단에 추가된 것을 확인할 수 있습니다.

![](<../../.gitbook/assets/img_008 (6).png>)\
8\.   마지막으로 세번째 정책 아이템인 ‘**공동 편집 문서함에서 엑셀 애플리케이션에 대한 쓰기/삭제를 허용하는 정책 아이템**’을 추가하기 위해 도구 모음에서 **새 아이템** 아이콘을 클릭한 후, ‘정책 아이템 추가’ 화면이 나타나면 화면의 각 항목의 값을 다음과 같이 입력한 후 **확인**을 클릭합니다. &#x20;

![](<../../.gitbook/assets/img_009 (5).png>)

9\.   세번째 정책 아이템까지 모두 추가되면 화면 우측 상단의 **적용**을 클릭하여 추가한 정책 아이템을 적용합니다.

![](<../../.gitbook/assets/img_010 (4).png>)
