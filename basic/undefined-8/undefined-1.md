# 자가승인 정책 관리 및 적용하기

자가 승인은 승인을 요청한 본인이 직접 승인권자가 되어 승인 업무를 담당하는 것으로, 자가 승인이 가능한 작업은 다음과 같습니다.

* **자료 반출**: 문서반출, 외부DRM 반출
* **링크 공유**: 문서링크, 보안 링크메일/웹링크 복사, 대외용 링크메일/웹링크 복사
* **보안 해제**: 프린트 워터마크나 인쇄 제어 등 일부 보안 정책의 해제

관리자(서비스관리자 및 정보보호관리자)는 위의 작업별로 자가 승인 허용 여부를 정책으로 설정하고, 이를 부서/사용자별로 적용할 수 있습니다.

#### <mark style="color:$primary;">자가승인 정책 관리하기</mark>

관리자는 다음과 같이 자가승인 정책을 생성하거나 기존 정책을 변경할 수 있습니다.&#x20;

1. 관리자 웹페이지에서 **일반 관리 – 인사 조직 관리 - 승인권자 관리 – 자가승인 – 정책 관리**를 선택합니다.
2. **정책 관리** 페이지의 좌측 정책 목록에서 추가 버튼  ![](<../../.gitbook/assets/img_000 (69).png>) 을 클릭하여 정책을 추가합니다. 기존 정책을 변경하는 경우에는 4단계로 넘어갑니다.

<figure><img src="../../.gitbook/assets/img_001 (64).png" alt=""><figcaption></figcaption></figure>

3. ‘**추가**’ 창에서 **정책명**(필수)과 **정책 설명**을 입력하고 **확인**을 클릭합니다.

<div align="left"><figure><img src="../../.gitbook/assets/img_002 (54).png" alt="" width="443"><figcaption></figcaption></figure></div>

4. **정책 관리** 페이지의 좌측 정책 목록에서 추가된 정책명을 클릭합니다. 기존 정책을 변경하고자 하는 경우, 변경하고자 하는 정책명을 클릭합니다.
5. 선택한 정책에 대하여 세부 항목을 설정합니다. 설정에 따라 해당 작업 신청 시 승인권자 선택 내용이 결정됩니다.

* **자료 반출 및 링크 공유**: 일반 문서와 보안등급문서의 자료 반출과 링크 공유 신청 시 자가 승인의 허용 여부를 결정합니다.

![](<../../.gitbook/assets/img_003 (44).png>)

> * 일반 문서에 대하여 자가 승인 허용 여부를 설정합니다.
> * 보안등급문서는 전체 등급에 대하여 허용 여부를 설정할 수 있고 보안 등급별로 세부 설정할 수 있습니다.
> * 보안 등급별로 세부 결정 시에는 우측의 **편집** 버튼을 클릭하여 ‘**보안 등급별 세부 설정**’ 창에서 보안 등급별로 자가 승인 허용 여부를 설정한 후 **확인** 버튼을 클릭합니다.

<div align="left"><img src="../../.gitbook/assets/img_004 (34).png" alt="" width="443"></div>



* **보안 해제**: 프린트 워터마크나 인쇄 제어 등 일부 보안 정책 해제 신청 시 자가 승인 허용 여부를 설정합니다.

<figure><img src="../../.gitbook/assets/img_005 (29).png" alt=""><figcaption></figcaption></figure>

{% hint style="warning" icon="square-poll-horizontal" %}
자가 승인을 허용하지 않는 경우 승인권자는 다음과 같이 설정됩니다.

* 자료 반출 및 링크 공유: 승인권자 종류 지정에서 설정된 승인권자(팀문서관리자 또는 폴더관리자)가 승인을 담당합니다.
* 보안 해제: 팀문서관리자가 승인을 담당합니다.
{% endhint %}

6. 페이지 우측 하단의 **적용** 버튼을 클릭합니다.

{% hint style="danger" %}
세부 항목 페이지에서 설정을 변경하더라도 **적용** 전에는 실제로 서버에 반영되지 않습니다. 반드시 **적용** 버튼을 클릭해야 변경 사항이 서버에 적용됩니다.
{% endhint %}

#### <mark style="color:$primary;">자가승인 정책 적용하기</mark>

사용자에게 적용된 자가승인 정책 내용에 따라 사용자의 자가 승인 허용 여부가 결정됩니다. 관리자는 다음과 같이 자가승인 정책을 부서 또는 사용자에게 적용할 수 있으며, 이때 임시 정책을 설정하여 지정한 기간 동안만 정책을 적용할 수도 있습니다.&#x20;

1. 관리자 웹페이지에서 **일반 관리 – 인사 조직 관리 - 승인권자 관리 – 자가승인 – 정책 적용**을 선택합니다.
2.  **조직도**에서 사용자 또는 부서를 선택한 후 화면 우측의 **정책 목록**에서 적용할 정책을 선택합니다.\
    &#x20;

    <figure><img src="../../.gitbook/assets/img_008 (19).png" alt=""><figcaption></figcaption></figure>



3. 부서/사용자에게 허용된 기간에만 정책을 적용할 경우, 우측의 임시 정책 **설정**을 클릭하면 ‘임시 정책 설정’ 창이 팝업됩니다.

<div align="left"><figure><img src="../../.gitbook/assets/img_009 (16).png" alt="" width="563"><figcaption></figcaption></figure></div>

* 임시 정책으로 설정할 정책을 선택하고 화살표를 클릭한 후 적용 기간을 설정하고 **확인** 버튼을 클릭합니다.
* 기간이 겹치지 않게 여러 개의 정책 선택이 가능합니다.
* 임시 정책을 모두 선택한 후 **확인** 버튼을 클릭합니다.

4. **적용** 버튼을 클릭합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
자세한 방법은 [**부서 및 사용자에게 정책 적용하는 방법**](../undefined-2/undefined-3.md)을 참고합니다.
{% endhint %}
