# 승인권자 종류 지정하기

​\
사용자가 자료의 반출(문서반출, 외부DRM 반출)이나 링크 공유(문서링크, 보안/대외용 웹링크) 신청 시 **팀문서관리자** 또는 **폴더관리자**가 승인을 담당할 수 있습니다.\
​\
관리자는 부서문서함/폴더별로 이러한 작업을 위한 승인권자 종류를 팀문서관리자와 폴더관리자 중에서 선택하여 한정할 수 있습니다.

{% hint style="warning" icon="square-poll-horizontal" %}
승인권자 종류별 특성은 다음과 같습니다.

* **팀문서관리자**

&#x20;**인사조직 관리 - 조직도 - 조직도 관리** 메뉴에서 조직도 편집기를 실행하여 팀별로 팀장/팀문서관리자를 설정할 수 있습니다. 팀장/팀문서관리자로 설정되는 즉시, 해당 팀의 일반 문서 작업에 대한 승인 권한이 부여됩니다.

* **폴더관리자**

윈도우 에이전트에서 **폴더관리자 설정** 컨텍스트 메뉴를 통해 폴더관리자를 설정할 수 있습니다. 관리자가 폴더에서 승인권자 종류를 폴더관리자로 지정하면, 해당 폴더관리자에게 일반 문서 작업에 대한 승인 권한이 부여됩니다.
{% endhint %}

승인권자 종류를 지정하는 방법은 다음과 같습니다.

1. 관리자 웹페이지에서 **일반 관리 - 인사 조직 관리 – 승인권자 관리 – 승인권자 지정 – 승인권자 종류 지정**을 선택합니다.
2. 좌측 **폴더 목록**에서 승인권자를 지정할 문서함/폴더를 선택하고, 우측 **승인권자 종류 지정**에서 팀 문서관리자와 폴더관리자 중에서 선택합니다.

* **지정하지 않음(정책제거, 상속)**&#xC744; 선택하면 해당 폴더에 대한 승인권자 종류 지정을 해제하고, 상위 폴더의 승인권자 종류를 따르도록 합니다.
* 문서함 또는 폴더에서 승인권자 종류 선택 시, 승인권자가 별도로 지정되지 않은 하위 폴더에도 선택 내용이 상속 적용됩니다.
* 복수의 문서함/폴더의 승인권자를 변경하려면 이 과정을 반복합니다.

<figure><img src="../../.gitbook/assets/img_001 (57).png" alt=""><figcaption></figcaption></figure>

3. 페이지 우측 하단의 **적용** 버튼을 클릭합니다.

{% hint style="danger" %}
승인권자 지정을 변경하더라도 **적용** 전에는 실제로 서버에 반영되지 않습니다. 반드시 **적용** 버튼을 클릭해야 변경 사항이 서버에 적용됩니다.
{% endhint %}

{% hint style="warning" icon="square-poll-horizontal" %}
루트 폴더에 위치하는 기본 승인권자는 승인권자를 지정하지 않은 모든 문서함/폴더의 기본 승인권자가 됩니다. 디폴트 값은 팀 문서관리자로 설정됩니다.
