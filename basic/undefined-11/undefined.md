# 파일 작업 로그 보기

문서중앙화 서버에서는 문서중앙화 웹페이지와 윈도우, 모바일, Mac 에이전트를 통한 로그인/로그아웃, 그리고 중앙문서함에서 발생하는 파일 작업에 대해 로그를 기록합니다.\
각각의 작업 로그는 접속한 에이전트, 문서함, 작업 종류별로 선택하여 확인할 수 있으며, 다양한 검색 조건을 사용하여 검색할 수 있습니다.&#x20;

![Notes](<../../.gitbook/assets/img_000 (92).png>)로그를 기록하는 파일 작업의 종류는 **시스템 설정**에 따라 달라질 수 있습니다. 자세한 내용은 **로그 설정** 아티클 내 항목 \*\*[파일 작업 로그](../../settings/undefined-1/undefined.md)\*\*을 참고합니다.공유 폴더에 대한 파일 작업과 게스트에 의한 파일 작업 로그는 별도 메뉴에서 확인할 수 있습니다.

1\.    **Basic - 로그 – 파일 작업 로그**를 클릭합니다.

![](<../../.gitbook/assets/img_001 (86).png>)

2\.    검색 조건을 입력하고 **검색**을 클릭합니다.

1. **기간 범위 선택(필수)**: 파일 작업 검색 기간을 한 달 단위로 선택합니다. (예:2024.06.01\~20204.06.30)
2. **범위내에서 기간 검색**: 선택한 한 달 범위 내에서 세부 기간을 선택합니다. 입력하지 않는 경우, 선택한 한 달 범위 기간이 그대로 선택됩니다.
3. **에이전트 구분(필수)**: 파일 작업을 수행한 에이전트를 웹, 윈도우, 모바일, Mac 에이전트 중에서 한 개 이상 선택합니다.
4. **문서함(필수)**: 전체 문서함을 선택하거나 일부 문서함을 한 개 이상 선택할 수 있습니다. 기본적으로 선택 가능한 모든 문서함이 표시되며, 검색에서 제외할 문서함을 문서함 이름 우측의 X표시를 클릭하여 삭제합니다. **문서함 선택** 버튼을 클릭하여 아래와 같은 ‘**문서함 선택**’ 창에서 문서함을 선택할 수도 있습니다.

> <img src="../../.gitbook/assets/img_002 (72).png" alt="" data-size="original">

1. **작업 종류(필수)**: 전체 작업 또는 한 개 이상의 작업을 선택할 수 있습니다.

> 1) **업로드**:중앙문서함으로 파일을 업로드하거나 중앙문서함 내 파일을 편집 저장하는 경우
> 2) **다운로드**: 중앙문서함에서 로컬 드라이브로 파일을 다운로드하거나 중앙문서함 내 파일 목록을 보거나 파일을 열어서 읽는 경우
>
> ![Notes](<../../.gitbook/assets/img_003 (60).png>)로그에 기록되는 작업 종류는 시스템에서 실제로 수행된 작업으로, 사용자가 실행한 작업과 다르게 나타날 수 있습니다.  예를 들어 사용자가 웹 브라우저를 통해 외부에서 중앙문서함으로 파일을 다운로드하는 경우, 시스템 내부적으로는 임시 파일의 생성, 업로드, 이름 변경 등의 작업을 수행하게 되며 그에 따른 로그가 기록됩니다.

1. **검색 조건(선택)**: 아래와 같은 검색 조건을 하나 이상 입력할 수 있으며 각 조건은 AND(조건을 모두 만족), OR(조건 중 최소 하나 만족), NOT(검색어를 포함하지 않음)으로 연결됩니다. 조건 항목 좌측의 +를 클릭하면 조건이 추가되고 -를 클릭하면 조건 항목이 삭제됩니다.

> 1) **작업자 ID/이름**:파일 작업을 수행한 작업자의 이름과 ID
> 2) **원본 파일명/경로**: 작업 대상 원본 파일의 경로와 이름
> 3) **변경 파일명/경로**: 파일 이름이나 경로가 변경된 경우 바뀐 경로 및 파일 이름
> 4) **작업자 IP**: 파일 작업을 수행한 작업자의 IP
> 5) **프로세스**: 윈도우 에이전트에서 작업한 로그의 경우, 파일 작업을 수행한 프로세스명으로 검색할 수 있습니다. 웹, 모바일, Mac 에이전트를 통한 작업의 경우, 프로세스 항목이 **웹, Android, MacOS**로 기록됩니다.
> 6) **파일 확장자명**: 작업한 파일의 확장자명
> 7) **파일 크기**: 작업한 파일의 크기
>
> ![Notes](<../../.gitbook/assets/img_004 (46).png>)검색 연산자(AND, OR, NOT)를 활용한 검색 방법은 \*\*[연산자를 이용하여 검색 범위 좁히기](https://github.com/manualcloudoc/mcloudoc-user-manual/blob/main/zoho-export/markdown/%EC%82%AC%EC%9A%A9%EC%9E%90-%EB%A7%A4%EB%89%B4%EC%96%BC/basic/%EC%97%B0%EC%82%B0%EC%9E%90%EB%A5%BC-%EC%9D%B4%EC%9A%A9%ED%95%98%EC%97%AC-%EA%B2%80%EC%83%89-%EB%B2%94%EC%9C%84-%EC%A2%81%ED%9E%88%EA%B8%B0.md)\*\*를 참고합니다.

3\.   검색 조건에 해당하는 파일 작업의 결과를 확인할 수 있습니다.
