# 서버 성능 모니터링 로그 보기

문서중앙화 각 서버에서는 일정한 시간 주기(기본 10분)로 서버 관제 배치가 실행됩니다. 관제 항목들의 로그를 체크하여 문서중앙화 서비스가 안정적으로 실행 중인지 확인하고 서버 성능 현황을 파악할 수 있습니다.

서버 관제의 진행 여부, 모니터링 로그 기록 사용 여부, 정상 기준 값 설정 등 서버 모니터링 관련 설정은 **로그 설정** 아티클 내 항목 [**서버 성능 모니터링** **로그**](../../settings/undefined-1/undefined.md#undefined-2)를 참고합니다.&#x20;

1\.     **Basic - 로그 – 서버 성능 모니터링 로그**를 클릭합니다.

![](<../../.gitbook/assets/img_001 (51).png>)

&#x20;2\.     검색 조건을 입력하고 **검색**을 클릭합니다.

1. **기간 선택(필수)**: 모니터링 로그를 조회할 기간을 선택합니다.
2. **사용률(선택)**: CPU, RAM, SWAP/RAM, DISK 사용량이 특정값 이상인 로그를 검색할 수 있습니다. 각 항목의 사용량을 10\~100% 사이에서 10% 간격으로 선택합니다. 각 항목의 사용량은 OR 조건으로 검색(1개 이상의 조건을 만족하면 검색)하며, 사용량을 선택하지 않고 검색할 수도 있습니다.
3. **서버 선택(필수)**: 전체 서버 또는 특정 서버를 선택합니다.

3\.     입력한 검색 조건에 해당하는 결과가 표시됩니다.

![](<../../.gitbook/assets/img_002 (44).png>)

1. **서버명(IP)**: 서버명과 해당 서버에 부여된 IP가표시됩니다. 하나의 서버가 여러 개의 IP를 가질 수 있습니다.
2. **CPU 사용률**: 전체 수행 시간 대비 CPU 사용 시간의 비율을 나타내며 퍼센트로 표시됩니다.
3. **RAM사용률**: 전체 RAM용량 대비 사용된 RAM 용량이 퍼센트로 표시됩니다.
4. **SWAP 사용률**: 할당된 SWAP메모리 대비 사용중인 SWAP메모리 비율이 퍼센트로 표시됩니다.
5. **DISK 사용률**: 각 DISK의 사용률이 퍼센트로 표시됩니다.
