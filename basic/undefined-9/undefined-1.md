# 제품 제거 요청 처리하기

DiskLock 모듈을 사용하는 경우 사용자는 관리자의 승인 후 제품을 제거할 수 있습니다.   이때, 사용자는 **온라인 제거** 또는 **오프라인 제거** 방식을 통해 제품을 제거할 수 있습니다.

* **온라인 제거**: 사용자가 PC에서 \[ClouDoc 제거] 또는 \[제어판]-\[프로그램 제거]를 실행하여 제품 제거를 요청할 경우 자동으로 관리자에게 제거 요청 사항이 전달됩니다. 관리자는 제품 제거 요청의 내용을 확인하고 승인 여부를 결정합니다.
* **오프라인 제거**: 관리자가 제품 제거를 위한 인증 코드를 발급하고, 사용자에게 메일이나 USB로 인증 코드를 전달합니다. 사용자는 전달받은 인증 코드를 입력하여 제품을 제거합니다.

PC 환경이 문서중앙화에 로그인할 수 있는 경우에는 온라인, 오프라인 방식 모두 사용할 수 있습니다. 반면, PC 환경이 문서중앙화에 로그인할 수 없는 경우에는 오프라인 방식으로만 제품을 제거할 수 있습니다.

{% hint style="warning" icon="square-poll-horizontal" %}
DiskLock 모듈을 사용하지 않는 경우 관리자의 승인 없이 사용자가 윈도우 에이전트를 직접 제거할 수 있습니다.
{% endhint %}

### <mark style="color:$primary;">온라인 제거 요청 처리하기</mark>

#### 사용자의 제거 요청에 대하여 승인/반려하는 방법

사용자가 제품 내에서 삭제를 요청하는 경우, 자동으로 관리자에게 삭제 요청이 전달됩니다. 관리자는 아래와 같은 방법으로 제거 요청에 대해 승인/반려 처리를 할 수 있습니다.

1. **일반 관리 - 윈도우 에이전트 설치 관리 - 온라인 제품 제거 - 요청 처리**를 클릭합니다.

<figure><img src="../../.gitbook/assets/img_001 (71).png" alt=""><figcaption></figcaption></figure>

2. **사용자 이름(ID), 사용자 IP, 제거 사유** 등의 항목에서 사용자가 요청한 내용을 확인합니다.
3. 목록에서 승인/반려할 항목의 체크박스를 선택한 후 **승인** 또는 **반려** 버튼을 클릭합니다.&#x20;

{% hint style="warning" icon="square-poll-horizontal" %}
관리자가 제거 요청을 승인하면 사용자는 다시 한 번 설치 제거를 시도하여 에이전트를 삭제할 수 있습니다.
{% endhint %}

#### 사용자의 제거 요청에 대한 처리 작업 이력 보기

사용자의 윈도우 에이전트 제거 요청 및 진행 상태, 제거 완료 여부 등 처리 작업 이력을 조회할 수 있습니다.&#x20;

1. **일반 관리 - 윈도우 에이전트 설치 관리 - 온라인 제품 제거 - 작업 이력**을 클릭합니다.

![](<../../.gitbook/assets/img_003 (48).png>)

2. 검색 항목을 입력합니다.

* **작업일시**: 관리자가 최종적으로 사용자의 요청 사항을 처리한 작업일시.
* **작업 상태**:

> - **요청:** 사용자가 제거 요청하였으나 아직 관리자가 처리하지 않음.
> - **승인:** 관리자가 승인 처리를 하였지만, 사용자가 아직 제거하지 않음.
> - **반려**\*\*:\*\* 관리자가 승인을 거부함.
> - **취소**: 관리자가 요청 처리를 하기 전에 사용자가 제거 요청을 취소함. 또는 사용자가 반려 메시지를 받고 제거 요청을 취소함.
> - **완료**: 관리자의 승인을 받고 사용자가 제품을 제거함.
> - **재요청**: 관리자가 반려하였지만 사용자가 다시 요청함.

3. **검색**을 클릭합니다.
4. 목록에서 최종 작업 일자를 기준으로 작업 상태, 제거 사유 등의 작업 이력을 확인할 수 있습니다.

### <mark style="color:$primary;">서버 접근이 불가능한 경우 제품 제거하기</mark>

#### 인증코드 발급하기

PC 환경이 문서중앙화에 로그인할 수 없는 경우, 즉 서버 접근이 불가능한 상태에서 윈도우 에이전트를 제거하려면 관리자가 인증코드를 발급 후 해당 사용자에게 전달하여야 합니다.

1. **윈도우 에이전트 설치 관리 - 오프라인 제품 제거 - 인증코드 발급**을 클릭합니다.

![](<../../.gitbook/assets/img_004 (37).png>)

2. 인증코드 종류를 선택합니다.

* **사용자용**: 특정 사용자의 윈도우 에이전트만 제거할 수 있는 인증코드. 사용자용을 선택한 경우 **사용자 ID**를 입력하여야 합니다.
* **관리자용**: 모든 사용자의 윈도우 에이전트를 제거할 수 있는 인증코드.

{% hint style="danger" %}
관리자용으로 인증코드를 발급할 경우에는 모든 사용자의 윈도우 에이전트 제거가 가능하므로 인증코드 관리에 주의하시기를 바랍니다.&#x20;
{% endhint %}

3. **제품 제거 유효기간**을 설정합니다. 해당 기간 내에서만 인증코드가 유효합니다.
4. **추가 정보 입력**에서 **요청자 이름, 제거 사유**를 입력합니다. 이 내용을 입력해야 발급 이력에서 추후 관련 정보를 다시 확인할 수 있고, 인증코드를 재발행할 때 해당 정보를 참고할 수 있습니다.
5. **확인**을 클릭합니다.
6. ‘**인증코드 발급**’ 창이 뜨면 **복사** 버튼을 클릭합니다. 인증코드가 클립보드에 복사됩니다.

<figure><img src="../../.gitbook/assets/img_006 (26).png" alt=""><figcaption></figcaption></figure>

7. 해당 인증코드를 사용자에게 메일이나 메신저, USB 등으로 전달합니다.

#### 발급한 인증코드 이력 보기

**윈도우 에이전트 설치 관리-오프라인 제품 제거-인증코드 발급 이력** 메뉴에서 오프라인 제거를 위해 인증코드를 발급한 이력을 확인할 수 있습니다. 이때 요청자 이름, 사용자 ID, 부서명 또는 발급자 ID 및 IP로 이력을 검색할 수 있습니다.

#### 인증코드 재발급하기

인증코드를 재발급하는 방법은 다음과 같습니다.

1\.    **윈도우 에이전트 설치 관리 - 오프라인 제품 제거 - 인증코드 발급 이력**을 클릭합니다.

![](<../../.gitbook/assets/img_007 (22).png>)

2. **발급 기간** 및 **검색 조건**을 입력합니다.

* 사용자 ID, 요청자 이름, 발급자 이름 및 IP, 부서명으로 검색할 수 있습니다.

3. **검색**을 클릭합니다.&#x20;
4. 목록에서 재발급할 **인증코드** 항목의 **복사** 버튼을 클릭합니다.&#x20;

{% hint style="warning" icon="square-poll-horizontal" %}
유효 기간이 지난 경우 복사 버튼이 보이지 않습니다.
{% endhint %}

5. 인증코드가 클립보드에 복사됩니다.&#x20;
6. 재발급된 인증코드를 사용자에게 메일이나 메신저, USB 등으로 전달합니다.
