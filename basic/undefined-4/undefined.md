# 신규 도메인 등록하기

### 신규 도메인 등록<mark style="color:$primary;">하기</mark>

이 장은 솔루션 구성을 멀티도메인으로 한 경우에만 해당합니다. 멀티도메인 구성에서는 기본 도메인을 운영하면서 여기 설명하는 기능을 이용하여 추가적으로 서브 도메인 운영이 가능합니다. 멀티도메인 관련 자세한 내용은 [**싱글도메인과 멀티도메인**](../../system/sw/undefined.md)을 참고해 주시기 바랍니다.

#### 추가 도메인 생성하기

1. **일반관리 - 도메인 관리 - 추가 도메인 생성**을 클릭합니다.**또는 일반관리 - 도메인 관리 - 검색**을 클릭한 후 **추가**를 클릭합니다.
2. 도메인 생성에 필요한 정보인 추가 도메인 정보, 정보보호관리자 정보, 서비스관리자 정보, 옵션 사항을 입력합니다.

![](<../../.gitbook/assets/img_000 (64).png>)

* **추가 도메인 정보(필수)**

> - **추가 도메인**: 사용자가 접속할 도메인의 주소(DNS명)를 입력합니다. 싱글도메인/멀티도메인 유형에 따라 아래 표와 같이 명명하는 것을 권장합니다. IP 주소는 사용할 수 없습니다.

<figure><img src="../../.gitbook/assets/img_001 (59).png" alt=""><figcaption></figcaption></figure>

> * **사이트명:** 쉽게 부를 수 있고 검색어로도 적합한 도메인의 이름을 만듭니다. 예) ‘엠클라우독’
> * **최대 사용자 수:** 이 도메인에 등록할 수 있는 최대 사용자 수입니다. 멀티도메인에 허용된 전체 사용자 수(**기본도메인**의 **사용자관리** 메뉴에서 **추가 가능한 사용자 수** 확인 가능)보다 클 수 없습니다.
> * **전체 용량:** 이 도메인에서 사용 가능한 최대 용량입니다. 멀티도메인에 허용된 전체 용량(**기본도메인**의 **서버관리** 메뉴에서 **전체 용량** 확인 가능)보다 클 수 없습니다. 개인문서함과 부서문서함에 분배하여 사용해야 합니다.
> * **서버 위치:** 도메인용 파티션의 지역을 선택합니다.
> * **중요도:** 도메인용 파티션의 중요도를 선택합니다.

* **정보보호관리자 정보**

> - 필수 입력 필드인 ID, 이름, 비밀번호, 비밀번호 확인, 이메일을 입력합니다.&#x20;
> - **메뉴 선택**을 클릭하여 필요한 메뉴를 정보보호관리자에게 제공할 수 있습니다.

* **서비스보호관리자 정보**

> - 필수 입력 필드인 ID, 이름, 비밀번호, 비밀번호 확인, 이메일을 입력합니다.&#x20;
> - 서비스관리자가 사용할 메뉴는 여기서 설정하지 않습니다.  이후 정보보호관리자가 설정합니다.

* **옵션 사항**

> - **부서문서함:** 도메인과 동시에 부서문서함을 생성하려면 **생성** 옵션을 선택합니다. 부서문서함에 할당할 용량도 입력합니다. **생성안함**을 선택하여 부서문서함을 생성하지 않아도 됩니다.

{% hint style="success" icon="lightbulb-exclamation-on" %}
부서문서함은 도메인 생성 이후에 언제라도 조직도 편집기에서 추가할 수 있습니다.
{% endhint %}

> * **로고이미지**: **파일 선택**을 클릭하여 로고 이미지 파일을 등록할 수 있습니다. 웹페이지와 윈도우 에이전트 등에서 사용합니다. 이미지 파일은 png만 가능합니다.
> * **CopyRight 정보 입력:** 회사의 저작권 표기를 할 수 있습니다. 접속하는 홈페이지 화면 중앙 하단에서 내용을 볼 수 있습니다.

3. **확인**을 클릭합니다.
