# 게시판 관리하기

게시판에서는 사내 소식과 중요한 이슈를 내부 직원들과 공유할 수 있습니다.

게시판의 최초 관리 권한은 정보보호관리자에게 있으며 정보보호관리자는 필요에 따라 서비스관리자에게 게시판 관리 권한을 부여할 수 있습니다.

{% hint style="warning" icon="square-poll-horizontal" %}
정보보호관리자가 서비스관리자에게 게시판 관리 권한을 부여하는 방법은 [**서비스관리자 계정 추가, 메뉴 설정하기**](undefined/undefined-4.md)를 참고하세요.
{% endhint %}

### <mark style="color:$primary;">게시판 종류</mark>

* #### **공지사항**

> 공지사항 게시판에서는 전사적으로 공유되어야 할 중요한 이슈를 간편하고 빠르게 공유할 수 있습니다. 중요한 공지사항의 경우 알림을 설정하여 사용자에게 **공지 알림 메시지**를 전송할 수 있습니다.

* #### **FAQ**

> 사용자들이 자주 묻는 말을 모아 FAQ 게시판에서 공유할 수 있습니다. FAQ 게시판은 관리자의 사용자 지원업무를 줄여줄 뿐만 아니라, 사용자의 궁금증을 빠르게 해결하는 데도 도움을 줄 수 있습니다.

* #### **Q\&A**

> 사용자는 궁금한 사항에 대해 Q\&A 게시판에 글을 올려 관리자에게 직접 질의할 수 있습니다. 이에 대해 관리자는 1:1로 답변을 남깁니다.

* #### **자료실**

> 자료실 게시판에서는 이미지나 파일 등의 업무 자료를 공유할 수 있습니다.

### <mark style="color:$primary;">공지사항, FAQ, 자료실 게시글 쓰기</mark>

1. **일반 모듈 관리 - Basic - 게시판**에서 게시글을 올릴 **공지사항/FAQ/자료실**을 클릭합니다.
2. 목록 우측 상단의 **글쓰기** 버튼을 클릭합니다.

![](<../.gitbook/assets/img_001 (21).png>)

3. 제목과 본문을 입력합니다. 하단의 에디터 옵션 도구 기능을 이용하여 글자 크기 및 스타일 변경, 들여쓰기, 글 정렬, 이미지 삽입, 파일 첨부를 할 수 있습니다.

* 첨부 파일 1개의 최대 용량은 1GB입니다.
* 첨부할 수 있는 파일의 개수 제한은 없으며 전체 파일의 합계 용량 1GB까지 업로드할 수 있습니다.
* 이미지는 전체 이미지의 합계 용량 5MB까지 업로드할 수 있습니다. 이미지 파일 형식(JPG, JPEG, GIF, BMP 등)이 아닌 경우에는 업로드할 수 없습니다.
* 게시글 내용은 100,000 byte까지 입력할 수 있습니다.
* 동영상은 지원하지 않습니다.

{% hint style="success" icon="lightbulb-exclamation-on" %}
이미지의 경우 이미지를 선택한 후 본문에 끌어다 놓아서 삽입할 수 있습니다.
{% endhint %}

4\.    **등록**을 클릭합니다.

{% hint style="danger" %}
게시판의 첨부 파일은 서버에 가장 먼저 생성된 전사문서함 경로에 저장됩니다. (팀용 부서문서함에는 저장되지 않습니다.) 전사문서함이 없는 경우에는 글쓰기 페이지에서 파일을 첨부할 수 없습니다. 파일 첨부를 위해서 전사문서함을 반드시 먼저 생성해야 합니다.&#x20;

공지사항/FAQ/자료실에서 첨부 파일이 저장되는 서버 위치는 각각 다음과 같습니다.&#x20;



<img src="../.gitbook/assets/img_004 (11).png" alt="" data-size="original">
{% endhint %}

### <mark style="color:$primary;">게시글 전체 표시 설정 및 해제하기</mark>

기본 도메인의 관리자는 공지사항/FAQ/자료실 게시글에 **전체 표시**를 설정할 수 있습니다. 전체 표시를 설정하면 추가 도메인 게시판에도 해당 게시글이 일괄적으로 등록됩니다. 등록 시에는 제목에 \[전체] 표시와 함께 게시글 목록 최상단에 올려져 사용자가 쉽게 조회할 수 있습니다.

#### **설정하기**

1. **일반 모듈 관리 - Basic -게시판**에서 게시글을 올릴 공지사항/FAQ/자료실을 클릭합니다.
2. 우측 상단의**글쓰기**버튼을 클릭하고 제목과 본문을 입력합니다.
3. 우측 하단의 **전체 표시** 버튼 ![](<../.gitbook/assets/img_005 (10).png>)을 활성화합니다.
4. **등록**을 클릭합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
멀티 도메인을 구입한 경우에만 사용할 수 있는 기능입니다.
{% endhint %}

{% hint style="warning" icon="square-poll-horizontal" %}
추가 도메인의 관리자는 게시글 전체 표시 기능을 사용할 수 없습니다. 기본 도메인의 관리자만 사용할 수 있는 기능입니다.
{% endhint %}

#### **해제하기**&#x20;

1. **공지사항/FAQ/자료실**에서 전체 표시 설정을 해제할 게시글을 클릭합니다.
2. **수정**을 클릭합니다.
3. 우측 하단의 **전체 표시** 버튼 ![](<../.gitbook/assets/img_008 (8).png>)을 비활성화합니다.
4. **등록**을 클릭합니다.

### <mark style="color:$primary;">공지사항 알림 설정하기</mark>

중요한 공지사항의 경우 알림을 설정할 수 있습니다. 공지사항 알림을 설정하면 사용자는 윈도우 에이전트에 로그인할 때마다 반복 알림을 받을 수 있습니다. 또한, 모바일로 접속할 때 알림창에서 공지 알림을 확인할 수 있습니다.

1. **일반 모듈 관리- Basic -게시판-공지사항**을 클릭합니다.
2. 우측 상단의**글쓰기**버튼을 클릭하고 제목과 본문을 입력합니다.
3. 본문 하단의 **알림 사용** 체크박스를 선택합니다.&#x20;

![](<../.gitbook/assets/img_009 (7).png>)

4. **알림 만료 일시**와 **로그인 시 항상 표시 여부**를 설정합니다.

* **알림 만료 일시**: 사용자가 알림 메시지를 받거나 알림을 확인할 수 있는 기한입니다.
* **로그인 시 항상 표시**:

> - **사용**을 선택한 경우사용자가 윈도우 에이전트에 로그인할 때마다 알림 메시지를 표시합니다.
> - **사용 안 함**을 선택한 경우 사용자가 로그인하여 알림 메시지를 한 번 받으면 그 이후의 로그인부터는 알림 메시지를 표시하지 않습니다.

{% hint style="warning" icon="square-poll-horizontal" %}
윈도우 에이전트 알림 메시지는 다음과 같습니다. 사용자가 로그인할 때 화면 우측 하단에 메시지를 표시합니다.



<img src="../.gitbook/assets/img_010 (6).png" alt="" data-size="original">
{% endhint %}

5. &#x20;**등록**을 클릭합니다.

### <mark style="color:$primary;">공지사항, FAQ, 자료실 게시글 수정하기</mark>

1. **공지사항/FAQ/자료실**에서 수정할 게시글의 제목을 선택합니다.
2. 우측 하단의 **수정**버튼을 클릭합니다.
3. 수정할 내용을 입력합니다.
4. **등록**을 클릭합니다.

### <mark style="color:$primary;">공지사항, FAQ, 자료실 게시글 삭제하기</mark>

1. **공지사항/FAQ/자료실**목록에서 삭제할 게시글의 체크박스를 선택합니다.
2. 우측 상단의 **삭제** 버튼을 클릭합니다.

{% hint style="danger" %}
삭제한 게시글은 영구 삭제되어 복구할 수 없습니다.&#x20;
{% endhint %}

### <mark style="color:$primary;">Q\&A 게시판 관리하기</mark>

#### **Q\&A 1:1 문의 답변하기**

사용자가 작성한 문의 사항에 관리자가 답변합니다. 1:1 문의이므로 작성한 사용자와 관리자만 볼 수 있습니다.

1. **일반 모듈 관리 - Basic - 게시판 - Q\&A**를 클릭합니다.

<figure><img src="../.gitbook/assets/img_012 (5).png" alt=""><figcaption></figcaption></figure>

2. 목록에서 사용자가 작성한 질문을 조회합니다.

* **구분**: 사용자가 질문을 등록할 때 선택한 카테고리입니다. 3가지 카테고리(서비스 이용방법, 회원관련, 기타문의)로 구분됩니다.
* **처리 상태**: 관리자가 답변을 등록하기 전에는 ‘접수’ 상태이며 관리자가 답변을 등록하면 ‘답변완료’ 로 표시됩니다.

3. 게시글 **제목**을 클릭하여 사용자가 작성한 문의 내용을 확인합니다.
4. 답변을 입력합니다.
5. **등록**을 클릭합니다.

#### **Q\&A 1:1 문의 삭제하기**

사용자가 등록한 게시글을 관리자가 삭제할 수 있습니다. 관리자가 게시글을 삭제할 경우 질문을 등록했던 사용자도 해당 게시글을 더 이상 조회할 수 없습니다.

1. **Q\&A** 목록에서 삭제할 게시글의 체크박스를 선택합니다.&#x20;
2. 우측 상단의 **삭제** 버튼을 클릭합니다.
