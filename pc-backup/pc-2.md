# 관리자가 PC 백업 이력 확인하는 방법

관리자는 PC 백업 기능에 의해 사용자PC 내 파일이 중앙문서함에 백업된 이력을 기간, OS 종류, 작업 결과, 사용자 이름/ID, PC 주소로 검색하여 확인할 수 있습니다.\
백업 이력에는 관리자의 PC백업 정책에 따라 실행된 자동 백업 이력과, 사용자가 설정한 즉시 백업, 예약 백업 이력이 모두 포함됩니다.\
​![Notes](<../.gitbook/assets/img_000 (169).png>)PC 백업 기능에 대한 설명은 **사용자 매뉴얼 -** [**중앙문서함에 PC 파일 백업하기**](https://github.com/manualcloudoc/mcloudoc-user-manual/blob/main/zoho-export/markdown/%EC%82%AC%EC%9A%A9%EC%9E%90-%EB%A7%A4%EB%89%B4%EC%96%BC/pc-backup/%EC%A4%91%EC%95%99%EB%AC%B8%EC%84%9C%ED%95%A8%EC%97%90-pc-%ED%8C%8C%EC%9D%BC-%EB%B0%B1%EC%97%85%ED%95%98%EA%B8%B0.md)를 참고합니다.1.   **백업 모듈 관리 - PC 백업 – 백업 이력**을 클릭합니다.

![](<../.gitbook/assets/img_001 (156).png>)\
2\.   **백업 이력** 페이지에서 조회를 원하는 **기간**, **OS\*\*\*\*종류, 작업결과**를 선택합니다.

1. **OS 종류**: 백업된 사용자 PC의 OS 종류를 **Windows, Mac**, 또는 **전체** 중에서 선택합니다.
2. **작업 결과**: 해당 백업 작업의 성공, 실패 여부 등 조회할 작업 결과를 선택합니다.

> 1) **성공**: 백업 작업이 성공적으로 완료된 경우
> 2) **부분성공**: 일부 파일에 대해서만 백업이 성공한 경우
> 3) **실패**: 백업 작업이 실패한 경우
> 4) **취소**: 백업 실행 도중 사용자가 백업을 중지한 경우
> 5) **변경사항 없음**: 증분 백업 시 백업할 원본 폴더/파일에 변동 사항이 없어 백업이 수행되지 않은 경우

3\.   필요 시 **검색 조건**에 **사용자 이름/ID, 사용자 IP, 디바이스 주소**를 입력하여, 검색 결과를 한정할 수 있습니다.  이때 검색어의 일부만 입력하여 검색할 수 있습니다.

1. **사용자 IP**: 백업이 수행된 PC의 IP 주소
2. **디바이스**: 백업이 수행된 PC의 MAC주소

4\.   **검색** 버튼을 클릭하면, 입력한 조건에 해당하는 백업 이력이 하단에 표시됩니다.

1. **백업 시작일시**: 백업 작업이 시작된 일시
2. **백업 종료일시**: 백업 작업이 종료된 일시
3. **사용자 이름(ID)**: 백업이 수행된 PC에서 문서중앙화 에이전트에 로그인한 사용자
4. **사용자 IP**: 백업이 수행된 PC의 IP 주소
5. **OS 종류**:백업이 수행된 PC의 OS 종류

> 1) **win**: Windows
> 2) **mac**: Mac OS

1. **디바이스**: 백업이 수행된 PC의 MAC주소
2. **백업된 파일 수**: {백업에 성공한 파일 수}/{백업 대상 파일 수}

> 1) 증분 백업의 경우, 백업 대상 파일에는 이전 백업 시와 변동 사항이 있는 파일만 포함됩니다.

1. **백업 용량**: {백업에 성공한 파일의 전체 용량}/{백업 대상 파일의 전체 용량}
2. **소요시간**:백업에 소요된 시간
3. **작업상태**

> 1) **백업(예약)**: PC 백업 정책에 따라 예약된 백업이거나 사용자가 예약한 백업이 수행된 경우
> 2) **백업(수동)**: 사용자가 즉시 백업을 수행한 경우

1. **작업 결과**: 2번의 작업 결과를 참고합니다.
2. **백업 실패 파일 목록**: 백업 실패, 취소 등으로 백업에 실패한 파일이 있을 경우, 이 항목에 **보기** 버튼이 표시됩니다. **보기**를 클릭하면 백업에 실패한 파일 목록을 확인할 수 있습니다.
