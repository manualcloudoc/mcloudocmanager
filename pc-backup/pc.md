# PC 백업 정책 관리하기

PC 백업 정책을 사용하면 사용자 PC의 중요 문서가 중앙문서함에 자동 백업되도록 설정할 수 있습니다. 관리자는 PC 백업 정책을 생성하여 백업 대상 파일의 정보, 백업을 저장할 목적지 경로와 백업 주기를 설정할 수 있으며, 부서/사용자에 따라 개별적인 정책을 적용할 수 있습니다.

PC 백업은 윈도우와 Mac PC에서 지원되며, 백업 과정에서 로컬디스크의 파일이나 폴더에 대한 추가, 변경, 삭제 등이 모두 서버에 동기화하여 반영됩니다. ​

#### <mark style="color:$primary;">정책 추가 및 변경하기</mark>

1. **백업 모듈 관리 - PC백업 - 정책 관리**를 클릭합니다.

<figure><img src="../.gitbook/assets/img_000 (167).png" alt=""><figcaption></figcaption></figure>

2. **정책 목록**에서 추가 버튼 ![](<../.gitbook/assets/img_001 (154).png>)을 클릭하여 정책을 생성합니다. 기존 정책을 변경할 경우에는 정책 목록에서 변경할 정책을 선택합니다.

* 정책 추가 시 **‘추가’** 창에서 **정책명**(필수)과 **정책 설명**을 입력합니다.

3. **PC 기기별 설정** 탭에서 백업 설정할 기기 종류(**윈도우, 맥**)를 선택한 후 각 항목 영역에서 다음과 같이 설정값을 선택 또는 입력합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
이하 메뉴 항목 설정은 윈도우 기기를 중심으로 설명합니다. 맥(Mac) 기기에 대한 설정도 다음 목록을 제외하고는 기본적으로 모든 항목에서 동일합니다.

* 백업 원본 경로 입력 시 사용 가능한 매크로&#x20;
* 기본 백업 제외 폴더 목록&#x20;
* 백업 제외 폴더 입력 시 사용 가능한 예약 폴더
{% endhint %}

![](<../.gitbook/assets/img_003 (116).png>)

* **백업 방식 지정자**: 관리자가 정책에서 백업 방식(백업 대상 파일, 목적지 경로, 백업 주기)을 설정하거나, 사용자가 PC에서 직접 설정하도록 선택할 수 있습니다.

> - **관리자**: 관리자가 정책에서 백업 원본 경로, 백업 목적지 경로, 백업 제외 폴더, 백업할 확장자, 백업 주기를 설정하며, 해당 정책이 적용된 사용자 PC에서는 설정된 정책에 따라 자동 백업이 실행됩니다. 우측의 **증분백업**을 체크하면 최초 백업 이후에는 변동 사항만 추가로 백업되는 증분 백업 방식이 적용됩니다. **관리자**로 선택한 경우에는 사용자가 PC에서 즉시백업을 실행하거나 백업을 예약할 수 없습니다.
> - **사용자**: 사용자가 PC에서 백업을 직접 실행하거나 예약할 수 있으며, 이 경우 PC백업 정책에서는 백업 설정을 할 수 없습니다.

* **백업 원본 경로**: 우측의 **추가** 버튼을 클릭하여 백업할 PC내 경로를 입력합니다. 복수의 경로를 입력할 경우 이 과정을 반복합니다. 이미 추가된 경로를 삭제하려면 삭제할 경로명 우측의 ![](<../.gitbook/assets/img_004 (90).png>)를 클릭합니다.

> - **‘추가’** 창에서 **백업 원본 경로** 입력란에 백업할 경로를 직접 입력하거나 **사용 가능한 매크로**를 선택하
> - 여 미리 설정된 경로를 선택할 수 있습니다.  ![](<../.gitbook/assets/img_005 (77).png>)를 클릭하면 매크로 내용을 확인할 수 있습니다.

<div align="left"><figure><img src="../.gitbook/assets/img_006 (65).png" alt=""><figcaption></figcaption></figure></div>

<div align="left"><figure><img src="../.gitbook/assets/img_007 (55).png" alt=""><figcaption></figcaption></figure></div>

> * 매크로를 선택할 경우 **백업 원본 경로** 입력란에 선택된 매크로(예: )가 표시되며, 필요 시 하위 경로를 덧붙여 입력할 수 있습니다.(예: /manual)
> * 하단의 **추가** 버튼을 클릭하면입력한 백업 원본 경로가 목록에 추가됩니다.

* **백업 목적지 경로**: 백업 파일을 저장할 위치를 개인문서함, 부서문서함 중에서 선택합니다. 설정된 백업 목적지 경로는 사용자가 변경할 수 없습니다.

> - **개인문서함**: 사용자의 개인문서함에 백업 파일들이 저장됩니다. 백업 파일이 실제 저장되는 경로는 다음과 같이 구성됩니다.

<div align="left"><figure><img src="../.gitbook/assets/img_008 (50).png" alt=""><figcaption></figcaption></figure></div>

> > ![](<../.gitbook/assets/img_009 (42).png>)  **{사용자 PC 구분}**: 관리자가 백업 경로에 PC이름이 포함되도록 설정한 경우에는 사용자 PC별로 폴더가 생성되어 백업 목적지 경로에 추가됩니다. 이 경우, 윈도우 PC에 대해서는 “W\_”, Mac PC에 대해서는 “M\_”을 PC명 앞에 덧붙인 폴더명이 사용됩니다. (예: 윈도우 PC명이 KIM일 경우 “W\_KIM”)
> >
> > ![](<../.gitbook/assets/img_010 (38).png>)  **{\*\*\*\*원본 파일의 PC 경로}**: 원본 파일의 PC 내 경로가 그대로 생성됩니다. (예: “고정 디스크(C) /Users/netid/Documents/Manual/”)

> * **부서문서함**: 백업 파일을 저장할 부서문서함 내 폴더 경로를 선택할 수 있습니다. 이미 **부서문서함**이 선택된 경우에는 우측의 **문서함/폴더 선택** 버튼을 클릭하면 ‘**폴더 선택**’ 창이 팝업됩니다. 백업 파일을 저장할 부서문서함과 해당 문서함 내 폴더를 선택한 후 **확인**을 클릭하면, **선택된 문서함/폴더** 영역에 해당 폴더 경로가 표시됩니다.

<div align="left"><figure><img src="../.gitbook/assets/img_011 (30).png" alt=""><figcaption></figcaption></figure></div>

> > 부서문서함 선택 시, 백업 파일이 실제 저장되는 경로는 다음과 같이 구성됩니다.

<div align="left"><figure><img src="../.gitbook/assets/img_012 (27).png" alt=""><figcaption></figcaption></figure></div>

> > ![](<../.gitbook/assets/img_013 (20).png>) **{선택한 문서함 경로}**: 선택한 부서문서함 내 폴더 경로
> >
> > ![](<../.gitbook/assets/img_014 (17).png>) **{PCBACKUP}**: 관리자가 백업 경로에 PC이름이 포함되도록 설정한 경우 “PCBACKUP” 폴더가 추가되며, 그 외의 경우에는 “PCBACKUP” 폴더 없이 경로가 생성됩니다.
> >
> > ![](<../.gitbook/assets/img_015 (16).png>) **{사용자 PC 구분}**: 관리자가 백업 경로에 PC이름이 포함되도록 설정한 경우에는 사용자 PC별로 폴더가 생성되어 백업 목적지 경로에 추가됩니다. 이 경우, 윈도우 PC에 대해서는 “W\_”, Mac PC에 대해서는 “M\_”을 PC명 앞에 덧붙인 폴더명이 사용됩니다. (예: 윈도우 PC명이 KIM일 경우 “W\_KIM”)
> >
> > ![](<../.gitbook/assets/img_016 (16).png>) **{원본 파일의 PC 경로}**: 원본 파일의 PC 내 경로가 그대로 생성됩니다. (예: “고정 디스크(C) /Users/netid/Documents/Manual/”)

**백업 경로에 PC이름 포함 여부**는 **설정홈 > 백업 모듈 설정 > 공통 백업**에서 설정할 수 있습니다.
{% endhint %}

* **백업 제외 폴더 목록**:

> - 기본적으로 가 포함됩니다. <의 내용은 **백업 제외 폴더 목록** 우측의 ![](<../.gitbook/assets/img_019 (11).png>) 를 클릭하여 확인할 수 있습니다.

<figure><img src="../.gitbook/assets/img_020 (8).png" alt=""><figcaption></figcaption></figure>

> * 우측의 **추가** 버튼을 클릭한 후, 하단의입력란에 백업 제외 폴더로 추가할 경로를 직접 입력하거나 **예약 폴더** 목록에서 미리 설정된 경로를 선택할 수 있습니다. 복수의 경로를 입력할 경우 이 과정을 반복합니다.

<figure><img src="../.gitbook/assets/img_021 (8).png" alt=""><figcaption></figcaption></figure>

> > * 예약 폴더를 선택할 경우 하단의입력란에 선택된 예약 폴더(예: )가 표시되며, 필요 시 하위 경로를 덧붙여 입력할 수 있습니다.(예: /mcloudoc)
> > * 하단의 **확인** 버튼을 클릭하면 입력한 백업 제외 폴더 경로가 목록에 추가됩니다.
>
> * 이미 추가된 폴더를 삭제하려면 삭제할 폴더 우측의 ![](<../.gitbook/assets/img_022 (8).png>)를 클릭합니다.

* **백업 확장자 설정**: 목록에서 백업할 확장자를 선택하거나, 백업에서 제외할 확장자를 선택할 수 있습니다. 목록에 없는 확장자가 필요할 경우, 기타 확장자 입력란에 확장자들을 세미콜론(;)으로 구분하여 입력합니다. 예) exe;dll

> - **모든 확장자 백업**: 목록에서 확장자를 선택할 필요 없이 모든 종류의 파일을 백업합니다.
> - **아래 확장자만 백업**: 목록에서 선택하거나 직접 입력한 확장자를 가진 파일만 백업합니다.
> - **아래 확장자만 백업 제외**: 목록에서 선택하거나 직접 입력한 확장자를 가진 파일은 백업에서 제외합니다.

* **백업 주기 설정**: 자동 백업이 한번 또는 주기별로 반복하여 수행되도록 예약할 수 있습니다.

> - **매 로그인 마다 백업 진행 여부**: 에이전트에서 로그인할 때마다 백업을 진행할 지 여부를 설정합니다.
> - **시작 시간**: 백업을 시작할 시간을 설정합니다.
> - **반복**: 백업 시작 시간으로부터 시간(분) 단위로 반복하여 백업이 수행되도록 설정할 수 있습니다. 시간(분) 단위 반복은 백업이 수행되는 날짜 내에서만 유효합니다. 예를 들어, 시작 시간을 오후3시, 반복을 6시간 마다, 예약 날짜를 매월 1일로 설정할 경우, 백업은 매월 1일 오후3시, 오후9시에 수행됩니다.
> - **예약 날짜**: 백업을 한 번 수행할 날짜를 지정하거나, 일, 주, 월 단위로 반복되도록 설정할 수 있습니다.

> > * **매일**: 매 n일마다 반복되거나, 평일에만 매일 백업이 수행되도록 설정할 수 있습니다.
> > * **매주**: 매 n주마다 특정 요일에 백업이 수행되도록 설정할 수 있습니다.
> > * **매월**: 매 n월마다 특정 일자, 또는 특정 요일(예: 넷째 주 일요일)에 백업이 수행되도록 설정할 수 있습니다.

4. 우측 하단의 **적용** 버튼을 클릭하여 설정 변경 사항을 서버에 저장합니다.

#### <mark style="color:$primary;">정책 적용하기</mark>

1. **백업 모듈 관리 - PC백업 - 정책 적용**을 클릭합니다.
2. **조직도**에서 정책을 적용할 부서 또는 사용자를 선택합니다.
3. 우측의**정책 목록**에서 적용할 정책을 선택하면, 해당 부서/사용자의 **설정된 정책**에 새로 선택된 정책이 표시됩니다.
4. 우측 상단의 **적용** 버튼을 클릭하여 변경 사항을 서버에 적용합니다.
