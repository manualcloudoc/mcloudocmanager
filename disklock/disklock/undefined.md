# 관리자가 파일수집 정책 관리하는 방법

## 파일수집 기능 소개

**파일수집**은 사용자의 로컬디스크에 남아 있는 중요 파일들을 수집하고, 수집한 파일들의 원본을 사용자가 로컬디스크에서 바로 열람할 수 없게 암호화함으로써, 사용자가 해당 파일들을 중앙문서함에 업로드하도록 유도하는 기능입니다.

**파일수집디스크**는 여러 경로에 흩어져 있는 수집 대상 파일들을 복사하여 한 데 모아 저장하는 영역으로,시스템 설정에 따라 파일수집 과정에서 파일수집디스크를 생성하여 사용자가 한시적으로 파일을 열람하며 편리하게 업로드 하도록 할 수 있습니다.

![Notes](<../../.gitbook/assets/img_000 (121).png>)파일수집 절차와 방법에 대한 상세한 내용은 **사용자 매뉴얼** [**파일수집 기능 소개**](https://github.com/manualcloudoc/mcloudoc-user-manual/blob/main/zoho-export/markdown/%EC%82%AC%EC%9A%A9%EC%9E%90-%EB%A7%A4%EB%89%B4%EC%96%BC/disklock/%ED%8C%8C%EC%9D%BC%EC%88%98%EC%A7%91-%EA%B8%B0%EB%8A%A5-%EC%86%8C%EA%B0%9C.md)를 참고하세요.

### 파일수집 옵션 설정

고객사에 적용될 파일수집의 구체적인 절차 및 방법은 시스템 설정에 따라 달라질 수 있습니다.

관리자는 시스템 설정을 통해 **파일수집 사용 여부, 파일수집디스크 생성 여부, 파일수집 강제 실행 사용 여부, 파일 정리 기능 사용 여부** 등 파일수집 옵션을 설정할 수 있습니다.

![Notes](<../../.gitbook/assets/img_001 (114).png>)시스템 설정 옵션의 상세한 내용과 설정 방법은 **Disklock 설정** 아티클 내 목차 \*\*[파일수집 정책](../../settings/pc/disklock.md)\*\*을 참고하세요.&#x20;

### 파일수집 정책

**파일수집의 대상이 되는 파일 종류, 파일수집에서 제외할 폴더** 등 파일수집의 세부적인 사항은 파일수집 정책에서 설정됩니다.

관리자는 사용자/부서별로 적용될 파일수집 정책을 생성하고 관리할 수 있습니다. 관리자가 설정할 파일수집 정책 항목은 다음과 같습니다.

|                                |                                                                                                                                                           |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **항목**                         | **내용**                                                                                                                                                    |
| **수집할 파일 확장자**                 | 파일 확장자를 사용하여 파일수집 대상이 되는 파일 종류를 지정합니다. 단, exe, dll 확장자는 사용할 수 없습니다.                                                                                       |
| **수집에서 제외할 폴더 목록**             | 파일수집 대상에서 제외할 폴더를 지정할 수 있습니다. Windows, Program Files, Program Files(x86) 폴더와 인터넷 브라우저의 캐시 폴더는 기본적으로 수집에서 제외할 폴더로 지정됩니다.                                   |
| **파일수집디스크 표시 기한**              | 파일수집디스크의 사용 기한을 지정합니다. 지정된 일자 이후에는 파일수집을 위한 가상디스크의 마운트가 해제되므로, 더 이상 파일수집디스크에 연결할 수 없게 됩니다.                                                                |
| **암호화된 원본 파일 저장위치**            | 암호화된 원본 파일을 저장할 위치를 지정합니다. 1. **원본 폴더**: 원본 파일의 경로에 저장. 2. **특\*\*\*\*정 \[수집] 폴더**: {수집} 폴더 하위에 원본 파일의 경로를 생성하여 저장                                        |
| **수집 원본 파일 정리 촉구 메시지 사용 여부**   | 사용자에게 암호화된 원본 파일 및 파일수집디스크의 삭제를 촉구하는 알림 메시지 사용 여부와 메시지 표시 일자를 지정합니다. 메시지 표시 일자는 **수집 디스크 보여줄 마지막 일자** 이후로 지정 가능하며, 지정된 날짜 이후부터 파일을 정리하기 전까지 촉구메시지가 표시됩니다. |
| **수집하지 않을 경우 중앙문서함 마운트 해제 일자** | **파일수집 강제 실행** 옵션 사용 시, 본 항목에서 지정된 날짜까지  파일수집을 하지 않을 경우 해당 사용자의 중앙문서함 이용이 제한됩니다.                                                                          |

## 파일수집 정책 관리하기

관리자는 다음과 같이 파일수집 정책을 설정할 수 있습니다.\
![](<../../.gitbook/assets/img_002 (98).png>)

1\.    **PC\*\*\*\*보안 모듈 관리 - DiskLock - 파일수집 - 정책 관리**를 클릭합니다.

2\.      **정책 목록**에서 추가 버튼 ![](<../../.gitbook/assets/img_003 (83).png>)  을 클릭하여 정책을 추가합니다. 기존 정책을 변경할 경우에는 정책 목록에서 변경할 정책을 선택합니다

1. 정책 추가 시 **‘추가’** 창에서 **정책명**(필수)과**정책 설명**을 입력합니다.

3\.     정책 항목 영역에 각 항목 값을 입력합니다.

1. **수집할 파일 확장자**: 업무용 파일 목록에서 수집할 파일을 선택하고, 그 외에 추가할 확장자들을 하단 텍스트 박스에 ‘,’로 구분하여 입력합니다.
2. **수집에서 제외할 폴더 목록**: 기본으로 표시된 폴더 목록 외에 추가할 폴더가 있는 경우 폴더 목록 우측 상단의 **추가** 버튼을 클릭한 후, 표시되는 예약 폴더 항목 중 제외할 폴더를 선택하거나 하단 텍스트박스에 폴더 경로를 직접 입력합니다.

> ![Idea](<../../.gitbook/assets/img_004 (64).png>)예약 폴더 항목의 일부 하위 폴더만 수집에서 제외하고자 할 경우, 예약 폴더 항목을 선택하고 하단 텍스트 박스에서 하위 폴더를 입력하여 덧붙일 수 있습니다. 예를 들어, 제외할 경로가  “C:\Users\netid\AppData\Roaming\Exception”일 경우, 예약 폴더에서  “C:\Users{username}\AppData\Roaming” 폴더에 해당하는   항목을 클릭하고 하단 텍스트 박스에  “\Exception”이 표시되도록 “\Exception”을 덧붙입니다.

1. **파일수집디스크 표시 기한**: 달력에서 지정할 날짜를 선택합니다.
2. **암호화된 원본 파일 저장위치**: 원본 폴더와 특정 \[수집] 폴더 중 지정할 위치를 선택합니다.
3. **수집 원본 파일 정리 촉구 메시지 사용 여부**: 해당 메시지 사용 여부를 선택하고, 메시지를 표시할 날짜를 달력에서 선택합니다.
4. **수집하지 않을 경우 중앙문서함 마운트 해제 일자**: 달력에서 지정할 날짜를 선택합니다.

> ![Notes](<../../.gitbook/assets/img_005 (54).png>)**수집하지 않을 경우 중앙문서함 마운트 해제 일자**는 시스템 설정에서 **파일수집 강제 실행 사용 여부**가 활성화된 경우에만 표시됩니다.

4\.    우측 하단의 **적용**버튼을 클릭하여 정책 항목 설정 사항을 해당 정책에 반영합니다.

## 사용자별 파일수집 현황 보기

각 사용자 PC의 파일수집 단계별 진행 현황은 서버에 저장됩니다. 관리자는 다음과 같은 방법으로 각 사용자의 파일수집 상태를 검색하여 조회하고, 검색 결과를 파일에 저장할 수 있습니다.![Notes](<../../.gitbook/assets/img_006 (46).png>)서버에는 시스템에서 실행된 파일수집 단계별 진행 현황만 저장되어 조회 가능하며, 사용자의 업로드 현황과 파일 정리 기능을 사용하지 않은 개별 삭제 현황은 포함되지 않습니다.\
1\.    **PC\*\*\*\*보안 모듈 관리 - DiskLock - 파일수집 - 현황 보기**를 클릭합니다.

2\.    **파일수집 현황** 페이지에서 조회를 원하는 **기간**과 파일수집 **상태**를 클릭하여 선택하고, **검색 조건**에 **사용자 이름** 또는 **ID**를 입력한 후 **검색** 버튼을 클릭합니다. 검색 조건을 입력하지 않을 경우 전체 사용자의 파일수집 현황이 표시됩니다.

3\.    해당 기간 내에 해당 사용자의 PC에서 파일수집을 시작한 건별로 **시작일시, 사용자 이름(ID), 부서명, 사용자 IP, 디바이스, 파일 수, 파일 크기, 수집 상태**가 표시됩니다.

4\.    검색 결과로 표시된 파일수집 건의 **로그** 항목 내 **보기** 버튼을 클릭하면 **파일수집 현황** 페이지 내에 ‘상세보기’ 창이 열리고, 해당 파일수집 건의 수집 시작 시점부터 최종 상태까지의 진행 단계별 로그가 표시됩니다.
