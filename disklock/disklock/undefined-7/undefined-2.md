# 수동으로 로컬저장금지 정책을 표준 정책 라이브러리에 취합하기

고객사의 문서중앙화 서버와 정책 관리 서버가 연결되어 있으면, 고객사의 로컬저장금지 정책에 새로 추가되거나 변경된 정책 아이템은 자동으로 취합되어 심사 과정을 거쳐 정책 관리 서버의 표준 정책 라이브러리에 반영됩니다.하지만, 두 서버 간의 통신 문제가 발생하여 이러한 자동 취합이 수행되기 어렵거나 자동 취합이 수행되기 전에 미리 로컬저장금지 정책의 변경 사항을 취합해야 하는 경우에는 엠클라우독 담당자에게 수동 취합을 요청할 수 있습니다.이 때, 정책 아이템은 ZIP 형식의 압축 파일로 저장하여 엠클라우독 담당자에 전달해야 합니다. 담당자는 전달받은 정책 아이템을 심사한 후, 심사를 통과한 정책 아이템을 정책 관리 서버의 표준 정책 라이브러리에 추가합니다수동 취합 요청 시, 담당자에게 전달할 로컬저장금지 정책 설정 정보는 다음과 같은 방법을 사용하여 ZIP 형식의 압축 파일로 저장합니다.​\
1\.    웹에서 **PC 보안 모듈 관리**의 **DiskLock** – **로컬저장금지** – **정책 라이브러리** – **고객사 정책 내보내기** 메뉴를 차례로 클릭합니다. \
![](<../../../.gitbook/assets/img_000 (129).png>)2.    ‘**고객사 정책 내보내기**’ 화면이 나타납니다. 화면에 있는 **내보내기**를 클릭합니다.![](<../../../.gitbook/assets/img_001 (121).png>) \
3\.    정책 아이템을 파일로 생성하고 압축하는 과정이 진행됩니다. 정책 아이템 압축 파일이 성공적으로 생성되면 다음과 같은 팝업 창이 나타납니다. **확인**을 클릭합니다. \
![](<../../../.gitbook/assets/img_002 (104).png>)4.    생성된 정책 아이템 압축 파일의 이름은 기본적으로 **policy\_items\_\[고객사의 사이트ID]\_년월일초.zip**으로 정해지고 **다운로드** 폴더에 저장됩니다. 다운로드 폴더에 저장이 완료되면 파일의 이름을 보여주는 다음과 같은 창이 나타납니다![](<../../../.gitbook/assets/img_003 (89).png>)5.    해당 압축 파일을 엠클라우독 담당자에게 전달하여 정책 관리 서버의 표준 정책 라이브러리에 반영하도록 요청합니다.
