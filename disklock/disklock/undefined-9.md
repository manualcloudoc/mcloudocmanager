# 웹 반출 예외 정책 관리하기

웹 반출 예외 정책은 인터넷 익스플로러(Internet Explorer), 크롬(Chrome) 등 웹 브라우저 사용 시 로컬저장금지 정책 및 클립보드 복사 제어 정책에 예외적으로 파일 첨부, ActiveX 설치, 또는 클립보드 붙여넣기를 허용하기 위한 정책입니다.

각 정책은 순서가 부여된 정책 아이템의 목록으로 구성하며, 정책 아이템으로 예외가 적용될 웹사이트 정보를 등록하고, 해당 웹사이트에서의 파일 첨부, ActiveX 설치, 클립보드 붙여넣기의 허용 여부 등을 설정합니다. 이때 파일의 종류와 위치에 따라 첨부를 제한할 수 있습니다. &#x20;

![](<../../.gitbook/assets/img_000 (133).png>)

웹 반출 예외 정책을 설정하는 방법은 다음과 같습니다.\
​\
1\.    **PC\*\*\*\*보안 모듈 관리 - DiskLock - 웹 반출 예외 정책 - 정책 관리**를 클릭합니다.

2\.    **정책 목록**에서 추가 버튼 ![](<../../.gitbook/assets/img_001 (125).png>) 을 클릭하여 정책을 추가합니다. 기존 정책을 변경할 경우에는 정책 목록에서 변경할 정책을 선택합니다.

1. 정책 추가 시 **‘추가’** 창에서 **정책명**(필수)과**정책 설명**을 입력합니다.

3\.    페이지 우측 상단의 **추가** 버튼을 클릭하여 정책 아이템을 추가하거나, 정책 아이템 목록 우측의 **연필** 아이콘을 클릭하여 기존의 정책 아이템을 변경할 수 있습니다.

4\.    ‘정책 아이템 추가’ 창에서 다음과 같은 항목을 설정합니다.

1. **정책 아이템 활성화**: 해당 정책 아이템을 활성화 또는 비활성화할 수 있습니다.
2. **웹 사이트 경로**: 파일 첨부 등을 예외적으로 허용할 웹사이트의 경로를 지정합니다.  URL 주소에서 “http://” 를 제외한 나머지 주소를 사용하여 입력하며, 별표(\*)를 사용하여 불특정 문자열을 대신할 수 있습니다.  (예: **\*.google.com**)
3. **웹 사이트 제목**: 해당 웹사이트에 표시되는 제목을 동일하게 입력하거나, 생략할 수 있습니다. 제목을 입력할 경우, 반드시 웹사이트의 제목이 일치할 경우에만 설정된 정책이 적용됩니다.
4. **첨부 허용**: 해당 웹사이트에 대해 파일 첨부를 허용할 경우 체크합니다. 이때 첨부를 허용/ 거부할 파일 종류와 첨부가 가능한 디스크 범위를 지정할 수 있습니다.

> 1) **첨부를 허용/거부할 파일 종류**:  지정된 파일 종류에 대해서만 첨부를 허용하도록 설정하거나, 지정된 파일 종류를 제외한 나머지 모든 파일에 대해 첨부를 허용하도록 설정할 수 있습니다. “\*.pdf”와 같이 확장자를 지정할 수 있으며, 복수 입력 시 콤마(,)로 구분합니다.

> 4. **첨부 가능한 디스크 범위 선택**: 파일 첨부를 허용할 디스크 종류를 제한할 수 있습니다.

1. **ActiveX 설치 허용**: 해당 웹사이트에서 ActiveX 설치를 허용할 경우 체크합니다. 본 항목은 인터넷 익스플로러 사용 시에만 적용됩니다.
2. **클립보드 붙여넣기**: 해당 웹사이트에서 클립보드 내용 붙여넣기를 허용할 경우 체크합니다.

5\.    **확인** 버튼을 클릭하면 정책 아이템이 추가(또는 변경)됩니다.

6\.    정책 아이템 목록에서 **순서** 항목을 상/하로 드래그하여 순서를 변경할 수 있습니다.  아이템의 순서가 앞에 있을수록(숫자가 작을수록) 우선 순위를 갖습니다.

7\.    정책 아이템 목록 하단의 **적용** 버튼을 클릭하여 정책 변경 사항을 서버에 적용합니다.
