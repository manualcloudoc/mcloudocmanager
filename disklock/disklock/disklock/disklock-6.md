# DiskLock 콘솔에서 탐색기 복사/이동 정책 관리하기

## 탐색기 복사/이동 정책 소개

DiskLock의 로컬저장금지 정책에서는 애플리케이션별로 파일 입출력 권한(읽기, 쓰기, 목록 보기, 삭제)을 다르게 지정하기 위해 애플리케이션을 카테고리로 분류합니다. 애플리케이션 카테고리 중 윈도우 탐색기와 같은 파일 관리자 프로그램이 주로 분류되는 **FullAccess Applications** 카테고리는 기본적으로 **모든 디스크**에 대해 **모든 권한**이 허용됩니다.\
FullAccess Applications 카테고리로 분류되는 다른 프로그램과 달리,**윈도우 탐색기**는 디스크 간의 파일 복사 및 이동이 가능한 애플리케이션입니다. 따라서, 윈도우 탐색기에 모든 권한이 주어져 있는 상황에서는 윈도우 탐색기를 통해 중앙 서버에 저장된 주요 정보가 외부로 유출될 위험이 있습니다. 이러한 정보 유출 위험을 막기 위해 DiskLock에서는 윈도우 탐색기에서 복사와 이동이 가능한 디스크를 제한할 수 있는 **탐색기 복사/이동 정책**을 제공합니다.

탐색기 복사/이동 정책을 설정할 때에는 다음 8가지 종류의 디스크 간에 파일 복사/이동을 허용할지 금지할지 여부를 지정합니다.

|           |                                                             |                                  |        |
| --------- | ----------------------------------------------------------- | -------------------------------- | ------ |
| **위치**    | **디스크 종류**                                                  | **기능**                           | **구역** |
| **로컬 PC** | 고정디스크                                                       | 사용자 PC의 하드 디스크                   | 비 안 전  |
| CD-ROM    | 사용자 PC의 CD-ROM                                              |                                  |        |
| 네트워크      | 사용자 PC에서 네트워크로 연결된 디스크                                      |                                  |        |
| 이동식       | 사용자 PC의 이동식 디스크(USB 디스크 등)                                  |                                  |        |
| 반출보안      | 외부로 반출이 승인된 문서를 저장하기 위해 임시로 만들어지는 보안 디스크                    | 안 전                              |        |
| 온라인보안     | 네트워크 부하 등의 이유로 중앙문서함 대신 로컬 디스크에서 작업하고자 할 때 임시로 만들어지는 보안 디스크 |                                  |        |
| 오프라인보안    | 문서중앙화 솔루션의 사용 중, 네트워크가 끊어졌을 때 복구되기까지 임시로 만들어지는 보안 디스크       |                                  |        |
| **중앙 서버** | 중앙문서함                                                       | 문서중앙화 서버의 개인 문서함, 부서 문서함, 공유 문서함 |        |

![Notes](<../../../.gitbook/assets/img_000 (120).png>)안전구역의 디스크들은 DiskLock의 보안 기능을 통해 보호됩니다. 문서 중앙화 솔루션에서 지원하는 보안 디스크의 종류와 기능에 대한 상세 설명은 **사용자 매뉴얼** - \*\*[보안 디스크 소개](https://github.com/manualcloudoc/mcloudoc-user-manual/blob/main/zoho-export/markdown/%EC%82%AC%EC%9A%A9%EC%9E%90-%EB%A7%A4%EB%89%B4%EC%96%BC/disklock/%EB%B3%B4%EC%95%88%EB%94%94%EC%8A%A4%ED%81%AC-%EC%86%8C%EA%B0%9C.md)\*\*를 참고합니다.\
예를 들어, **문서중앙화 서버에 저장된 문서 유출을 막기 위해** 다음과 같은 탐색기 복사/이동 정책을 설정하여 사용할 수 있습니다.

1. 중앙문서함에서 다른 디스크로 복사/이동: **차단**
2. 다른 디스크 간의 복사/이동: **모두 허용**\

## 탐색기 복사/이동 정책 설정하기

### 탐색기 복사/이동 정책 관리 화면 살펴보기

DiskLock 콘솔 화면의 메뉴에서 **탐색기 복사/이동 정책** – **정책 관리** 메뉴를 선택하면 탐색기 복사/이동 정책을 설정하는 화면이 표시됩니다. 화면은 **탐색기 복사/이동 정책 목록**과 **도구모음**으로 구성되어 있습니다. 화면 좌측에 있는 탐색기 복사/이동 정책 목록에서 하나의 정책을 클릭하면 화면 우측에 정책의 설정을 보여주는 표가 다음과 같이 나타납니다. 화면의 각 부분에 대해 알아봅니다.

![](<../../../.gitbook/assets/img_002 (97).png>)

![](<../../../.gitbook/assets/img_003 (82).png>) **도구모음**화면 상단에 있는 도구모음은 탐색기 복사/이동 정책을 관리하는 데 사용되는 메뉴들이 아이콘 형태로 제공됩니다. 도구모음의 아이콘은 상황에 따라 사용 가능한 아이콘만 활성화됩니다. 각 아이콘의 기능은 다음과 같습니다.

|          |                                                                                                   |
| -------- | ------------------------------------------------------------------------------------------------- |
| **아이콘**  | **기능**                                                                                            |
| **새 정책** | 새로운 탐색기 복사/이동 정책을 생성합니다.                                                                          |
| **삭제**   | 선택한 정책을 삭제합니다. 부서나 사용자에게 적용되어 사용 중인 정책은 삭제할 수 없습니다.                                               |
| **이름변경** | 선택한 정책의 이름을 변경합니다.                                                                                |
| **사본생성** | 선택한 정책의 복사본을 생성합니다. 새로 생성하려는 정책이 기존 정책과 유사할 경우 복사본을 사용하면 모든 디스크의 정책을 설정할 필요 없이 새로운 정책을 만들 수 있습니다. |
| **적용**   | 부서나 사용자에 적용한 보안디스크 정책에 대한 설정 정보를 서버에 저장합니다.                                                       |

![](<../../../.gitbook/assets/img_004 (63).png>) **탐색기 복사**\*\*/\*\***이동 정책 목록**\
현재 정의된 탐색기 복사/이동 정책들을 보여주는 부분입니다. 정책은 가나다 순으로 정렬되어 표시됩니다. 정책 이름을 클릭하면 해당 정책에 설정된 디스크 간의 복사/이동 허용 여부가 화면의 오른쪽에 표시됩니다.&#x20;

![](<../../../.gitbook/assets/img_005 (53).png>) **정책 설정 표**\
정책 목록에서 선택한 정책의 현재 설정을 표 형식으로 보여주는 부분입니다. 표의 왼쪽에 표시된 디스크들은 복사나 이동할 원본 파일이 저장된 \*\*원본 디스크(source)\*\*이고 오른쪽 상단의 디스크들은 원본 파일이 복사되는 \*\*대상 디스크(destination)\*\*입니다. 원본 디스크에서 대상 디스크로의 파일 복사/이동이 허용된 경우에는 **허용** 아이콘 ![](<../../../.gitbook/assets/img_006 (45).png>)이, 차단되어 있는 경우에는 **금지** 아이콘 ![](<../../../.gitbook/assets/img_007 (38).png>)이 표시됩니다.

![](<../../../.gitbook/assets/img_008 (35).png>)\
&#x20;\
![](<../../../.gitbook/assets/img_009 (31).png>) **버튼**\
이전 설정으로 되돌리거나 모든 디스크 간 복사/이동의 허용/차단 여부를 동일하게 변경하는 버튼입니다.

|           |                                 |
| --------- | ------------------------------- |
| **버튼**    | **기능**                          |
| **원래대로**  | 변경한 설정을 무시하고 이전에 저장된 상태로 되돌립니다. |
| **모두 허용** | 모든 디스크 간의 복사/이동을 허용하도록 설정합니다.   |
| **모두 차단** | 모든 디스크 간의 복사/이동을 차단하도록 설정합니다.   |

![](<../../../.gitbook/assets/img_010 (27).png>)  **컨텍스트 메뉴**\
화면에 나타나 있지 않지만 정책 목록에서 마우스를 우클릭하면 현재 사용할 수 있는 메뉴들을 보여주는 데 이를 컨텍스트 메뉴라고 합니다. 각 메뉴의 기능은  도구모음에 있는 같은 이름의 메뉴와 기능이 같습니다.

![](<../../../.gitbook/assets/img_011 (20).png>)

### 탐색기 복사/이동 정책 생성하기

새로운 탐색기 복사/이동 정책을 생성하는 방법은 다음과 같습니다.

1\.   정책 관리 화면의 상단 도구모음에서 **새 정책**을 클릭하거나 화면 왼쪽의 정책 목록에서 마우스를 우클릭하여 나오는 컨텍스트 메뉴에서 **새 정책**을 클릭합니다.

2\.   정책 목록의 가장 아래에 다음과 같이 **<이름없음>** 새 정책이 만들어집니다.

![](<../../../.gitbook/assets/img_012 (18).png>)

3\.  정책의 이름을 입력하고 Enter키를 누릅니다.

![](<../../../.gitbook/assets/img_013 (11).png>)

4\.   탐색기 복사/이동 정책은 기본적으로 탐색기에서 아래 3가지 경우를 제외하고 나머지 디스크 간의 파일 복사/이동이 차단되도록 설정되어 있습니다. 이러한 기본 설정을 변경하려면 \*\*[탐색기 복사/이동 정책 변경하기](disklock-6.md)\*\*의 설명을 참고하여 원하는 설정으로 변경합니다.

1. 같은 디스크 간에 파일 복사/이동
2. 모든 디스크에서 중앙 문서함으로 복사/이동
3. 중앙 문서함에서 온라인 보안 디스크로 복사/이동

### 탐색기 복사/이동 정책 변경하기

기존의 탐색기 복사/이동 정책을 변경하는 방법은 다음과 같습니다.

1\.   정책 목록에서 변경할 탐색기 복사/이동 정책을 클릭합니다.

2\.   선택한 탐색기 복사/이동 정책의 현재 설정 정보가 오른쪽에 표시됩니다. 원본 파일이 저장된 디스크와 복사/이동할 대상 디스크 별로, 해당 위치의 **허용/차단** 아이콘을 클릭하여 **허용** 또는 **차단** 여부를 설정합니다. 아이콘을 클릭할 때마다 허용 아이콘 ![](<../../../.gitbook/assets/img_015 (9).png>)으로, 차단 아이콘 ![](<../../../.gitbook/assets/img_016 (9).png>)으로, 차단 아이콘 ![](<../../../.gitbook/assets/img_017 (6).png>)이 허용 아이콘 ![](<../../../.gitbook/assets/img_018 (6).png>)으로 바뀝니다.

![](<../../../.gitbook/assets/img_019 (6).png>)

![Idea](<../../../.gitbook/assets/img_020 (4).png>)모든 디스크 간의 파일 복사/이동을 허용하려면 **모두 허용** 버튼을, 모두 금지하려면 **모두 차단** 버튼을 사용하면 편리합니다. 변경 전의 설정으로 되돌리려면 **원래대로** 버튼을 클릭합니다.\
3\.   변경된 설정에는 다음 그림 같이 빨간색 점이 표시됩니다. 화면 오른쪽 상단에 있는 적용 버튼을 클릭하여 변경된 정책을 서버에 저장합니다.

![](<../../../.gitbook/assets/img_021 (4).png>)

4\.   적용 여부를 묻는 팝업 창이 나타나면 **예**를 클릭합니다.\
&#x20;\
5\.   변경된 정책이 서버에 정상적으로 저장되면 허용, 차단 아이콘 위에 표시되었던 붉은 점이 사라집니다.

### 탐색기 복사/이동 정책 이름 변경하기

탐색기 복사/이동 정책의 이름을 변경하는 방법은 다음과 같습니다.\
1\.   화면 왼쪽의 탐색기 복사/이동 정책 목록에서 이름을 바꿀 정책을 클릭한 후 정책 목록 상단의 도구모음에서 **이름변경**을 클릭하거나 정책 목록에서 정책을 마우스 우클릭한 후 컨텍스트 메뉴에서 **이름변경**을 클릭합니다.

![](<../../../.gitbook/assets/img_022 (4).png>)\
&#x20;2\.    선택한 정책의 이름이 수정 가능한 상태로 바뀝니다. 새로운 정책 이름을 입력한 후 Enter 키를 누릅니다.

![](<../../../.gitbook/assets/img_023 (5).png>)

### 탐색기 복사/이동 정책 사본 생성하기

기존 탐색기 복사/이동 정책의 복사본을 만드는 방법은 다음과 같습니다.\
1\.   화면 왼쪽의 탐색기 복사/이동 정책 목록에서 복사본을 생성할 정책을 클릭한 후 정책 목록 상단의 도구모음에서 **사본생성**을 클릭하거나 정책 목록에서 사본을 생성할 정책을 마우스 우클릭한 후 컨텍스트 메뉴에서 **사본생성**을 클릭합니다.

![](<../../../.gitbook/assets/img_024 (6).png>)

2\.   정책 목록의 가장 아래에 선택한 정책 이름 뒤에 **-복사본**이 붙여진 새로운 정책이 만들어집니다. 적절한 이름을 입력한 후 Enter를 누릅니다.

![](<../../../.gitbook/assets/img_025 (2).png>)

3\.   정책을 복사한 후에는\*\*[탐색기 복사/이동 정책 생성하기](disklock-6.md)\*\*의 설명을 참고하여 기존 정책의 설정을 변경하는 작업을 수행한 후 서버에 적용하도록 합니다.

### 탐색기 복사/이동 정책 삭제하기

탐색기 복사/이동 정책을 삭제하는 방법은 다음과 같습니다.

1\.   화면 왼쪽의 탐색기 복사/이동 정책 목록에서 삭제할 정책을 클릭한 후 정책 목록 상단의 도구모음에서 **삭제**를 클릭하거나 정책 목록에서 복사할 정책을 마우스 우클릭한 후 컨텍스트 메뉴에서 **삭제**를 클릭합니다.\
&#x20;![](<../../../.gitbook/assets/img_026 (2).png>)\
2\.    정책의 삭제 여부를 확인하는 팝업 창이 나타나면 **예**를 클릭합니다.&#x20;

## 탐색기 복사/이동 정책 적용하기

문서중앙화 솔루션에서 모든 보안 기능들(DiskLock, DiskLock Plus, 외부 DRM 반출 정책 등)을 통합된 정책을 통해 관리하는 **통합 정책**을 사용하지 않는 경우에는 DiskLock 콘솔에서 사용자나 부서에 적용할 탐색기 복사/이동 정책을 선택할 수 있습니다.&#x20;

![Notes](<../../../.gitbook/assets/img_027 (2).png>)통합 정책을 사용하는 경우에는 DiskLock 콘솔 화면 상단에 다음과 같은 메시지가 표시되고, **탐색기 복사/이동** **정책** 메뉴에 **정책 적용** 메뉴가 제공되지 않습니다.\
![](<../../../.gitbook/assets/img_028 (1).png>)

### 탐색기 복사/이동 정책 적용 화면 살펴보기

DiskLock 콘솔 화면의 메뉴에서 **탐색기 복사/이동 정책** – **정책 적용** 메뉴를 선택하면 다음과 같이 탐색기 복사/이동 정책을 적용하는 화면이 표시됩니다. 화면은 다음과 같이 ![](<../../../.gitbook/assets/img_029 (1).png>)**도구모음**과 ![](<../../../.gitbook/assets/img_030 (1).png>) **조직도**, ![](<../../../.gitbook/assets/img_031 (1).png>)**탐색기 복사/이동 정책 목록**으로 구성되어 있습니다.

![](<../../../.gitbook/assets/img_032 (1).png>)

![](<../../../.gitbook/assets/img_033 (1).png>) **도구모음**\
화면 상단의 도구모음은 탐색기 복사/이동 정책을 적용하는데 필요한 메뉴들이 아이콘 형태로 제공됩니다. 도구모음의 아이콘은 상황에 따라 사용 가능한 아이콘만 활성화됩니다. 각 아이콘의 기능은 다음과 같습니다.

|           |                                                            |
| --------- | ---------------------------------------------------------- |
| **아이콘**   | **기능**                                                     |
| **새로고침**  | 서버로부터 최신 정보를 가져와 화면의 조직도와 탐색기 복사/이동 정책 목록을 최신 정보로 업데이트합니다. |
| **사용자검색** | 사용자의 이름이나 ID, 혹은 부서 이름을 사용하여 조직도에서 부서나 사용자를 검색합니다.         |
| **적용**    | 부서나 사용자에 적용한 탐색기 복사/이동 정책에 대한 설정 정보를 서버에 저장합니다.            |

![](<../../../.gitbook/assets/img_034 (1).png>) **조직도**조직도 상에서 각 부서와 사용자에게 적용된 탐색기 복사/이동 정책을 보여주는 부분입니다. 부서에 설정된 정책은 하위 부서와 부서에 속한 모든 사용자에게 동일하게 적용됩니다. 사용자나 하위 부서에 다른 정책을 설정한 경우에는 상위 부서에 설정된 정책은 무시되고 사용자나 하위 부서에 설정된 정책이 사용됩니다. ![Notes](<../../../.gitbook/assets/img_035 (1).png>)상위 부서에 설정된 정책이 하위 부서에 적용되거나 부서에 설정된 정책이 부서에 속한 모든 사용자에게 자동으로 적용되는 것을 **정책 상속**이라고 합니다\
![](<../../../.gitbook/assets/img_036 (1).png>) **탐색기 복사/이동 정책 목록**\
현재 정의된 탐색기 복사/이동 정책의 목록입니다. 왼쪽 조직도에서 부서나 사용자를 클릭하면 해당 부서나 사용자에 현재 적용된 정책이 목록에서 선택됩니다. 조직도에서 클릭한 부서나 사용자에 탐색기 복사/이동 정책이 적용되어 있지 않거나 상위 부서의 정책을 상속받아 사용하는 경우에는 \*\*지정하지 않음(정책제거, 상속)\*\*이 선택됩니다.

### 부서/사용자 검색하기

조직도에서 탐색기 복사/이동 정책을 적용할 부서나 사용자를 찾기 어려운 경우에는 다음과 같은 방법을 사용하여 부서나 사용자를 쉽게 검색할 수 있습니다.

1\.   DiskLock 콘솔 화면의 메뉴에서 **탐색기 복사/이동** **정책** – **정책 적용** 메뉴를 클릭합니다.&#x20;

2\.   도구모음에서 **사용자검색**을 클릭합니다.

![](<../../../.gitbook/assets/img_037 (2).png>)

3\.   다음과 같은 **사용자 및 부서 검색** 창이 나타납니다. 검색 창의 입력란에 검색할 사용자 ID나 사용자 이름, 혹은 부서 이름을 입력한 후 Enter를 누릅니다. 사용자의 ID나 이름의 일부만 입력하는 것도 가능합니다.

![](<../../../.gitbook/assets/img_038 (2).png>)

4\.   입력한 문자열이 포함된 사용자와 부서의 목록이 화면에 출력됩니다. 목록에서 정책을 설정할 부서나 사용자를 선택한 후 **위치로 이동** 버튼을 클릭합니다.

![](<../../../.gitbook/assets/img_039 (2).png>)

5\.    **사용자 및 부서 검색** 창이 닫힙니다. 그리고, 탐색기 복사/이동 정책 관리 화면의 조직도에 검색한 부서 혹은 사용자가 선택되고 정책 목록에서는 검색한 부서나 사용자에게 현재 설정된 탐색기 복사/이동 정책이 선택됩니다.

![](<../../../.gitbook/assets/img_040 (1).png>)

6\.   선택한 부서나 사용자의 탐색기 복사/이동 정책을 변경하려면 이어지는 **부서/사용자별 탐색기 복사/이동 정책 적용하기**의 내용을 참고합니다.

### 부서/사용자별 탐색기 복사/이동 정책 적용하기

탐색기 복사/이동 정책은부서 및 사용자 단위로 지정할 수 있습니다.부서나 사용자에 적용할 탐색기 복사/이동 정책을 지정하는 방법은 다음과 같습니다.

1\.    DiskLock 콘솔 화면의 메뉴에서 **탐색기 복사/이동** **정책** – **정책 적용** 메뉴를 클릭합니다.&#x20;

2\.    부서/사용자 조직도에서 정책을 지정할 부서나 사용자를 선택합니다. 트리 형태로 된 조직도에서 \*\*+\*\*를 클릭하면 하위 부서를 볼 수 있고 최하위 부서의 경우에는 부서에 속한 사용자를 볼 수 있습니다. 부서나 사용자를 선택하면 선택한 사용자에게 현재 적용된 탐색기 복사/이동 정책이 오른쪽 정책 목록에서 선택됩니다. \*\*지정하지 않음(정책제거, 상속)\*\*이선택된 경우에는탐색기 복사/이동 정책을 사용하지 않거나 상위 부서의 정책을 상속 받아 사용하는 부서나 사용자입니다.\
![](<../../../.gitbook/assets/img_041 (1).png>)

3\.   사용자가 속한 부서를 모르거나 부서를 찾기 어려운 경우에는 **부서/사용자 검색하기** 내용을 참고하여 부서나 사용자를 검색합니다.

4\.   우측의 정책 목록에서 선택한 부서나 사용자에게 적용할 탐색기 복사/이동 정책을 클릭합니다. 현재 적용된 정책을 더 이상 적용하지 않거나 상위 부서의 정책을 그대로 사용하려는 경우에는 \*\*지정하지 않음(정책제거, 상속)\*\*을 선택합니다. 정책을 변경하면 조직도의 **정책이름**에 설정이 변경되었음을 알려주는 붉은 점이 표시됩니다.

![](<../../../.gitbook/assets/img_042 (1).png>)

![Idea](<../../../.gitbook/assets/img_043 (1).png>)탐색기 복사/이동 정책 목록에서 각각의 정책 위에 마우스를 위치시키면 해당 정책의 설정을 보여주는 창이 나타납니다.![](<../../../.gitbook/assets/img_044 (1).png>)\
5\.   우측 상단 도구모음에 있는 **적용**을 클릭하여 변경한 정책 설정을 서버에 적용합니다.

![](<../../../.gitbook/assets/img_045 (1).png>)\
&#x20;\
6\.   적용 여부를 묻는 팝업 창이 나타나면 **확인**을 클릭합니다. 부서나 사용자에게 탐색기 복사/이동 정책이 성공적으로 적용되면 조직도의 정책 이름 옆에 있던 붉은 점이 사라집니다.
