# DiskLock 콘솔에서 보안디스크 정책 관리하기

문서 중앙화 솔루션은 부서나 사용자별로 사용할 보안디스크와 보안디스크 용량을 다르게 지정할 수 있습니다. 이를 위해 각 보안디스크의 **사용 여부**와 부서나 사용자에게 할당할 **보안디스크의 용량**이 정의된 **보안디스크 정책**을 사용합니다. 부서나 사용자에 적용할 보안디스크 정책을 지정하는 작업은 여기에서 설명하는 DiskLock 콘솔의 **보안디스크 정책 관리 메뉴**를 사용하여 수행할 수 있습니다.

{% hint style="warning" icon="square-poll-horizontal" %}
보안 디스크의 종류와 기능에 대한 설명은 [**보안디스크 소개**](https://app.gitbook.com/s/vQ0BiQsGY4PT08D7Nfay/disklock/undefined-2)를 참고합니다.&#x20;
{% endhint %}

### <mark style="color:$primary;">보안디스크 정책 관리 화면 살펴보기</mark>

DiskLock 콘솔 화면의 메뉴에서 **보안디스크 정책 – 정책 관리** 메뉴를 선택하면 다음과 같이 보안디스크 정책을 설정하는 화면이 표시됩니다. 화면은 <img src="../../../.gitbook/assets/image (161).png" alt="" data-size="line">**도구모음**과 <img src="../../../.gitbook/assets/image (160).png" alt="" data-size="line">**부서/사용자 조직도**, <img src="../../../.gitbook/assets/image (164).png" alt="" data-size="line">**보안디스크 정책**으로 구성되어 있습니다. <br>

<figure><img src="../../../.gitbook/assets/img_004 (60).png" alt=""><figcaption></figcaption></figure>

#### <img src="../../../.gitbook/assets/image (158).png" alt="" data-size="line"> **도구 모음**

화면 상단의 도구 모음은 보안디스크 정책을 적용하는데 필요한 메뉴들이 아이콘 형태로 제공됩니다. 도구 모음의 아이콘은 상황에 따라 사용 가능한 아이콘만 활성화됩니다. 각 아이콘의 기능은 다음과 같습니다.

<table><thead><tr><th width="146">아이콘</th><th>기능</th></tr></thead><tbody><tr><td><strong>새로고침</strong></td><td>서버로부터 최신 정보를 가져와 화면의 조직도와 보안디스크 설정을 최신 정보로 업데이트합니다.</td></tr><tr><td><strong>사용자검색</strong></td><td>사용자의 이름이나 ID, 혹은 부서 이름을 사용하여 조직도에서 부서나 사용자를 검색합니다.</td></tr><tr><td><strong>적용</strong></td><td>부서나 사용자에 적용한 보안디스크 정책에 대한 설정 정보를 서버에 저장합니다.</td></tr></tbody></table>

#### <img src="../../../.gitbook/assets/image (163).png" alt="" data-size="line"> **부서/사용자 조직도**

조직도 상에서 각 부서와 사용자에게 할당된 보안디스크의 용량을 보여주는 부분입니다. 부서나 사용자의 오른쪽에 표시된 숫자가 할당된 보안디스크의 용량(MB 혹은 GB)입니다. 사용하지 않는 보안디스크는 ‘0(B)’으로 표시됩니다. \
부서에 적용된 보안디스크 정책은 하위 부서와 부서에 속한 모든 사용자에게 동일하게 적용됩니다. 하지만, 사용자나 하위 부서에 다른 정책이 설정되어 있으면 상위 부서에 설정된 정책은 무시되고 사용자나 하위 부서에 설정된 정책이 적용됩니다.

{% hint style="warning" icon="square-poll-horizontal" %}
상위 부서에 설정된 정책이 하위 부서에 적용되거나 부서에 설정된 정책이 부서에 속한 모든 사용자에게 자동으로 적용되는 것을 **정책 상속**이라고 합니다.
{% endhint %}

{% hint style="warning" icon="square-poll-horizontal" %}
온라인, 오프라인, 반출 보안디스크 중 시스템 설정에서 사용이 허용된 보안디스크에 대해서만 설정이 가능합니다. 임시폴더 보안디스크의 경우, 임시폴더 보호 정책 사용 시에만 설정할 수 있습니다.
{% endhint %}

#### <img src="../../../.gitbook/assets/image (162).png" alt="" data-size="line"> **보안디스크 정책**

왼쪽의 조직도에서 선택한 부서나 사용자에 적용된 보안디스크 정책을 보여주는 부분입니다. 조직도에서 클릭한 부서나 사용자에서 보안디스크 정책을 설정하지 않거나 상위 부서의 정책을 상속받아 사용하는 경우에는 **지정하지 않음(정책제거, 상속**) 항목이 선택됩니다.

### <mark style="color:$primary;">보안디스크 정책을 적용할 부서/사용자 검색하기</mark>

보안디스크 정책을 조회하거나 적용할 사용자나 부서를 찾기 어려운 경우에는 다음과 같은 방법을 사용하여 부서나 사용자를 검색할 수 있습니다.

1. DiskLock 콘솔 화면의 메뉴에서 **보안디스크 정책** – **정책 관리** 메뉴를 클릭합니다. &#x20;
2. 도구모음에서 **사용자검색**을 클릭합니다.&#x20;
3. 다음과 같은 **사용자 및 부서** **검색** 창이 나타납니다. 검색 창의 입력란에 검색할 **사용자** **ID**나 **사용자 이름**, 혹은 **부서 이름**을 입력한 후 Enter를 누릅니다. ID나 이름의 일부만 입력하는 것도 가능합니다.

<div align="left"><img src="../../../.gitbook/assets/img_010 (24).png" alt="" width="375"></div>

4. 입력한 문자열이 포함된 사용자와 부서의 목록이 화면에 출력됩니다. 목록에서 정책을 설정할 부서나 사용자를 선택한 후 **위치로 이동** 버튼을 클릭합니다.

<div align="left"><figure><img src="../../../.gitbook/assets/img_011 (17).png" alt="" width="563"><figcaption></figcaption></figure></div>

5. ‘**사용자 및 부서 검색’** 창이 닫히고 보안디스크 정책 관리 화면의 조직도에 검색한 사용자 혹은 부서가 선택되고 현재 설정된 보안디스크 정책이 표시됩니다.

![](<../../../.gitbook/assets/img_012 (15).png>)

6. 선택한 부서나 사용자의 보안디스크 정책을 변경하려면 이어지는  **부서/사용자별 정책 설정 및 적용하기**의 내용을 참고합니다.

### <mark style="color:$primary;">보안디스크 정책 설정 및 적용하기</mark>

부서나 사용자에게 적용할 보안디스크 정책을 설정하는 방법은 다음과 같습니다.

1. DiskLock 콘솔 화면의 메뉴에서 **보안디스크** **정책** – **정책 관리** 메뉴를 클릭합니다.&#x20;
2. 부서/사용자 조직도에서 정책을 지정할 부서나 사용자를 선택합니다. 트리 형태로 된 조직도에서 **+**&#xB97C; 클릭하면 하위 부서를 볼 수 있고 최하위 부서의 경우에는 부서에 속한 사용자를 볼 수 있습니다.

![](<../../../.gitbook/assets/img_013 (8).png>)

3. 사용자가 속한 부서를 모르거나 부서를 찾기 어려운 경우에는 [**보안 디스크** **정책을 적용할** **부서/사용자 검색하기**](disklock-5.md#undefined-4) 내용을 참고하여 부서나 사용자를 검색합니다.
4. 오른쪽 **정책적용**에서 선택한 부서나 사용자에 적용할 보안디스크 정책을 설정합니다.

* 보안디스크 정책을 사용하지 않거나 상위 부서의 정책을 그대로 사용하려는 경우: \
  **지정하지 않음(정책제거, 상속**)을 선택합니다.

<div align="left"><figure><img src="../../../.gitbook/assets/img_014 (7).png" alt=""><figcaption></figcaption></figure></div>

* 보안디스크 정책을 변경하려는 경우:\
  해당 보안디스크를 체크한 후 **용량** 항목에 할당할 디스크의 용량을 입력합니다. ‘Gbyte’ 단위로 지정하려면 **GB** 앞의 입력란에, ‘Mbyte’ 단위로 지정하려면 **MB** 앞의 입력란에 각각 입력합니다. 보안디스크를 체크하고 용량을 입력하지 않으면 온라인 보안디스크와 오프라인 보안디스크는 기본 값으로 자동 설정됩니다. **온라인**이나 **오프라인 보안디스크**를 체크하면 **반출 보안디스크**가 자동으로 체크되고, 반출 보안디스크 체크를 해제해도 ‘0’이 아닌 최소 용량으로 설정됩니다.

<div align="left"><figure><img src="../../../.gitbook/assets/img_015 (6).png" alt="" width="326"><figcaption></figcaption></figure></div>

5. 보안디스크 정책이 변경된 부서나 사용자는 조직도의 해당 보안디스크에 붉은 점이 표시됩니다. 우측 상단에 있는 **적용**을 클릭하여 변경한 정책 설정을 서버에 저장하고 적용합니다.

![](<../../../.gitbook/assets/img_016 (6).png>)

6. 보안디스크 용량과 관련된 경고 사항을 알려주는 팝업 창이 나타나면 **확인**을 클릭합니다. 변경된 보안디스크 설정이 서버에 성공적으로 적용되면 조직도의 보안디스크 용량에 표시되어 있던 붉은 점이 사라집니다.
