# 클립보드 복사 제어 정책 관리하기

관리자는 클립보드 복사 제어 정책을 설정하여 클립보드 복사를 제한할 애플리케이션을 등록할 수 있습니다.  클립보드 복사 제어 정책을 적용하면, 정책에 등록된 애플리케이션에서의 클립보드 복사가 차단되어 중요 데이터의 유출을 막을 수 있습니다. 단, 등록된 애플리케이션 간의 복사/붙여넣기는 허용됩니다.

| 복사할 애플리케이션     | 붙여넣기할 애플리케이션   | 복사/붙여넣기 허용 여부 |
| -------------- | -------------- | ------------- |
| **등록된 애플리케이션** | **미등록 애플리케이션** | 차단            |
|                | **등록된 애플리케이션** | 허용            |
| **미등록 애플리케이션** | **모든 애플리케이션**  | 허용            |

{% hint style="warning" icon="square-poll-horizontal" %}
클립보드 복사를 제한할 애플리케이션으로 등록된 경우에도 웹 반출 예외 정책을 설정하여 특정 웹사이트에 대한 붙여넣기를 허용할 수 있습니다. 자세한 사항은 [**웹 반출 예외 정책 관리하기**](../../disklock/disklock/undefined-9.md)를 참고하세요.
{% endhint %}

#### <mark style="color:$primary;">클립보드 복사 제어 정책 생성하기</mark>

1. **윈도우 보안 모듈 관리 - DiskLock Plus - 클립보드 복사 제어 - 정책 관리**를 클릭합니다.

<figure><img src="../../.gitbook/assets/img_001 (101).png" alt=""><figcaption></figcaption></figure>

2. **정책목록**에서 추가 ![](<../../.gitbook/assets/img_002 (86).png>) 버튼을 클릭하여 정책을 추가합니다.

* **‘추가’창에서정책명**(필수)과정책에 대한 설명을 입력합니다.

3. **정책 목록** 우측 영역에서 **추가** 버튼을 클릭합니다.
4. **‘추가’** 창에서 클립보드 복사를 제한할 애플리케이션 카테고리를 선택한 후 오른쪽 화살표를 클릭하여 **선택한 카테고리**에 추가합니다.

* 애플리케이션 카테고리 트리에서 카테고리를 선택하거나, 검색창에 프로세스명(예: winword.exe)을 입력하여 카테고리를 검색할 수 있습니다.

![](<../../.gitbook/assets/img_003 (71).png>)

* 반복하여 다수의 카테고리를 **선택한 카테고리**에 추가할 수 있습니다.
* 이미 선택되어 있는 카테고리를 클릭하고 왼쪽 화살표를 클릭하면 해당 카테고리를 **선택한 카테고리**에서 제거할 수 있습니다.

5. 카테고리 선택을 모두 마친 후 **확인** 버튼을 클릭하면 선택한 카테고리가 정책에 추가됩니다.

애플리케이션 카테고리에 대한 설명은 [**애플리케이션 카테고리 소개**](../../disklock/disklock/undefined-1.md)를 참고하세요.

#### <mark style="color:$primary;">클립보드 복사 제어 정책 변경하기</mark>

클립보드 복사를 제한할 애플리케이션을 정책에 추가하거나, 등록된 애플리케이션을 목록에서 삭제하여 클립보드 복사 제한을 해제할 수 있습니다.

1. **윈도우 보안 모듈 관리 - DiskLock Plus - 클립보드 복사 제어 - 정책 관리**를 클릭합니다.
2. **정책 목록**에서 변경할 정책을 선택합니다.
3. 우측 영역에서 **추가** 버튼을 클릭하여 애플리케이션 카테고리를 추가로 등록하거나, **휴지통**아이콘을 클릭하여 등록된 카테고리를 목록에서 삭제할 수 있습니다.

* **선택한 카테고리** 중 클립보드 복사 제한을 해제할 카테고리 우측의 **휴지통** 아이콘을 클릭하면 해당 카테고리가 목록에서 삭제됩니다.
* 클립보드 복사를 제한할 애플리케이션 카테고리를 추가로 등록하는 방법은 정책 생성 시와 동일합니다.
