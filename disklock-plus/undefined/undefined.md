# 보안 해제 정책 관리하기

사용자가 업무상 필요한 경우에 한하여 승인권자 또는 관리자의 승인 하에 프린트 워터마크나 인쇄 제어 등 일부 보안 정책을 일시 해제하도록 허용할 수 있습니다. 관리자는 사용자/부서별로 보안 해제 정책을 설정하여 사용자의 보안 해제 신청 시 적용될 **결재 방식, 보안 해제 허용 기간, 해제 신청 가능한 보안 정책**을 제한할 수 있습니다.

보안 해제 정책을 설정하는 방법은 다음과 같습니다.

![](<../../.gitbook/assets/img_000 (105).png>)

1. **PC 보안 모듈 관리 - 보안 해제 - 정책 관리**를 클릭합니다.
2. **정책 목록**에서 추가 버튼 ![](<../../.gitbook/assets/img_001 (98).png>) 을 클릭하여 정책을 추가합니다. 기존 정책을 변경할 경우에는 정책 목록에서 변경할 정책을 선택합니다.

<div align="left"><img src="../../.gitbook/assets/img_002 (83).png" alt=""></div>

3. 새로 추가할 정책명과 정책 설명을 입력하고 **확인**을 클릭합니다. 정책명은 필수로 입력해야 하며 기존에 정책 목록에 있는 이름과 중복되어서는 안됩니다.
4. 정책 항목 영역에 각 항목 값을 입력합니다.

* **결재 방식**: 보안 해제 신청 시 적용될 결재 방식을 설정합니다.

> - **선결재**: 보안 해제 신청 시 결재 방식이 선결재로 고정됩니다. 선결재 시 승인권자의 승인이 완료된 후에만 보안 해제가 가능합니다.
> - **후결재**: 보안 해제 신청 시 결재 방식이 후결재로 고정됩니다. 후결재 시 신청 후 곧바로 해당 보안 정책이 해제되며 사후 승인이 수반됩니다
> - **선결재/후결재**: 사용자가 보안 해제 신청 시 선결재와 후결재 중에서 결재 방식을 선택할 수 있도록 합니다.
> - **\[선결재/후결재]를 선택한 경우 기본표시**: 선결재/후결재를 선택할 수 있는 경우 사용자의 신청 화면에 기본으로 표시될 결재 방식을 선택합니다.

* **최대 허용 기간**: 보안 해제를 허용할 최대 기간을 설정합니다.

> - **고정**: 사용자는 관리자가 지정한 기간 동안 보안 해제 신청을 할 수 있습니다.
> - **최대**: 보안 해제 신청일로부터 사용자가 지정 가능한 최대 일자를 설정합니다.
> - **제한하지 않음**: 사용자가 자유롭게 보안 해제 기간을 선택하여 신청할 수 있습니다.
> - **\[최대, 제한하지 않음]으로 선택한 경우 기본표시**: 최대 허용 기간을 **최대** 혹은 **제한하지 않음**으로 선택한 경우 사용자의 신청 화면에 기본으로 표시되는 해제 기간을 설정합니다.

* **해제할 보안 정책**:  사용자가 해제 신청할 수 있는 보안 정책을 선택합니다.

> - **프린트 워터마크**: 인쇄물 출력 시 회사 로고 또는 인쇄하는 사용자 정보 등을 인쇄물에 추가하는 정책
> - **인쇄 제어**:  프린터별로 인쇄가 허용되는 어플리케이션을 제한하는 정책

5. 우측 하단의 **적용** 버튼을 클릭하여 정책 항목 설정 사항을 해당 정책에 반영합니다.

{% hint style="warning" icon="square-poll-horizontal" %}
생성된 정책은 [**모듈 통합정책 적용하기**](../../disklock/undefined/undefined.md)를 통해 부서/사용자별로 적용할 수 있습니다.
{% endhint %}
