# 네트워크 통신 로그의 구성 및 확인 방법

에이전트는 HTTP(HTTPS)로 서버와 다양한 통신을 수행합니다. 문서함 파일 작업을 위해 파일 서버와 통신하고, 관리자가 설정한 정책이나 알림 정보 등을 수신하기 위해 웹 서버와 통신을 수행합니다.

에이전트를 통해 통신 로그를 확인할 수 있습니다.

에이전트에 로그인 후, 트레이 메뉴의 **프로그램 정보** 메뉴를 Ctrl 키와 Shift 키를 동시에 누른 상태에서 클릭합니다.

![](<../../.gitbook/assets/img_000 (224).png>)

통신 로그 창이 표시되며 아래 정보를 표시합니다. 창 좌측 하단의 **모든 로그 보기**는기본 체크 해제인데 오류가 발생한 통신 로그만 목록에 표시합니다. **모든 로그 보기** 항목의 체크박스를 선택하면 성공한 로그도 표시하나 문서함 입출력 성능이 다소 저하됩니다.

* **시간** : 통신 요청이 발생한 시작 시간
* **소요시간** : 통신 요청부터 통신 종료까지 걸린 시간 (단위: 1/1000초)
* **프로세스**: 파일 작업을 요청한 프로세스 이름
* **작업** : 요청한 파일 작업 (약어로 표시)
* **경로** : 파일 작업 경로 이름
* **결과** : 서버 처리 결과 (오류가 없으면 ‘-‘로 표시)
* **정보** : 각 작업의 부가 정보

소켓 리셋, 소켓 타임아웃(timeout)과 같은 통신 오류가 발생하면 **결과**에 Socket Error(오류코드,…) 가 표시됩니다. 통신 오류 빈도가 높다면 서버 및 네트워크 상태를 체크해야 합니다.

그리고, **소요 시간** 정보는 문서함 입출력 성능에 대한 객관적인 지표로 활용될 수 있습니다.
