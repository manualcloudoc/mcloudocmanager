# Netstat 서버 관제 로그 항목 및 비정상 판단 기준

​서버 관제 로그중의 하나인 netstat로그에 대해 설명합니다.

문서에서 경로는 리눅스 기준으로 명시하였습니다. 윈도우 서버인 경우에는 C: 드라이브 하위의 동일 경로입니다.

* **로그의 위치 : /plusdrive/log/monitoring\_log/netstat\_log**
* **로그 파일 명 : netstat\_년원일\_시분초.txt**

### <mark style="color:$primary;">로그 항목별 상세 설명</mark>

현재 열린 네트워크 연결 목록과 해당 연결의 상태, 프로토콜, 로컬 및 원격 IP 및 포트 번호 등의 정보를 보여줍니다.

| <p><strong>항목별 상세</strong></p><p></p><p><strong>Proto</strong>: 이 필드는 연결에 사용되는 프로토콜을 나타냅니다. TCP, UDP, ICMP 등이 여기에 해당됩니다.</p><p><strong>Local Address</strong>: 이 필드는 로컬 컴퓨터의 IP 주소와 포트 번호를 나타냅니다. 로컬 호스트의 포트 번호는 연결을 열고 있는 프로그램을 식별하는 데 사용됩니다. </p><p><strong>Foreign Address</strong>: 이 필드는 연결이 수립되는 대상 컴퓨터의 IP 주소와 포트 번호를 나타냅니다. </p><p><strong>State</strong>: 이 필드는 현재 연결의 상태를 나타냅니다. 연결이 열려 있는지, 대기 중인지, 닫힌 상태인지 등을 나타내며, 특정 상태에 따라 특정한 작업을 수행할 수 있습니다. </p><p><strong>ESTABLISHED</strong>: 연결이 수립되어 데이터가 주고받고 있는 상태입니다. </p><p><strong>LISTEN</strong>: 해당 포트에서 연결을 수신 대기 중인 상태입니다. 서버 프로그램에서 많이 사용됩니다. <strong>SYN_SENT</strong>: 연결 요청을 보낸 상태이며, 아직 연결이 수립되지 않은 상태입니다. </p><p><strong>SYN_RECV</strong>: 연결 요청을 받아들인 상태이며, 아직 연결이 수립되지 않은 상태입니다. </p><p><strong>FIN_WAIT1</strong>: 연결 종료를 요청한 후 상대방의 응답을 기다리는 상태입니다. </p><p><strong>FIN_WAIT2</strong>: 상대방의 연결 종료 응답을 받고, 자신의 연결 종료를 요청한 후 상대방의 종료 응답을 기다리는 상태입니다. </p><p><strong>TIME_WAIT</strong>: 연결 종료를 요청하고, 상대방의 연결 종료 응답을 받은 후, 일정 시간 동안 대기하는 상태입니다. </p><p><strong>CLOSE_WAIT</strong>: 상대방이 연결 종료를 요청하고, 자신이 종료 응답을 보내는 것을 기다리는 상태입니다. </p><p><strong>CLOSED</strong>: 연결이 종료되었거나, 초기 상태로 돌아간 상태입니다. </p><p><strong>CLOSING</strong>: 자신이 연결 종료를 요청하고, 상대방의 응답을 기다리는 상태입니다. </p><p><strong>LAST_ACK</strong>: 상대방이 연결 종료를 요청한 후, 자신이 종료 응답을 보내는 상태입니다. </p><p><strong>UNKNOWN</strong>: 알 수 없는 연결 상태입니다. </p><p><strong>Receive-Q</strong>: 이 필드는 현재 수신 대기 중인 데이터의 양을 나타냅니다. </p><p><strong>Send-Q</strong>: 이 필드는 현재 전송 대기 중인 데이터의 양을 나타냅니다. </p><p><strong>Timer</strong>: 이 필드는 TCP 연결에서 사용되며, 타이머 값을 나타냅니다. 타이머는 연결 시간 제한, 재전송 제한 등과 같은 연결 설정에 사용됩니다.</p> |
