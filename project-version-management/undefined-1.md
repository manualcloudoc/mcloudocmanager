# 프로젝트 버전 관리 신청 처리 및 관련 이력 보기

#### <mark style="color:$primary;">프로젝트 버전 관리 신청 처리하기</mark>

관리자가 폴더관리자의 프로젝트 버전 관리 신청에 대해 승인/반려하는 방법을 설명합니다. 이미 버전 관리가 진행되고 있는 프로젝트 폴더에 대해 폴더관리자가 버전 관리 등록 내용을 수정하거나 삭제할 때에도 관리자의 승인이 필요합니다.

1. **백업 모듈 관리 - 프로젝트 버전 관리 - 백업 신청 처리**를 클릭합니다.

<figure><img src="../.gitbook/assets/img_000 (183).png" alt=""><figcaption></figcaption></figure>

2. 폴더관리자가 버전 관리를 신청/수정/삭제 요청한 프로젝트 목록을 확인합니다. 신청일시, 프로젝트명, 프로젝트 폴더 경로, 문서함, 신청자 이름(ID)를 확인할 수 있습니다.

* **구분** 항목에는 신청 유형(신청/수정/삭제)이 표시됩니다.
* 프로젝트명을 클릭하면 상세보기 페이지로 이동할 수 있습니다.

3\.   승인/반려 처리하는 방법에는 2가지가 있습니다.

* **목록에서 선택하여 승인 또는 반려하는 방법**

> - 승인 또는 반려 처리할 요청 건 왼쪽의 체크 박스에 체크한 후 우측 상단에서 승인 또는 반려 버튼을 클릭합니다. 여러 요청 건, 혹은 신청 목록의 모든 요청 건을 동시에 선택하여 한 번에 승인 또는 반려 처리할 수도 있습니다. 이 경우에는 관리자가 작성하는 승인 또는 반려 사유가 선택한 목록 모두에 동일하게 입력됩니다.

<figure><img src="../.gitbook/assets/img_001 (168).png" alt=""><figcaption></figcaption></figure>

* **요청 건 별로 상세 내역을 확인한 후 승인 또는 반려하는 방법**

> - 버전 관리를 신청한 목록에서 해당 **프로젝트명**을 클릭합니다.
> - **상세보기** 페이지로 이동하여 신청한 백업 정보에 대한 자세한 정보를 확인합니다. 페이지 하단에서 **승인** 또는 **반려** 버튼을 클릭합니다.

<figure><img src="../.gitbook/assets/img_002 (143).png" alt=""><figcaption></figcaption></figure>

> * **‘승인’/’반려’** 창에서 사유를 입력합니다.
> * 승인 또는 반려가 완료되면 신청자에게 승인/반려 알림 메일이 발송됩니다.

{% hint style="warning" icon="square-poll-horizontal" %}
폴더관리자가 프로젝트 버전 관리의 등록/수정/삭제를 요청한 경우 관리자에게 메일이 발송됩니다. 관리자는 메일에서 **결재 페이지로 이동하기**를 클릭하여 신청 건의 **상세보기** 페이지로 이동할 수 있습니다.

![](<../.gitbook/assets/img_004 (96).png>)

프로젝트 버전 관리 메뉴의 상세보기 페이지에서 바로 승인/반려할 수 있습니다.

![](<../.gitbook/assets/img_005 (83).png>)
{% endhint %}

{% hint style="danger" %}
프로젝트 버전 관리를 삭제하면, 일정 기간 보관(기본 30일) 후 백업된 자료들을 모두 삭제하니 **삭제** 요청에 대해서는 유의하여야 합니다.
{% endhint %}

### 프로젝트 버전 관리 신청 처리 이력 보기

프로젝트 버전 관리의 신청/수정/삭제 요청에 대한 승인/반려/취소 처리 이력을 확인할 수 있습니다.

1. **백업 모듈 관리 - 프로젝트 버전 관리 - 백업 신청 처리 이력**을 클릭합니다.&#x20;

<figure><img src="../.gitbook/assets/img_007 (60).png" alt=""><figcaption></figcaption></figure>

2. 검색 조건을 입력하고 **검색**을 클릭합니다.

* **작업 기간**(필수): 백업 신청을 처리한 날짜
* **문서함**(필수): 프로젝트 폴더가 위치한 문서함

> 선택할 수 있는 문서함이 표시되며, 검색에서 제외할 문서함을 삭제합니다. **문서함 선택**을 클릭하면 ‘**문서함 선택**’ 창에서 문서함을 선택할 수 있습니다.

* **구분**(필수): 프로젝트 버전 관리 폴더에 대한 요청을 구분

> - **신청**: 프로젝트 버전 관리 폴더를 신규로 신청
> - **수정**: 버전 관리 등록 내용을 변경 요청
> - **삭제**: 더 이상 버전 관리를 하지 않기 위해 삭제 요청

* **상태**(필수): 처리 상태

> - **승인**: 폴더관리자의 요청을 승인
> - **반려**: 폴더관리자의 요청을 반려
> - **취소**: 관리자가 처리하기 전에 폴더관리자가 요청을 취소

* 기타 검색 조건(선택):

> - **프로젝트명**: 백업 신청한 프로젝트 이름
> - **프로젝트 폴더 경로**: 백업 신청한 프로젝트 폴더의 경로
> - **신청자 이름**: 백업 신청한 사람의 이름
> - **신청자 ID**: 백업 신청자ID

3. 검색 조건에 해당하는 처리 이력의 **작업일시, 신청 유형, 프로젝트명, 프로젝트 폴더 경로, 문서함, 신청자 이름(ID), 작업 상태**가 목록 형태로 표시됩니다. 목록에서 프로젝트명을 클릭하면 **상세보기** 페이지에서 **신청일시, 정기적 백업 정보, 승인/반려 사유**를 확인할 수 있습니다.
