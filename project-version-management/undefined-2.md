# 프로젝트 버전 관리 백업 이력 보기

관리자는 프로젝트 폴더의 버전 관리 등록 정보에 따라 주기적으로 실행된 정기 백업 이력과 폴더관리자가 임의로 실행한 즉시 백업 이력을 모두 확인할 수 있습니다.

{% hint style="warning" icon="square-poll-horizontal" %}
백업 작업이 처음에 설정한 최대 보관 스냅샷 수(예: 100)를 초과하게 되는 경우, 가장 오래된 순서로 삭제되고 최근 스냅샷들이 보관됩니다. 그러나 **백업 이력** 메뉴에서는 과거 백업이 실행되었던 이력 전체를 조회할 수 있습니다.&#x20;
{% endhint %}

1. **백업 모듈 관리 - 프로젝트 버전 관리 - 백업 이력**을 클릭합니다.

<figure><img src="../.gitbook/assets/img_001 (167).png" alt=""><figcaption></figcaption></figure>

2. 검색 조건을 입력하고 **검색**을 클릭합니다.

* **시작일 선택**(필수): 각 백업 작업의 시작 날짜
* **작업 종류**(필수): **정기** 백업 또는 **즉시** 백업

> - **정기:** 폴더관자가 프로젝트 버전 관리 신청 시 설정한 백업 스케줄에 따라 실행된 백업
> - **즉시:** 폴더관리자가 직접 실행한 백업

* 검색 조건(선택):

> - **프로젝트명**:폴더관리자가 프로젝트 버전 관리 신청 시 명명한 프로젝트 이름
> - **작업명**: 작업 종류가 **정기**인 경우에는 **백업 유형**을, **즉시**인 경우에는 작업자가 백업을 실행하면서 직접 입력한 **작업명**을 입력하여 검색
>
> > * 백업 유형 :
> >
> > > - **매시간**: 지정한 시간마다 정기백업을 진행
> > > - **매일**: 매일 지정한 시간마다 정기백업을 진행
> > > - **매주**: 지정한 요일의 각 지정한 시간마다 정기백업을 진행
> > > - **매월**: 지정한 날짜의 지정한 시간마다 정기백업을 진행
>
> * **작업자 이름/ID**: 즉시 백업인 경우에는 작업한 관리자(폴더관리자 또는 서비스관리자)의 이름/ID, 정기 백업인 경우에는 ‘관리자’로 표시

3. 검색 조건에 해당하는 백업 이력이 표시됩니다. 각 백업 건에 대하여 시작 및 종료 시간과 전송된 파일 수 및 용량 등을 확인할 수 있습니다.

* **시작일시/종료일시**: 백업이 시작된 시간과 종료된 시간
* **작업 종류**: 정기 백업 또는 즉시 백업
* **프로젝트명**: 프로젝트 버전 관리 신청 시 지정한 프로젝트 이름
* **작업명**:

> - 정기 백업인 경우 매시간, 매일, 매주, 매월로 스케줄을 표시합니다.
> - 즉시 백업인 경우 작업자가 입력한 작업 이름으로 표시합니다.\
>   (작업 이름 예: \[202404301035\_kdkim] 스냅샷을 생성합니다)

* **작업자(ID)**: 백업을 진행한 관리자의 이름과 ID(정기 백업인 경우에는 ‘관리자’)
* **결과**: 성공 여부
* **전송 파일 수**: 백업된 파일들의 총 개수입니다.
* **전송 용량**: 백업된 파일들의 총 용량입니다.
* **목적지 파티션 용량**: **프로젝트 폴더 백업에 사용되는 스토리지의 전체 용량** 대비 **현재 프로젝트 폴더의 백업에 사용되는 용량**이 표시됩니다. 예: 1.32TB(사용량)/1.79TB(전체용량)
* **백업 후 목적지 용량**: 해당 백업 작업으로 인해 백업된 파일들의 총 용량입니다. 증분 백업으로 작업된 백업 용량이 표시됩니다.

{% hint style="warning" icon="square-poll-horizontal" %}
**증분 백업**은 백업이 수행된 후 변경된 사항(추가/수정/삭제)을 선택적으로 백업하는 방식입니다. 전체 백업보다 백업하는 데이터의 양이 적어 속도가 빠르다는 장점이 있습니다.
{% endhint %}

4. **작업명** 항목을 클릭하면 **상세보기** 페이지로 이동합니다.

<figure><img src="../.gitbook/assets/img_003 (122).png" alt=""><figcaption></figcaption></figure>

* **작업 설명**: 즉시 백업인 경우에만 작업자가 입력한 설명이 표시됩니다.

> (설명 예: 서비스관리자가 작업한 2024년 4월 9일 18시 즉시 백업입니다.)

* **전송 파일 목록**: 백업된 파일의 파일명, 크기, 백업 구분을 표시합니다.

> - 백업구분
>
> > * **신규:** 이전 백업 스냅샷과 비교 시 이번 백업에서 추가된 파일
> > * **삭제:** 이전 백업 스냅샷과 비교 시 이번 백업에서 삭제된 파일
