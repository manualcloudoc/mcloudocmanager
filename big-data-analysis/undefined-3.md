# 빅데이터 - 파일 작업 통계 보기

​\
중앙문서함에서 발생한 파일 작업에 대한 통계를 날짜별로 수집하여 제공하는 기능입니다. 선택한 기간 동안에 특정 문서함에서 발생한 업로드, 다운로드, 삭제 등의 파일 작업 통계(항목별 합계)를 확인할 수 있으며, 포함되는 항목은 다음과 같습니다.

1. **업로드**: 업로드 용량과 업로드 횟수
2. **다운로드**: 다운로드 용량과 다운로드 횟수
3. **삭제**: 삭제 용량과 삭제 횟수
4. **기타**: 이름 변경 횟수, 파일 생성 횟수, 폴더 생성 횟수, 복사 횟수, 이동 횟수, 휴지통 비우기 횟수

또한 위의 항목에 대해 입력한 기간 내의 일자별 변동 추이를 그래프로 확인할 수 있습니다. 특히, 특정 부서의 파일 작업 통계를 조회할 경우에는 업로드, 다운로드, 삭제 작업에 대해 부서 내 사용자별 통계와 사용자 순위를 차트로도 확인할 수 있습니다.

파일 작업 통계는 회사 내 파일의 생산(업로드)과 소비(다운로드) 패턴을 분석하는데 유용하게 활용할 수 있습니다.

![Notes](<../.gitbook/assets/img_000 (103).png>)빅데이터 통계는 빅데이터 서비스 구매 시 사용 가능한 기능입니다.\
1\.    **빅데이터 통계 – 파일 작업**을 선택합니다.![](<../.gitbook/assets/img_001 (96).png>)  \
2\.   검색 조건을 입력하고 **검색** 버튼을 클릭합니다.

1. **전체** 탭 선택 시 다음과 같은 조건으로 검색할 수 있습니다.

> 1) **기간 선택(필수)**: 조회하고자 하는 기간을 선택합니다.
> 2) **문서함(필수)**: 중앙문서함의 문서함 단위로 선택할 수 있습니다.

1. **부서** 탭 선택 시 특정 부서의 사용자가 실행한 파일 작업 현황을 확인할 수 있습니다.
2.  위의 검색 조건에 추가하여 **부서명** 또는 **부서코드**를 필수적으로 입력합니다.

    ![](<../.gitbook/assets/img_002 (81).png>)
3. **사용자** 탭 선택 시 특정 사용자가 실행한 파일 작업 현황을 확인할 수 있습니다.

> 1) 위의 검색 조건에 추가하여 **사용자 이름 또는 사용자 ID**를 필수적으로 입력합니다.\
>    &#x20;     ![](<../.gitbook/assets/img_003 (67).png>)

3\.   위의 검색 조건에 해당하는 파일 작업 통계가 다음과 같이 표시됩니다.  항목별 합산 건수/용량이 다음과 같이 표시됩니다.

1. **파일 작업 항목별 합계**![](<../.gitbook/assets/img_004 (50).png>)
2. 각 파일 작업 항목에 대한 일자별 변동 추이 그래프

> ![](<../.gitbook/assets/img_005 (42).png>)
