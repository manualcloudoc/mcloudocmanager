# 버전관리 설정

![Info](<../../.gitbook/assets/img_000 (206).png>)메뉴 경로: 시스템 설정 - 설정 홈 연결 - 일반 모듈 설정 - Basic - 버전 관리​

|                           |                                                                                                                                                                 |             |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------- |
| **버전관리 옵션**               | **내용**                                                                                                                                                          | **기본값**     |
| **버전 관리 사용 여부**           | 버전 관리 기능의 사용 여부를 설정합니다.                                                                                                                                         | 사용함         |
| **임시파일 보관 기간 설정**         | 임시파일은 사용자가 파일을 저장할 때마다 자동으로 생성되는 이전 버전 파일입니다. 설정한 기간이 지나면 자동으로 삭제됩니다. ‘삭제 안 함’의 경우 보관 기간 제한이 없습니다.                                                              | 180일        |
| **파일별 최대 보관 개수**          | 파일별 생성되는 이전 버전 파일의 최대 보관 개수를 설정합니다.  개수가 초과되면 기간순으로 오래된 버전 파일이 자동으로 삭제됩니다. ‘사용 안 함’의 경우 보관 개수 제한이 없습니다.                                                         | 사용 안 함      |
| **버전관리 대상 파일 최대 크기(MB)**  | 버전관리 대상 파일의 최대 크기를 설정합니다. 최대 크기를 초과하면 백업하지 않기 때문에 버전관리가 되지 않습니다. ‘제한 없음’의 경우 최대 크기 제한이 없습니다. 구매하신 서버 용량과 사용 현황을 고려하여 설정을 해주세요.                                  | 1024MB      |
| **버전관리 대상에서 제외될 확장자 목록**  | 버전관리 대상에서 제외할 파일의 확장자를 입력합니다.                                                                                                                                   | -           |
| **버전관리 대상에서 제외될 프로세스 목록** | 버전관리 대상에서 제외할 프로세스를 설정합니다. outlook.exe와 같은 프로세스는 계속 파일(pst파일 등)을 변경하기 때문에 버전관리를 위해 백업되는 파일양이 무한정 커질 수 있습니다. 이를 방지하기 위해 outlook.exe 프로세스는 버전관리 대상에서 기본적으로 제외합니다. | outlook.exe |

![Warning](<../../.gitbook/assets/img_001 (172).png>)‘**임시파일 보관 기간 설정: 삭제 안 함**’, ‘**파일별 최대 보관 개수: 사용 안 함**’을 동시에 설정하여 사용할 경우 서버 스토리지 공간 부족 문제가 발생할 수 있으니 주의하세요.
