# 검출된 개인정보 파일 관리하기

관리자는 개인정보가 검출된 파일들의 내역을 조회하고 파일별로 격리, 삭제, 보유, 오탐 등 적절한 조치를 취할 수 있습니다.

1. **일반 모듈 관리 – 개인정보보호 – 검출 파일 관리**를 클릭합니다.

![](<../.gitbook/assets/img_000 (178).png>)

2. 다음과 같이 개인정보 **검출 파일을** 검색할 수 있습니다.

* 조회를 원하는 **기간**, **문서함** 종류, **검출 항목, 처리 상태**를 선택합니다. 필요 시 **검색 조건**을 입력한 후 **검색** 버튼을 클릭합니다.
* **문서함 선택** 버튼을 클릭하면 체크 박스 형태로 표시되는 전체 문서함 중 개인정보 검출 파일을 검색할 문서함을 선택하거나 문서함 이름으로 키워드 검색하여 추가할 수 있습니다. 기본값은 전체 문서함 선택입니다.

> <img src="../.gitbook/assets/img_001 (164).png" alt="" data-size="original">

* **처리 상태**의 종류는 다음과 같습니다.

> - **미처리**: 미처리 상태
> - **보유**: 정상적으로 개인정보가 검출되었으나 보유해야 하는 경우
> - **격리**: 검출된 파일을 접근할 수 없는 경로로 격리한 경우
> - **오탐**: 실제로 개인정보에 해당하지 않지만 개인정보로 잘못 인식되어 오류로 분류된 경우

* **검색 조건**: 필요시 검색 조건으로 **파일명, 파일 경로, 파일 생성자 이름, 파일 생성자 ID**를 입력합니다. 이때 검색어의 일부만 입력하여 검색할 수 있습니다. **검색 조건**을 입력하지 않을 경우 전체 검출 파일 목록이 표시됩니다.

3\.      검색된 개인정보 **검출 파일**에 대해 다음과 같이 조치하여 처리 상태를 변경할 수 있습니다.

* 처리 상태를 변경할 파일 왼쪽에 있는 체크 박스를 선택합니다.
* 파일 목록 오른쪽 상단에 있는 처리 상태 변경 버튼(**해제/복원, 보유, 오탐, 격리, 삭제**)을 클릭합니다.

> - **해제/복원**: 기존 처리 상태를 초기화하여 미처리 상태로 복구
> - **보유**: 정상적으로 개인정보가 검출되었으나 보유해야 하는 경우 보유 처리
> - **오탐**: 실제로오류가 존재하지 않지만 개인정보로 잘못 인식되어 오탐 처리
> - **격리**: 검출된 파일을 접근할 수 없는 경로로 격리
> - **삭제**: 지정된 파일을 개인

{% hint style="warning" icon="square-poll-horizontal" %}
**격리** 또는 **삭제** 처리 후에는 해당 파일을 문서함에서 조회할 수 없습니다. **삭제**할 경우 개인정보 검출 파일 목록에서 제거되고, **해제/복원**으로 복구할 수 없습니다.
{% endhint %}
