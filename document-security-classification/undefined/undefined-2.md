# 문서보안등급 변경 정책 관리하기

문서보안등급 기능 사용 시, 문서의 보안등급은 서비스관리자나 폴더관리자가 설정할 수 있습니다. 일반 사용자의 경우에는 관리자가 허용한 범위 내에서 문서의 보안등급 변경을 신청할 수 있으며, 신청 후 폴더관리자의 승인이 필요합니다.관리자는 **문서보안등급 변경 정책**을 생성하여 사용자가 변경 신청 가능한 보안등급 범위와 결재 방식 등을 설정하고, 부서 및 사용자에 따라 적절한 정책을 적용하여 부서/사용자별로 보안등급 변경 신청을 제한할 수 있습니다. 정책이 적용되지 않은 사용자는 보안등급 변경 신청을 할 수 없습니다.

#### <mark style="color:$primary;">문서보안등급 변경 정책 설정하기</mark>&#x20;

관리자는 다음과 같이 문서보안등급 변경 정책을 생성하거나 기존 정책을 변경할 수 있습니다. 사용자의 보안등급 변경 신청 시 적용될 결재 방식과 변경 가능한 등급 범위를 설정합니다.

1. 관리자 웹페이지에서 **일반 모듈 관리 - 문서 보안 등급 - 등급 변경 정책 관리 - 정책 설정**을 선택합니다.&#x20;
2. &#x20;**정책 설정** 페이지의좌측 정책 목록에서 추가 버튼 <img src="../../.gitbook/assets/img_000 (145).png" alt="" data-size="line">을 클릭하여 정책을 추가합니다. 기존 정책을 변경하는 경우에는 4단계로 넘어갑니다.

<figure><img src="../../.gitbook/assets/img_001 (136).png" alt=""><figcaption></figcaption></figure>

3. **‘추가’** 창에서 **정책명**(필수)과 **정책 설명**을 입력하고 **확인**을 클릭합니다.

<div align="left"><img src="../../.gitbook/assets/img_002 (116).png" alt="" width="392"></div>

4. **정책 설정** 페이지의 좌측 정책 목록에서 추가된 정책명을 클릭합니다. 기존 정책을 변경하고자 하는 경우, 변경하고자 하는 정책명을 클릭합니다.
5. 페이지 우측의 정책 세부 항목을 설정합니다. 설정에 따라 사용자의 ‘**문서 보안등급 변경 신청**’ 화면 내용이 결정됩니다.&#x20;

* **결재 방식**:사용자가 변경 신청 시 적용될 결재 방식을 설정합니다.

![](<../../.gitbook/assets/img_003 (100).png>)

> * **일반 문서**: 다음의 결재 방식 중 선택할 수 있습니다.     &#x20;
>
> > - 선결재: 승인권자 승인 후에 문서 보안등급 변경이 가능합니다.
> > - 후결재: 문서 보안등급 변경이 먼저 진행된 후 승인권자에게 승인을 받습니다.
> > - 선결재/후결재: 사용자가 선결재, 후결재 중 선택하여 신청할 수 있습니다. 이 경우, 문서 보안등급 변경 신청 화면에 기본값으로 표시될 결재방식을 설정합니다.
> > - 금지: 사용자의 문서 보안등급 변경 신청을 금지합니다.

> * **보안등급문서**: 보안등급이 설정된 문서의 결재 방식을 설정하며 선택 가능한 옵션은 일반 문서와 같습니다.
>
> > - 보안 등급별 세부 설정: 보안등급별로 결재 방식을 지정할 수 있습니다. 우측의 **편집** 버튼을 클릭하여 ‘**보안 등급별 세부 설정**’ 창에서 보안등급별로 결재 방식을 다르게 선택할 수 있습니다.

<div align="left"><img src="../../.gitbook/assets/img_004 (77).png" alt="" width="563"></div>

> * **개인정보문서**: 개인정보가 포함된 문서의 결재 방식을 선택하며 선택 가능한 옵션은 일반 문서와 같습니다.

* **변경 가능 등급 범위**: 문서 보안등급 변경 신청 시 새로 지정할 등급으로 허용할 등급 범위를 선택합니다. 사용자는 체크된 등급 중에서 새 등급을 선택할 수 있습니다.

<figure><img src="../../.gitbook/assets/img_005 (65).png" alt=""><figcaption></figcaption></figure>

6\.  페이지 우측 하단의 **적용** 버튼을 클릭합니다.

{% hint style="danger" %}
세부 항목 페이지에서 설정을 변경하더라도 **적용** 전에는 실제로 서버에 반영되지 않습니다. 반드시 **적용** 버튼을 클릭해야 변경 사항이 서버에 적용됩니다.
{% endhint %}

#### <mark style="color:$primary;">​문서보안등급 변경 정책 적용하기</mark>

생성한 등급 변경 정책을 부서 또는 사용자에게 적용합니다. 사용자는 등급 변경 정책이 적용되어야 문서의 보안등급 변경을 신청할 수 있습니다.&#x20;

1. 관리자 웹페이지에서 **일반 모듈 관리 - 문서 보안 등급 - 등급 변경 정책 관리 - 정책 적용**을 선택합니다.
2. **조직도**에서 사용자 또는 부서를 선택한 후 화면 우측의 **정책 목록**에서 적용할 정책을 선택한 후 **적용** 버튼을 클릭합니다.

<figure><img src="../../.gitbook/assets/img_007 (49).png" alt=""><figcaption></figcaption></figure>

{% endhint %}
