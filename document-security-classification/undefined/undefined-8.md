# 보안등급이 다른 문서들의 동시 열람 시 유의 사항

하나의 애플리케이션에서 보안등급이 서로 다른 문서들을 동시에 열람하는 경우, 다음과 같이 의도하지 않은 상황이 발생할 수 있습니다.

* **문서의 보안등급 상승**: 다양한 보안등급의 문서를 하나의 애플리케이션으로 동시에 열람하다가 그 중 한 문서를 저장하는 경우, 저장되는 문서의 보안등급이 동시에 열람 중이던 문서들의 보안등급 중 가장 높은 등급으로 변경됩니다. 이에 따라 문서 저장 시 보안등급이 의도치 않게 상향 변경될 수 있습니다.
* **금지 권한의 적용 범위 확대**: 특정 보안등급에 대해 **저장,** **다른 이름 저장, 클립보드 복사, 화면 캡처, 인쇄,** 또는 **프린트 워터마크** 등의 문서 작업 권한이 **금지**(워터마크의 경우에는 **출력**)로 설정된 경우, 이 보안등급의 문서를 열람 중인 애플리케이션은 동시에 열람하는 모든 문서에 대해 해당 작업을 수행할 수 없습니다.

위의 각 항목에 대해 다음에서 자세히 알아봅니다.

#### <mark style="color:$primary;">문서의 보안등급 상승</mark>

하나의 애플리케이션을 사용하여 다양한 보안등급의 문서를 동시에 열람하다가그 중 한 문서를**저장** 또는 **다른 이름으로 저장**하는 경우, 저장되는 문서의 보안등급은 해당 애플리케이션이 동시에 열람 중인 문서들의 보안등급 중 가장 높은 등급으로 변경 설정됩니다.

예를 들어, MS Word로 2등급 보안 문서, 3등급 보안 문서, 일반 문서를 동시에 열람하다가 3등급 문서, 또는 일반 문서를 저장할 경우, 문서의 보안등급이 모두 2등급으로 설정됩니다.&#x20;

#### <mark style="color:$primary;">금지 권한의 적용 범위 확대</mark>

특정 보안등급에 대해 **저장, 다른 이름 저장, 클립보드 복사, 화면 캡처, 인쇄,** 또는 **프린트 워터마크** 등 문서 작업 권한이 **금지**(워터마크의 경우에는 **출력**)로 설정된 경우, 해당 보안등급의 문서를 열람 중인 애플리케이션이 동시에 열람하는 모든 문서(일반 문서 및 보안 문서)에 대해 **금지** 권한이 한시적으로 확대 적용됩니다. 확대 적용된 **금지** 권한은 해당 애플리케이션의 종료 시까지 유효합니다.

예를 들어 MS Word를 사용하여 **다른 이름 저장** 권한이 **금지**로 설정된 보안등급의 문서 A, **다른 이름 저장** 권한이 **허용**으로 설정된 보안등급의 문서 B와 일반 문서 C를 동시에 열람할 경우, 문서 A뿐만 아니라 문서 B, C에 대해서도 **다른 이름으로 저장**이 **금지**됩니다. 이때 다른 애플리케이션(예: MS Excel)이 열람하는 문서에는 영향을 미치지 않으며, MS Word가 문서 A, B, C를 모두 닫고 종료하면 문서 B, C에 대해 임시로 적용되었던 **금지** 권한이 해제됩니다.

한편, MS Office 등의 애플리케이션을 사용하여 **저장**이 **금지**(단, 읽기는 허용)된 보안등급의 문서를 열람할 경우, 문서가 **읽기 전용**으로 열리며, 해당 문서가 열려 있는 상태에서 동일한 애플리케이션이 열람을 시도하는 다른 모든 문서도 보안등급과 관계없이 **읽기 전용**으로 열립니다. 또한, 해당 문서를 열기 전에 이미 편집 가능한 상태로 열려 있던 문서에서도 더 이상 저장이 허용되지 않습니다

{% hint style="warning" icon="square-poll-horizontal" %}
**클립보드 복사, 화면 캡처, 인쇄, 프린트 워터마크**는 **DiskLock Plus** 기능에 의해 제어되는 작업으로, 각각의 DiskLock Plus 정책에 의해 작업이 **허용**(워터마크의 경우 **미출력**)된 경우에 한하여, 보안등급별 문서 작업 권한에 따라 **금지**(또는 **출력**) 여부가 결정됩니다. 따라서, 위와 같이 **금지**로 설정된 문서 작업 권한이 확대 적용되는 상황 또한 각 **DiskLock Plus 정책**에 의해 작업이 허용된 경우에만 발생할 수 있으며, DiskLock Plus 정책에서 작업이 금지된 경우에는 보안등급과 관계없이 애플리케이션이 열람하는 모든 문서에서 작업이 금지됩니다.

특별히,**원본 문서**의 내용을 복사하여 **대상 문서**에 붙여 넣는 **클립보드 복사** 작업의 경우에는 원본 문서와 대상 문서를 각각 열람하는 애플리케이션에 따라 클립보드 복사 금지 여부가 달라지므로, 보다 다양한 케이스가 발생할 수 있습니다. 자세한 내용은 [**보안등급문서 열람 시 클립보드 복사 금지 권한과 클립보드 복사 제어 정책의 적용 사례**](undefined-9.md)를 참고합니다.
